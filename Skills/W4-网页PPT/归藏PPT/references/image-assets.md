# 图片素材规范

本文件用于指导 PPT 中的图片素材如何选择、裁切、命名和放入模板。WorkBuddy 版本不包含图片生成提示词,不要求 AI 调用图像模型。

## 通用规则

- 先判断当前 deck 风格:风格 A = 电子杂志 × 电子墨水;风格 B = 瑞士国际主义 / Swiss Style。
- 图片是嵌入 PPT 的素材,不是一张独立 slide;图片内部不要自带页眉、页脚、页码、标题栏、角标、署名、装饰边框或 slide chrome。
- 信息图、图表、截图中的文字语言必须跟随 deck 语言:中文 deck 用中文,英文 deck 用英文。
- 图片要给标题或正文留出可叠加空间,不要满屏堆细节。
- 同一页或同一组图片必须使用同一比例、同一视觉缩放、同一边距密度。
- 图片放到 `images/`,命名为 `{页号}-{语义}.{ext}`,例如 `03-product-ui.png`。

## 比例选择

| 用途 | 推荐比例 | HTML 落位 |
|------|---------|-----------|
| 章节封面 / 全屏主视觉 | 16:9 | `.frame-img.r-16x9` 或 hero 背景参考 |
| 瑞士风顶部横幅 / Image Hero | 21:9 | S22 顶图 cover / `.frame-img.r-21x9` |
| 左文右图主图 | 16:10 或 4:3 | `.frame-img.r-16x10` / `.frame-img.r-4x3` |
| 信息图 / 系统关系图 | 16:9 或 16:10 | 原始截图用 `.fit-contain`;标准比例素材用对应 `.frame-img.r-*` |
| 截图 / UI 情景图 | 16:10 或 21:9 | 文字密集图用 `.fit-contain`;标准化图用 `.frame-img.r-16x10` / `.frame-img.r-21x9` |
| 图文混排小图 | 3:2 或 3:4 | `.frame-img.r-3x2` / `.frame-img.r-3x4` |
| 图片网格 | 统一横图 | `.frame-img.h-22` / `.frame-img.h-26` |
| 小型面板组 | 统一横图 | `.frame-img.h-16` / `.frame-img.h-18` |

信息图和截图如果来自不可控原始素材,优先用 `fit-contain`,避免文字被裁切。纪实照片优先用默认 `cover`,保持画面张力。

## 图片标准化策略

### A. 先选目标槽位

不要先拿图片硬塞页面。先决定图片落位:

1. 主视觉:16:9
2. 瑞士风 S22 顶图:21:9
3. 左文右图:16:10 或 4:3
4. 信息图/截图:16:9 或 16:10,并根据文字密度选择 `fit-contain`
5. 多图网格/面板组:统一高度类,同一组内禁止混用高度

### B. 用户原始图片/截图的处理

原始截图比例通常不可控,不要直接作为最终视觉标准。按下面顺序处理:

1. 如果原图比例接近目标槽位,直接放入统一 `.frame-img` 中,用 `cover` 或 `fit-contain`。
2. 如果原图过高、过窄、过长,优先拆成 2-3 张同尺寸局部面板。
3. 如果一张 UI 图被拉成巨长条,不要做成单张横幅;改用多图网格或局部截图。
4. 如果必须保留原图,用 `fit-contain` 放进统一 frame,接受留白,不要裁掉关键文字。

## 类型建议

### 类型 1: 人文纪实照片

用于增加现场感、情绪和真实世界锚点。适合风格 A 的图文页,也可以作为风格 B 的 S22 顶图。

放置建议:
- 风格 A:16:9 / 16:10 / 4:3,照片默认 `cover`。
- 风格 B:S22 顶图优先 21:9,主体放中央安全区,HTML 用 `object-position:center 35%` 或 `center center`。

### 类型 2: 信息图 / 流程图 / 系统关系图

用于解释概念、流程、对比、系统关系。图片内部只保留核心图形和必要短标签。

放置建议:
- 文字密集时使用 `.fit-contain`。
- 白底图不要放进灰色容器;容器背景用 `var(--paper)`。
- 不要额外套阴影、圆角或厚边框。

### 类型 3: 截图 / UI 情景图

用于呈现真实产品、代码、设计稿、工作区或数据看板。

放置建议:
- 单张截图优先 16:10。
- 过长截图拆成多张同尺寸面板。
- 文字密集截图使用 `.fit-contain`。
- 同组 UI 图统一高度、边距和缩放。

### 类型 4: 数据视觉 / 指标图

用于突出一个关键数字或少量指标。优先使用模板内的 KPI / bar / matrix 组件,不要把所有数据都做成图片。

放置建议:
- 需要真实图像证据时放入 S15/S16 网格。
- 纯数据表达优先用 HTML 结构,可编辑且更清晰。

## 风格 B:瑞士国际主义图片素材规则

当 deck 选择 `assets/template-swiss.html` / `layouts-swiss.md` 时,图片必须服从原始登记版式:

- 单张大图用 S22 Image Hero。
- 多图用 S15/S16 的原始网格骨架改造。
- 图片容器直角、无阴影、无圆角。
- 只使用一个 accent 色;不要在图片里混入额外高饱和色。
- 图片内部不要生成 PPT 外壳:不要页眉、页脚、页码、标题栏、角标、署名、外框。
- UI / 信息图文字必须短,保持中文/英文语言一致;真实照片尽量不要带文字。
- 21:9 图片必须让核心主体落在中央 70% 安全区,四周留白;不要把人脸、关键节点或 UI 文字贴边。
