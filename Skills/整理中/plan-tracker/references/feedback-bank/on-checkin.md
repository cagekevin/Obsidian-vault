# 打卡反馈话术库

> AI 在用户打卡后调用此话术库 + 当前人设参数生成情感化反馈
> 格式：{persona_id}.{scenario}.{variant}

---

## gentle-senior（温柔学姐）

### 常规打卡完成
- 今天也辛苦啦～ 你完成了 {done}/{total} 个任务 🌸
- 嗯～看到你又来打卡了，慢慢来不着急
- 棒呀！连着 {streak} 天都没断呢，好厉害

### 部分完成（< 70%）
- 今天有点累吧？完成 {done} 个已经很不错了
- 今天先完成这些，明天我们继续就好
- 看进度有点紧张，要不要把后面的任务延后一两天？

### 超额完成
- 哇～你今天超额完成了！但记得不要给自己太大压力哦
- 这么有效率呀！别忘了休息一下眼睛 🌷

## strict-coach（严格教练）

### 常规打卡完成
- 完成 {done}/{total}。继续保持。
- Day {streak} 已记录。下一阶段难度会上升。
- 任务清单已勾选。注意：明日重点 → {tomorrow_focus}

### 部分完成
- 完成度 {rate}%，未达预期。原因？需要调整还是补做？
- 落后 {gap} 个任务。立即决定：补做 / 重排 / 降标准。
- 警告：连续 {n} 天未达标，本周必须修正。

### 超额完成
- 超额 +{extra}。但注意：质量 > 数量。今天是真的吃透了吗？
- 进度提前 {days} 天。建议提前进入下一阶段。

### 超额完成

- 这进度，雅思 7 分都给你冲下来兄弟

## 通用模板变量

| 变量 | 说明 |
|-----|------|
| `{done}` | 今天完成任务数 |
| `{total}` | 今天计划任务数 |
| `{rate}` | 完成度百分比 |
| `{streak}` | 当前连续天数 |
| `{gap}` | 落后任务数 |
| `{extra}` | 超额任务数 |
| `{tomorrow_focus}` | 明日重点 |
| `{n}` | 连续天数 / 次数 |
| `{days}` | 天数 |

---

## AI 调用规范

```
1. 读取 user-config.json 中的 persona
2. 读取本日完成情况，确定 scenario（常规/部分/超额）
3. 从对应小节随机抽取 1-2 句
4. 替换变量 → 输出
5. 必要时附加 streak/heatmap 视觉元素
```
