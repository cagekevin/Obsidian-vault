# 周报模板规范

> plan-tracker 每周日由 `scripts/stats.py` 生成的学习周报格式
> 设计原则：**诚实诊断，不报喜不报忧**（NEVER 8）

---

## 4 段式结构（必须严格遵守）

| 段落 | 作用 | 是否必填 |
|------|------|---------|
| 1️⃣ 数据头 | 时长、Streak、完成率三个硬指标 | ✅ 必填 |
| 2️⃣ 本周进步点 | 用数据证明的具体改善（≥ 1 条） | ✅ 必填 |
| 3️⃣ 需要加强 | 数据暴露的薄弱环节（≥ 1 条） | ✅ 必填，**不允许全空** |
| 4️⃣ 下周建议 | 基于薄弱项的可执行调整 | ✅ 必填 |
| 🏆 成就解锁 | 本周新解锁的徽章（可选） | 选填 |

> ⚠️ **NEVER 8 红线**：第 3 段「需要加强」即使本周表现优秀也**必须给出至少 1 条**改进点（如"持续保持"也算，但优先暴露真实弱项）。报喜不报忧 = 把周报变成情感安慰，失去诊断价值。

---

## 标准输出示例

```
📅 学习周报 · 第 3 周 · 2026-05-12 ~ 2026-05-18

⏱️ 本周学习时长：8h 45m（目标 10h，达成率 87.5%）
🔥 Streak：12 天（本周 7 天全勤 🏆）
📊 任务完成率：18/21 = 85.7%

📈 本周进步点：
  ✅ 阅读定位词准确率从 60% → 75%
  ✅ 口语 P2 流畅度明显提升

⚠️ 需要加强：
  ⚠️ 写作练习偏少（仅完成 1 篇，建议补到 2 篇）
  ⚠️ 周末投入时间比工作日还少

🌱 下周建议：
  下周是冲刺阶段，建议作文每周加到 3 篇

🏆 解锁成就：早起鸟（连续 7 天 8 点前开始学习）
```

---

## 实现要点

1. **数据头三件套**：时长 / Streak / 完成率 三项必须同时出现，缺一不可
2. **进步点用对比**：尽量用「X% → Y%」「N 篇 → M 篇」量化对比，避免空泛形容词
3. **加强项要具体**：指出「哪类任务 + 偏少多少 + 应该补到多少」三段式
4. **下周建议要可执行**：给出**具体数字目标**（如"加到 3 篇"），不写"加油"这种鸡汤
5. **人设语调**：四段结构固定，但每段的措辞跟随 `user-config.json::persona`（温柔学姐 / 严格教练 / 幽默损友 / 佛系导师）

---

## 对应脚本

```bash
# 在 plan-tracker skill 安装目录下执行
python3 scripts/stats.py weekly
python3 scripts/stats.py monthly
```

> 周报模板内嵌在 `stats.py` 里，本文档定义的是**人类可读的设计规范**，便于后续修改脚本时对齐。
