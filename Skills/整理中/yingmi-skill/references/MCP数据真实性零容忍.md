# MCP 数据真实性（零容忍·共用）

适用于依赖盈米 MCP 生成 HTML/结构化报告的场景技能。**各技能 `SKILL.md` 中的「样式与数据隔离」在引用本节基础上，可再列本域核验清单（工具语义、表格裁剪等）。**

---

## §1 通用条款（fund-analyst / fund-screener / market-morning-brief 等）

1. **禁止估算/插值/推导**：当 MCP 返回的原始数据量大或无法整段嵌入时，必须编写脚本提取所需数据点，严禁凭趋势估算、线性插值或用系数推导缺失数据。
2. **禁止超出工具返回的推断**：描述性文本（行业、风格、持仓特征等）仅可使用 MCP **明确返回的字段值**，禁止凭常识或经验补充未返回的信息。
3. **分类口径必须匹配**：
   - **基金场景**：涉及「同类平均/同类排名」时，必须使用各基金**自身分类**对应的 `similarAvgReturn` 等字段，禁止混用不同基金类别的同类均值。
   - **家庭财务场景**：收支与资产负债分类必须与工具入参及 **§3** 一致，禁止混口径计算。
4. **图表数据全量真实**：嵌入 ECharts（或其他图表）的数值数组必须 **100%** 来自 MCP 返回值或用户输入，禁止包含任何 AI 估计值。
5. **AI 文本定量引用必须可溯源**：报告中涉及具体数值的表述须能追溯到工具返回字段或由本次数据直接计算得出。
6. **缺失处理**：数据无法获取时标注「数据暂未获取」，严禁用估算值填充；若整列/整行均无数据，按各技能「空列/空行裁剪」规则处理（见 fund-analyst 等 SKILL）。

---

## §2 组合诊断补充（portfolio-doctor）

在遵守 §1 前提下，对**大规模净值序列**须额外遵守：

- 当 `BatchGetFundNavHistory` 等返回大量日频数据时，必须用脚本（Node.js/Python 等）抽取展示所需粒度（如月末净值、对齐交易日），**禁止**凭 beta/相关系数或手工「补齐」缺失基金的净值序列。

---

## §3 家庭财富补充（wealth-family-advisor）

在遵守 §1 前提下，**计算口径**须额外遵守：

- 收支分类（必要/弹性等）、资产分类（流动性/投资性/自用性等）必须与 MCP 工具入参及 [planning-playbook.md](../../wealth-family-advisor/references/planning-playbook.md) 约定一致，禁止跨分类标准混加混比。
