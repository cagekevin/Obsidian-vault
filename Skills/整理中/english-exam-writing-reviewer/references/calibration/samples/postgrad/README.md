## 二、设计策略：从锚点档位 → 全档覆盖（A 节优先）

### 策略演进

| 版本 | 考研策略 |
|------|---------|
| v1.1 | 锚点档位（英一 A 2 档 + 英一 B 3 档 + 英二 A 1 档 + 英二 B 2 档 = 8 篇）|
| v1.2 | 补文体分支 + 档次锁点（+6 篇 → 14 篇）|
| **v1.5** | **A 节全 5 档覆盖**（英一 A 补第五/三/一档 + 英二 A 补第五/三/一档 = +6 篇 → **20 篇**）|

### 样例的 4 个锚点类型

1. **Top anchor**（第五档/满分参考）★v1.5 补齐：
   - 用途：Skill 判"非常好地完成"时有明确对照，避免"趋中倾向"错失满分
   - 样例：`postgrad1a-05-letter`、`postgrad1b-05-cartoon`、`postgrad2a-05-notice`、`postgrad2b-05-chart`

2. **High anchor**（第四档）：
   - 用途：Skill 判"达到考研良好水平"时有明确对照
   - 样例：`postgrad1a-04-letter`、`postgrad1b-04-descriptive`、`postgrad2a-04-notice`、`postgrad2b-04-chart`

3. **Mid anchor**（第三档，典型中等作文）：
   - 用途：帮助 Skill 区分"基本完成"与"较好完成"的边界
   - 样例：`postgrad1a-03-letter` ★v1.5、`postgrad1b-03-cartoon`、`postgrad1b-03-expository`、`postgrad2a-03-notice` ★v1.5、`postgrad2b-03-chart`

4. **Bottom anchor**（第一/二档，规则触发 + 极低档）★v1.5 新增第一档：
   - 用途：Skill 判极低档次时避免"同情加分"，同时校准扣分叠加机制
   - 金标样例：
     - `postgrad1a-02-letter`：Directions 原句照搬（15 词连续）→ 触发扣分
     - `postgrad2a-02-notice`：Directions 照搬（10+9 词双段）+ 署名违规
     - `postgrad1b-01-cartoon`：字数不足（78 词 vs 160）→ 触发降档
     - `postgrad2b-01-chart`：数据错位 4 处 + 字数 52% 不足
     - `postgrad1a-01-letter` ★v1.5：偏题 + 签名错 + 6 grammar critical + 字数 44% 不足（4 类 critical 叠加）
     - `postgrad2a-01-notice` ★v1.5：偏题 + 格式 4 重违规 + 8 critical + 字数 42% 不足

## 三、与 CET 校准集的交叉引用

### 跨试卷同档位对比（读者可自行对照）

| 题型 | CET 样例 | Postgrad 样例 | 差异点 |
|------|---------|--------------|-------|
| 议论文（高）| `cet6-14-band-01.md` | `postgrad1b-05-band-cartoon-01.md` | 考研要求**论述性**而非议论性 |
| 应用文书信（中）| `cet4-11-band-letter-01.md` | `postgrad1a-04-band-letter-01.md` | 考研 A 节有 **Directions 照搬检测** |
| 图表题（中/高）| `cet4-11-band-chart-01.md` | `postgrad2b-05-band-chart-01.md` | 考研英二 B 节把**数据精准度**作为独立观察点 |

Skill 在 `prompt` 中自动引用**对应级别**的样例作为 few-shot，不要混引跨考试的样例。

## 四、文件命名规范

```
postgrad{1|2}{A|B}-{band}-band-{subtype}-{serial}.md
```

示例：

- `postgrad1a-04-band-letter-01.md`：考研英语**一**、**A 节**、**第四档**、**书信**、第 1 篇
- `postgrad2b-03-band-chart-01.md`：考研英语**二**、**B 节**、**第三档**、**图表**、第 1 篇

**task_subtype 枚举**（与 output-schema.md 一致）：

- A 节：`letter` / `notice` / `announcement` / `memorandum`
- B 节：`cartoon`（英一）/ `chart`（英二）
