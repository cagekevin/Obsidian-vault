# 输出文档格式规范

## 完整输出模板

```markdown

> 🗓️ 生成时间：YYYY-MM-DD HH:MM　　📦 素材总数：N 项　　✅ 成功处理：M 项
```

```markdown
|------|------|
| **来源** | [期刊/会议名](URL) |

**核心贡献**：

> "..."
```

## 批次进度播报格式

每完成一批处理后，在正文前输出进度行（不计入最终文档）：

```
✅ 第 K/N 批处理完成（已处理 M 项，剩余 R 项）→ 继续处理下一批...
```

## 文件保存规范

研究笔记输出完毕后，使用 `write_to_file` 保存：

```
主文档：research-notes-{主题关键词}-{YYYYMMDD}.md
关键词索引：keyword-index-{主题关键词}-{YYYYMMDD}.md
```

关键词索引文件格式：

```markdown
# 🔑 关键词索引：[主题描述]

> 生成时间：YYYY-MM-DD HH:MM　　来源文档：research-notes-{主题}-{YYYYMMDD}.md

## 按频次排序

| 关键词 | 出现条目 | 频次 |
|--------|:-------:|:----:|
| `#标签1` | #1, #3, #4 | 3 |
| `#标签2` | #1, #2 | 2 |

## 按主题分组

### 领域标签
- `#AI`（3次）：条目#1, #3, #4
- `#教育`（2次）：条目#2, #5

### 类型标签
- `#研究报告`（2次）：条目#1, #3
- `#工具`（1次）：条目#4
```

## 合并条目格式

当两条素材被去重合并时，在条目末尾添加合并说明行：

```markdown
- **合并说明**：`🔀 已与条目 #N 合并（相似度 XX%）`
```

合并说明独立成行，不混入核心观点列表。

## 视觉美化规范

1. **分隔线**：每个主题节之间使用 `---` 分隔
2. **表格对齐**：数值列右对齐（`:---:`），文字列左对齐
3. **标签格式**：使用反引号包裹 `` `#标签` ``，便于视觉识别
4. **可信度标记**：紧跟观点文字末尾，不换行
5. **来源信息**：优先使用表格展示，比列表更整洁
6. **文档页脚**：使用 `<div align="center">` 居中显示生成信息
7. **目录**：使用表格形式展示，比列表更直观，含条目数统计
