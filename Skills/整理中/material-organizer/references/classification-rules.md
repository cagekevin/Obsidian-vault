# 主题分类规则

## 分类模式

### 模式 A：用户指定分类（优先）

若用户在输入时指定分类维度，严格按指定维度分组：

| 用户指令示例 | 分类维度 |
|------------|---------|
| "按技术栈分类" | 前端 / 后端 / AI / 数据库 / DevOps |
| "按时间线归档" | 2022年 / 2023年 / 2024年 / 2025年 |
| "按重要程度" | 必读 / 参考 / 备用 |
| "按项目分类" | 项目A / 项目B / 通用 |
| "按内容类型" | 教程 / 论文 / 工具 / 数据 |

### 模式 B：自动聚类（默认）

未指定分类时，按以下优先级自动聚类：

1. **标签聚类**：将共享 ≥2 个标签的条目归为同一主题
2. **语义聚类**：基于核心观点的语义相似性分组
3. **来源聚类**：同一网站/作者的多篇内容可归为一组（可选）

## 自动分类参考维度

| 维度 | 典型分类 |
|------|---------|
| **学科领域** | 计算机科学 / 经济学 / 心理学 / 医学 / 教育学 |
| **内容类型** | 研究报告 / 教程指南 / 工具资源 / 数据集 / 案例分析 |
| **应用场景** | 学习资料 / 工作参考 / 项目调研 / 竞品分析 |
| **技术栈** | Python / JavaScript / AI/ML / 云计算 / 数据库 |
| **时效性** | 最新（≤1年）/ 经典（>3年）/ 时效不明 |

---

## 分类输出规范

每个主题分类包含：

```markdown
## [主题图标] 主题名称（N条）

> 主题简介：一句话描述该分类的核心内容

[条目列表...]
```

主题图标参考：
- 🔬 研究/学术
- 🛠️ 工具/技术
- 📊 数据/统计
- 📚 学习/教程
- 💡 观点/洞察
- 🏢 行业/商业
- 🌐 资讯/新闻
- 📌 综合/其他

---

## 分类数量控制

| 素材数量 | 建议分类数 |
|---------|----------|
| 1-5 项 | 1-2 个主题 |
| 6-15 项 | 2-4 个主题 |
| 16-30 项 | 3-6 个主题 |

若自动聚类产生 > 8 个主题，合并相似主题，确保每个主题 ≥ 2 条内容。

---

## 内容路由优先级（来自 knowledge-base 最佳实践）

分类时优先按内容实质路由，而非仅按标题：

| 内容类型 | 路由到 | 说明 |
|---------|--------|------|
| 工具/软件/库 | 工具类主题 | 应用场景为主要维度 |
| 文章/概念/研究 | 内容类主题 | 按内容匹配，不仅看标题 |
| 数据集/报告 | 数据类主题 | 单独归类，便于后续引用 |
| 无法判断类型 | → 收件筐 | 待用户确认后再归类 |

---

## 未分类处理（Inbox 机制）

无法归入任何主题的条目，放入「📥 收件筐」分类，并说明原因：

```markdown
## 📥 收件筐（待归类）

> 以下条目内容过于独特或信息不足，无法自动归类，建议用户手动分配。

### [N]. [条目标题]
- **来源**：...
- **建议归类**：[AI 推测的可能主题]
- **待归类原因**：内容过短 / 主题过于独特 / 访问失败
```
