# Combat 风格技能模板

轻量级 pipeline 技能的标准结构。适合**步骤明确、有分支、需要防呆**的技能类型。

## 适用场景

- 用户有一组固定输入，需要按顺序转换/处理
- 涉及用户确认 checkpoint（不可跳过)
- 有异常/边界情况需要 fallback 处理
- 需要明确告诉 AI"什么算做完"、"什么时候该问人"

不适合：纯信息查询、简单生成器、纯审查类技能。

## 0. 什么时候启动

收到 [核心输入文件] 就启动。用户说"[触发词1]""[触发词2]"也算。

**找到 [输入文件]：** 按顺序试：
1. 用户对话中给了路径
2. 当前工作目录下搜索 `**/[文件名]`
3. 用户打开的文件中找
4. 都没找到 → 问用户路径

**如果内容不符合要求**（如纯剧情无画面描述）→ 打印说明，停下等用户。

**管线顺序不能乱：**
```
[步骤1]
  ↓
[步骤2] → 打印结果 → 用户确认
  ↓
[步骤3]
  ↓
...
```

## 与标准模板的对比

| 维度 | 标准 template | Combat 模板 |
|------|-------------|-------------|
| 行数 | 取决于复杂度 | ~130-180 行 |
| 适合 | 复杂多模式、多领域 | 单 pipeline + 分支 |
| 用户确认点 | 可选 | 标配 |
| 完成标准 | 可选 | 必有 |
| 退路表 | 可选 | 必有 |
| 不碰清单 | 可选 | 必有 |
