# 咨询分析框架库

根据问题类型选择合适的分析框架：

## MECE 分解（Mutually Exclusive, Collectively Exhaustive）
**适用场景：** 问题拆分、原因分类
**用法：** 将问题拆分为互不重叠且合在一起完整的子项
**示例：** 收入下降 → 客单价 × 客户数 → 分别找原因

## 金字塔原理 (Pyramid Principle)
**适用场景：** 方案汇报、观点论证
**用法：** 结论先行 → 三个支撑论点 → 每个论点有论据
**示例：** 建议裁员 → 原因1/原因2/原因3 → 每项有数据支撑

## 第一性原理 (First Principles)
**适用场景：** 创新方案、颠覆性思考
**用法：** 拆掉所有假设 → 找到不可拆的基础事实 → 重新推理
**示例：** 马斯克造火箭 → 原材料成本占火箭总价 2% → 自己做

## SWOT 分析
**适用场景：** 战略定位、竞争分析
**用法：** 内部优势/劣势 + 外部机会/威胁 → 四象限交叉

## 5 Whys（五问法）
**适用场景：** 根因分析、故障排查
**用法：** 连续问 5 次"为什么"，剥到根本原因

## 影响/可行性矩阵
**适用场景：** 方案排序、优先级决策
**用法：** 横轴=可行性，纵轴=影响 → 右上角优先
