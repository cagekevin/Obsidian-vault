## 9 步完整流程

```
用户输入 → 意图识别 → 参数提取 → 问询(缺失) → 工具选择 → Prompt构建 → generate_media → 等待 → 输出
```

## 模型路由决策树（3 层优先级）

| 优先级 | 规则 | 示例 |
|-------|------|------|
| P1 | 用户显式指定 > 一切 | "用MJ画" → midjourney |
| P2 | 任务特征匹配 | "插画/艺术" → midjourney；"高清/4K" → gpt_image_2 high |
| P3 | 系统偏好 fallback | 无特征 → prefer_models[0] |

## 6 条路由规则

| # | 规则 |
|---|------|
| R1 | 用户显式指定 > 一切 |
| R2 | 质量关键词 → gpt_image_2 high |
| R3 | 速度关键词 → nano_banana_2 / gpt_image_2 low |
| R4 | 艺术关键词 → midjourney |
| R5 | 无特征 → fallback 系统偏好 |
| R6 | tool_hint 格式必须严格匹配 |

## tool_hint 解析流程

```
"generate_image_nano_banana_2"
→ split('_', 2) → ["generate", "image", "nano_banana_2"]
→ prefix=generate, task_type=image, model_name=nano_banana_2
→ MODEL_REGISTRY lookup → 获取配置（服务地址、参数规则）
```

## 参数转换映射表

| 通用参数 | nano_banana_2 | gpt_image_2 | midjourney |
|---------|---------------|-------------|------------|
| prompt | text | prompt | prompt |
| aspect_ratio | 计算 w/h | 原生支持 | 转 --ar |
| quality=high | 忽略 | quality=high | 转 --q 2 |
| negative_prompt | negative_text | negative_prompt | 转 --no |
| reference_image | 支持(最多3张) | 不支持 | 不支持 |
| seed | 支持 | 支持 | 不支持 |

## 模型预处理差异

**nano_banana_2:** 自动加 "masterpiece, best quality" 前缀，输出尺寸 64 的倍数，固定 30 步
**gpt_image_2:** 保持 prompt 原样，支持 quality 参数，支持精确尺寸
**midjourney:** 转 Discord 命令格式，异步轮询，不支持参考图

## 错误处理
- 超时 → 重试 1 次 → 仍失败返回
- 模型不可用 → 备用模型（nano_banana_2 → seedream_v5）
- 安全拦截 → 不重试，直接返回
- 参数错误 → 不重试，返回具体错误
