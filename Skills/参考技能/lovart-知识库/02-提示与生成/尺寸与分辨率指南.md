我来详细说明图像和视频处理中的尺寸/分辨率决策规则，以及各平台的默认规格。
---
## 一、尺寸参数体系
| 参数 | 用途 | 示例值 |
|------|------|--------|
| `target_size` | 精确像素尺寸 | `"1920x1080"`, `"1024x1024"` |
| `aspect_ratio` | 宽高比 | `"16:9"`, `"9:16"`, `"1:1"`, `"3:4"` |
| `resolution` | 分辨率档位 | `"4K"`, `"2K"`, `"1080p"`, `"720p"` |
---

## 四、用户未指定尺寸时的判断逻辑
### 决策优先级（从高到低）
```
1. 平台关键词 → 按平台默认规格
   例："做个小红书封面" → aspect_ratio: "3:4"
2. 内容类型推断 → 按内容最佳实践
   - 视频/动态内容 → 16:9 或 9:16
   - 产品展示 → 1:1 或 3:4
   - 海报/横幅 → 根据用途判断
3. 参考图片 → 继承参考图尺寸
   - 如果用户提供了参考图，分析其尺寸并沿用
4. 通用默认 → 安全中间值
   - 图片默认: 1024×1024 (1:1) 或 1024×1536 (2:3)
   - 视频默认: 1080p, 16:9
```
### 内容类型默认映射
| 内容描述 | 默认尺寸策略 |
|---------|-------------|
| "海报" / "poster" | 根据用途：竖版海报 2:3，横版海报 3:2 |
| "头像" / "avatar" | `target_size: "512x512"` 或 `aspect_ratio: "1:1"` |
| "banner" / "横幅" | `aspect_ratio: "16:9"` 或 `target_size: "1920x600"` |
| "壁纸" / "wallpaper" | 询问设备类型，或默认 9:16（手机）/ 16:9（电脑）|
| "商品图" / "产品图" | `aspect_ratio: "1:1"` 或 `aspect_ratio: "3:4"` |
| "插画" / "illustration" | 根据内容方向判断，或询问偏好 |
| "社交媒体配图" | 询问具体平台，或默认 1:1 |

## 五、关键执行规则
| 规则 | 说明 |
|-----|------|
| **零默认规则** | 用户明确或暗示尺寸要求时，**严禁**使用工具默认值 |
| **参数拦截** | 调用生成工具前必须扫描尺寸关键词 |
| **具体转换** | 自然语言尺寸必须转为精确像素或宽高比 |
| **多尺寸冲突** | 优先使用 `target_size` > `aspect_ratio` > `resolution` |
---
## 六、实际应用示例
| 用户请求 | 参数设置 |
|---------|---------|
| "生成一张小红书封面" | `aspect_ratio: "3:4"` |
| "1920×1080的电脑壁纸" | `target_size: "1920x1080"` |
| "4K海报" | `resolution: "4K"`, 询问横竖版 |
| "抖音竖版视频" | `aspect_ratio: "9:16"`, `resolution: "1080p"` |
| "淘宝主图，800×800" | `target_size: "800x800"` |
| "做个海报"（无尺寸） | 询问：横版还是竖版？用途是什么？ |
---
**总结**：尺寸决策的核心是**从用户输入中提取明确信号**，平台关键词和内容类型是主要推断依据，无法确定时必须询问用户，绝不擅自假设。
