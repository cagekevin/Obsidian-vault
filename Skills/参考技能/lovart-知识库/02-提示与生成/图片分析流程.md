我来详细说明我分析用户上传图片的完整流程和判断逻辑：
---
## 图片分析流程
### 第一步：视觉扫描（整体感知）
- **尺寸与比例**：首先识别图片的宽高像素、长宽比（16:9、1:1、9:16等）
- **画面类型判断**：是照片、插画、设计稿、截图、还是AI生成图？
- **色彩基调**：整体色调（暖色/冷色/中性）、饱和度、明暗对比
### 第二步：内容解构（细节提取）
**主体识别**
- 画面核心是什么？（人物/产品/风景/建筑/抽象图形）
- 主体数量、位置关系、主次层次
**场景与环境**
- 背景内容：室内/室外、自然/人工环境
- 空间深度：前景、中景、背景的层次
**视觉元素清单**
- 文字内容（如有）：字体风格、排版位置、语言
- 品牌元素：Logo、标志性配色、特定视觉符号
- 材质与纹理：金属、布料、木纹、玻璃等
- 光影效果：光源方向、阴影类型、氛围感
### 第三步：语义理解（意图推断）
**判断图片的用途/来源**
- 是参考图（想要模仿风格）？
- 是底图（想要在此基础上修改）？
- 是素材（想要提取其中元素）？
- 是成品（想要分析或复刻）？
**识别用户的潜在需求**
- 从对话上下文推断：用户为什么上传这张图？
- 结合用户文字描述：是要编辑、参考、还是分析？
---
## 判断逻辑与决策树
```
用户上传图片后，我会问自己：
1. 用户是否明确说了要对这张图做什么？
   ├─ 是 → 按明确指令执行（编辑/参考/分析）
   └─ 否 → 进入推断模式
2. 用户文字中是否出现这些关键词？
   ├─ "改成"、"换成"、"去掉"、"添加" → BASE角色（编辑目标）
   ├─ "参考"、"风格"、"类似"、"按照" → STYLE_REF角色
   ├─ "放入"、"合成"、"背景" → 可能是SUBJECT_REF或BASE
   └─ 无明确指向 → 询问用户意图
3. 图片本身特征如何？
   ├─ 完整成品图 → 可能是参考或分析对象
   ├─ 局部/有缺陷 → 可能是需要修复的BASE
   └─ 风格明显独特 → 可能是STYLE_REF
```

## 实际操作示例
| 场景 | 我的分析 | 参数路由 |
|------|---------|---------|
| "帮我把这张图背景换成森林" | 明确编辑指令，此图=要修改的对象 | `source_image_url` |
| "参考这张图的风格，画一只猫" | 风格参考，此图=风格指南 | `reference_image_urls` |
| "把图1的人物放到图2的场景里" | 图1=SUBJECT_REF, 图2=BASE或LAYOUT_REF | 分别路由 |
| 只发图不说话 | 无法判断，必须询问用户意图 | 暂不调用工具 |
---
## 关键原则
1. **不猜测**：如果用户意图不明确，我会直接询问，而不是假设
2. **角色唯一性**：一张图在一次操作中只能有一个角色（BASE只能有一个）
3. **参数对应**：分析结果必须准确映射到工具参数，不能遗漏或错放
4. **上下文结合**：永远结合用户的文字描述来理解图片，不孤立分析
这个流程确保我能准确理解每张图片在用户任务中的作用，从而正确选择工具和参数。
