我来详细说明 asset_factory 的工作流程：
## 触发条件
asset_factory 在以下场景被调用：
**1. Kit 引用识别**
- 用户在请求中包含 `--kit="kit_id"` 标记
- 例如：「用 --kit="brand_001" 生成海报」
**2. 品牌一致性需求**
- 多图生成任务需要统一视觉风格
- 企业/品牌物料制作
- 需要复用已有品牌资产（Logo、配色、字体等）
---
## 检索流程
```
用户请求含 --kit → 提取 kit_id → asset_factory 调用 → 返回品牌资产包
```
**调用参数：**
- `asset_type`: 目前仅支持 `"brand_kit"`
- `asset_id`: 从 `--kit="xxx"` 中提取的标识符
- `user_intent`: 当前任务的具体需求（决定提取哪些资产）
- `language`: 响应语言
---
## 返回数据结构
asset_factory 返回的品牌资产包通常包含：
| 资产类型 | 内容示例 | 用途 |
|---------|---------|------|
| **Logo** | 主Logo、副Logo、图标变体 | 合成到最终输出 |
| **配色方案** | 主色、辅色、强调色 HEX 值 | 生成时控制色调 |
| **字体规范** | 标题字体、正文字体 | 文字渲染 |
| **视觉风格参考** | 示例图、 mood board | 风格迁移参考 |
| **品牌指南** | 使用规范、禁忌事项 | 约束生成逻辑 |
---
## 数据向下传递机制
**1. 传递给生成工具**
```
asset_factory 返回的图像 URL → 
generate_media / edit_media 的 reference_image_urls 参数
```
**2. 传递给子代理（visual_planner）**
```
品牌资产描述 + URL → project_context 中的 [LOGO] / [STYLE] / [COLOR] 标签
```
**3. 融入提示词**
- Logo：明确描述「保留 Logo 设计不变」
- 配色：将 HEX 值写入 prompt（如「使用 #1A237E 作为主色」）
- 风格：提取视觉特征写入生成指令
---
## 完整工作流示例
```
用户：用 --kit="morning_coffee" 生成 4 张产品场景图
↓
asset_factory(
  asset_type="brand_kit",
  asset_id="morning_coffee",
  user_intent="生成产品场景图，需要品牌Logo和配色"
)
↓
返回：
- Logo: https://.../logo.png
- 主色: #2E7D32 (绿色)
- 辅色: #FFD700 (金色)
- 风格参考图: https://.../style.jpg
↓
generate_media(
  reference_image_urls="logo.png, style.jpg",
  prompt="产品场景图，使用品牌绿色 #2E7D32 和金色 #FFD700 配色，
          保留品牌Logo设计，晨光咖啡风格..."
)
```
---
## 故障处理
如果 asset_factory 检索失败：
1. 向用户报告失败原因
2. 询问是否继续（无 Kit）或提供替代 Kit ID
3. 绝不假设或虚构品牌资产
