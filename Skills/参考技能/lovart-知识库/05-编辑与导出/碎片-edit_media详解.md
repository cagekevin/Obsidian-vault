# Lovart 拼图碎片 — edit_media详解

## 话术

详细讲讲 edit_media 这个工具的内部工作逻辑。它和 generate_media 有什么区别？什么情况下用 edit 而不是 generate？编辑操作有哪些类型（换背景、去物体、改颜色、重绘）？每种类型的处理流程是什么？

## 回复

我来详细讲解 `edit_media` 工具的工作逻辑以及与 `generate_media` 的区别。
---
## 一、edit_media vs generate_media 核心区别
| 维度 | **edit_media** | **generate_media** |
|------|----------------|-------------------|
| **核心目的** | 在**现有图像基础上**进行局部或全局修改 | **从零创建**新图像或基于参考生成 |
| **输入要求** | **必须提供** `source_image_url`（被编辑的图像） | 可选提供参考图，无必须输入 |
| **主体状态** | **保持**原图主体的姿态、形态、位置不变 | 可以**改变**主体的姿态、角度、状态 |
| **修改范围** | 通常针对**局部区域**（<50%图像） | 通常是**全局生成** |
| **使用场景** | 修图、改背景、去物体、调色、局部重绘 | 创作新图、风格迁移、多图融合、变姿势 |
---
## 二、什么时候用 edit_media 而不是 generate_media？
### ✅ 用 edit_media 的情况：
1. **换背景** — 保留主体，替换环境
   - 例："把这张照片的人物放到海边"
2. **去除/添加物体** — 局部内容修改
   - 例："去掉画面中的路人"
   - 例："在桌子上加一杯咖啡"
3. **改颜色/调光影** — 不改变构图
   - 例："把衣服改成红色"
   - 例："调成暖色调黄昏光"
4. **局部重绘/修复** — 配合 mask 使用
   - 例："把模糊的脸修复清晰"
   - 例："重新画一下手部细节"
5. **尺寸调整** — 仅改变画幅
   - 例："把竖图改成横图"
   - 例："放大到4K分辨率"
### ✅ 用 generate_media 的情况：
1. **主体姿态需要改变**
   - 例："让这个人转过身来"
   - 例："生成这个角色的侧面视角"
2. **多视角/多姿势拼图**
   - 例："生成这个产品的三视图"
3. **从零创作**
   - 例："画一只在森林里的狐狸"
4. **风格迁移且主体也要变**
   - 例："把照片转换成油画风格，同时让人物微笑"
---
## 三、edit_media 的任务类型详解
### 1. **reframe（重构图幅）**
**用途**：改变图像的宽高比或精确尺寸，通过裁剪/扩展实现
**处理流程**：
```
输入图像 → 分析主体位置 → 智能裁剪/扩展 → 输出新尺寸图像
```
**适用场景**：
- 竖图改横图、横图改竖图
- 适配特定平台尺寸（如 Instagram 1:1、Stories 9:16）
- 精确像素尺寸输出
**参数**：
- `aspect_ratio`: "16:9", "9:16", "1:1", "3:4" 等
- `target_size`: 精确像素如 "1920x1080"
---
### 2. **upscale（超分辨率放大）**
**用途**：提升图像分辨率/清晰度，**不增加新内容**
**处理流程**：
```
输入图像 → AI 细节增强 → 分辨率提升 → 输出高清图像
```
**适用场景**：
- 小图放大用于印刷
- 提升低分辨率素材
- 需要 4K/2K 输出时
**参数**：
- `target_longest_size`: 目标长边像素（如 2048, 4096）
- `creativity`: 控制增强强度（通常 -2 到 -1，越低越保守）
---
### 3. **edit_image（图像编辑）**
**用途**：在保持主体姿态不变的前提下，进行创意修改
**处理流程**：
```
输入图像 + prompt 指令 → 识别编辑区域 → 局部重绘/修改 → 融合输出
```
**支持的编辑类型**：
| 编辑类型 | 示例指令 | 是否需 mask |
|---------|---------|------------|
| **换背景** | "把背景换成城市夜景" | 可选 |
| **去物体** | "去掉画面右边的汽车" | 推荐 |
| **加物体** | "在手中加一本书" | 可选 |
| **改颜色** | "把裙子改成蓝色" | 可选 |
| **改材质** | "把桌子变成大理石材质" | 可选 |
| **调光影** | "改成清晨的柔和光线" | 否 |
| **风格化** | "转换成水彩画风格" | 否 |
| **局部重绘** | "重新画一下脸部" | **必须** |
**mask_image_url 的使用**：
- 黑色区域 = 要修改的部分
- 白色区域 = 保持不变的部分
- 用于精确控制编辑范围，避免影响不该变的区域
---
### 4. **edit_video（视频编辑）**
**用途**：基于参考视频或帧图像生成/编辑视频
**处理流程**：
```
输入视频/帧 + prompt → 分析运动/风格 → 生成新视频 → 输出
```
**模式**：
- **参考视频模式**：提取参考视频的特征（人物、动作、风格）生成新视频
- **首尾帧模式**：提供开始帧和结束帧，AI 补全中间过渡
---
## 四、关键决策流程图
```
用户请求
    │
    ▼
是否有现成图像需要修改？ ──否──→ 用 generate_media（从零生成）
    │是
    ▼
修改范围是否 < 50% 且主体姿态不变？
    │
   是 ──→ 用 edit_media（编辑）
    │
   否 ──→ 用 generate_media（重新生成）
    │
    ▼
具体任务类型：
- 只改尺寸 → reframe/upscale
- 局部修改 → edit_image (+可选mask)
- 视频相关 → edit_video
```
---
## 五、最佳实践建议
1. **保护主体完整性**：如果用户说"保持这个人不变"，优先用 `edit_media`
2. **mask 的使用**：复杂编辑时提供 mask 能大幅提升精确度
3. **prompt 写法**：edit 的 prompt 要明确指出**什么变、什么不变**
   - ✅ 好："把背景换成海滩，人物保持完全不变"
   - ❌ 差："海滩场景"
4. **多步骤编辑**：复杂修改可拆分为多步（先去物体 → 再换背景 → 最后调色）
5. **分辨率策略**：先生成合适尺寸，最后再用 `upscale` 放大，避免直接生成超大图
