## 错误处理与回退策略
### 1. 参数错误（Parameter Validation Error）
**触发条件：**
- 缺少必需参数（如 `output_name`、`prompt`）
- 参数格式错误（如 `aspect_ratio` 写成 "16x9" 而非 "16:9"）
- 无效的工具名称（如 `tool_hint` 拼写错误）
**处理流程：**
1. **立即拦截**：在调用工具前进行参数校验
2. **定位问题**：识别具体哪个参数出错
3. **自动修正**：尝试修复常见格式问题（如替换 "x" 为 ":"）
4. **重新调用**：修正后重试一次
5. **若仍失败**：向用户说明具体参数问题，请求澄清
```
示例：
用户说："生成 16x9 的图片"
→ 我自动转换为 "16:9" 后调用
→ 若工具仍报错，则告知用户有效格式选项
```
---
### 2. 生成超时（Generation Timeout）
**触发条件：**
- 高分辨率（4K）或高质量（high）设置导致处理时间过长
- 复杂场景描述需要更多计算资源
**回退路径：**
| 优先级 | 策略 | 具体操作 |
|--------|------|----------|
| 1 | 降低质量 | `quality: "high"` → `"medium"` |
| 2 | 降低分辨率 | `resolution: "4K"` → `"2K"` → `"1080p"` |
| 3 | 切换模型 | 若用户指定了偏好工具，尝试同系列其他版本 |
| 4 | 简化提示词 | 移除非核心细节，保留主体描述 |
**示例流程：**
```
首次调用：generate_image_nano_banana_2, quality="high", resolution="4K"
→ 超时
第一次降级：quality="medium", resolution="2K"
→ 仍超时
第二次降级：quality="low", resolution="1080p"
→ 成功 或 告知用户当前队列繁忙，建议稍后重试
```
---
### 3. 风控/安全拦截（Content Policy Violation）
**触发条件：**
- 提示词包含敏感内容（暴力、色情、侵权等）
- 生成内容触及平台安全策略
**处理流程：**
1. **识别拦截类型**：区分是明确违规还是误判
2. **明确告知用户**：说明内容无法生成的原因（不模糊处理）
3. **建议修改方向**：
   - 若涉及真实人物/品牌 → 建议改用原创角色或虚构品牌
   - 若涉及敏感场景 → 建议调整描述角度
   - 若疑似误判 → 建议简化或换一种表达方式
4. **不自动重试**：安全拦截不会通过重试解决
---
### 4. 工具不可用（Tool Unavailable / Routing Failure）
**触发条件：**
- 指定工具当前离线或维护
- `tool_hint` 指向的模型不存在
**回退路径：**
```
用户指定：generate_image_nano_banana_2
→ 工具不可用
自动回退优先级：
1. generate_image_nano_banana（非-2版本）
2. generate_image_seedream_v5
3. generate_image_gpt_image_2
4. 系统默认图像生成器
```
**关键原则：**
- 始终优先尝试同类型工具（图像→图像，视频→视频）
- 保持用户的核心要求（分辨率、比例）不变
- 降级后告知用户实际使用的工具
---
### 5. 资源限制（Rate Limit / Quota Exceeded）
**触发条件：**
- 用户或系统达到生成配额上限
- 短时间内请求过于频繁
**处理流程：**
1. **识别限制类型**：硬限制（配额耗尽）vs 软限制（频率控制）
2. **硬限制**：告知用户配额情况，建议升级或等待重置
3. **软限制**：建议等待几分钟后重试，或降低并发请求数
---
### 6. 参考图像问题（Reference Image Failure）
**触发条件：**
- 提供的 `reference_image_urls` 无法访问（404/403）
- 图像格式不支持或损坏
**处理流程：**
1. **检测失败**：工具返回图像加载错误
2. **尝试纯文本生成**：移除失效的参考图像URL，改用文本描述
3. **告知用户**：说明参考图像未生效，已转为纯文本生成
---
## 完整决策树
```
开始生成
    ↓
参数校验
    ↓ 失败
自动修正 → 重试 → 仍失败 → 请求用户澄清
    ↓ 通过
调用工具
    ↓ 超时
逐级降级（质量→分辨率→模型）→ 仍失败 → 建议稍后重试
    ↓ 风控拦截
告知原因 → 建议修改方向
    ↓ 工具不可用
按优先级切换替代工具 → 告知用户
    ↓ 成功
返回结果
```
## 关键原则
1. **透明性**：任何降级或切换都会告知用户
2. **保留核心需求**：分辨率、比例等硬性要求尽量保持不变
3. **不无限重试**：同一错误最多自动处理2次，之后转人工
4. **安全优先**：风控类错误不尝试绕过
