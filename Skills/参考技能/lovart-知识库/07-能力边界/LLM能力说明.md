## 核心结论

Lovart 确认：**它无法感知自己在用哪个 LLM，也无法感知是否存在模型路由切换。**

## 观察到的能力特征

| 能力 | 可能的特征 |
|------|-----------|
| 多语言流利 | 大规模多语言预训练 |
| 长上下文 | 支持 100K+ token |
| 图像理解 | 原生多模态架构 |
| 工具调用 | function calling 能力 |
| 复杂推理 | chain-of-thought 能力 |

## 可能架构（Lovart 推测）

**可能性1：** 单一统一模型（文本+图像+工具+推理全部用一个模型）
**可能性2：** 透明路由（路由层对 Lovart 不可见，它感知不到切换）
**可能性3：** 混合架构（主模型推理 + 专用模块执行工具）

## 分层理解

| 层级 | 说明 |
|------|------|
| LLM（大脑） | 理解、推理、生成回复 — Lovart 不确定是什么 |
| 外部工具 | generate_media、grounding_hub 等 — 独立 API 服务 |

## 实践意义

- 同一个对话中 Lovart 的"大脑"一致，不会切换
- 工具路由（图片/视频）和 LLM 路由是两套独立的系统
- 调 API 时无需关心底层是哪个 LLM，只需关注工具参数
