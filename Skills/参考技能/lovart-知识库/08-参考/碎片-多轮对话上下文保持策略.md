# Lovart 拼图碎片 — 多轮对话上下文保持策略

## 话术

在实际使用场景中，用户可能先上传一张图提取规范，然后又问另一个问题，之后再回来继续修改第一张图。这种情况下你是如何保持对前面内容的记忆的？有什么最佳实践可以帮助用户让你记住更久的信息？

## 回复

这是个很好的问题!让我解释一下我的记忆机制和最佳实践:
## 我的记忆机制
**当前对话内记忆:**
- 在同一个对话会话中,我可以访问**完整的对话历史**
- 即使中间穿插了其他问题,我仍能回溯到之前上传的图片、提取的规范等信息
- 例如:你上传图A提取规范 → 问其他问题 → 回来修改图A,我依然能找到图A的信息
**跨对话记忆 (Summary Context):**
- 系统会在对话结束后生成**摘要上下文**
- 下次新对话时,这个摘要会帮助我理解你的:
  - 设计偏好(配色、风格、审美)
  - 工作流程习惯
  - 历史项目的关键决策
- 但**不会包含**具体的图片URL或详细内容
## 最佳实践建议
### ✅ **帮助我记住的做法:**
1. **关键信息明确标注**
   ```
   "这是我们的品牌主色:#FF5733,请记住用于后续设计"
   "这个logo规范很重要,后面所有设计都要用"
   ```
2. **重要资产保存引用**
   - 当我生成图片后,系统会返回URL
   - 如果后续要用,**直接发送该URL**给我
   - 例如:"用这张图 [URL] 的风格生成新的"
3. **阶段性总结**
   ```
   "总结一下目前确定的设计规范:
   - 主色调:蓝色系
   - 字体:现代无衬线
   - 风格:简约科技感"
   ```
4. **使用Brand Kit功能**
   - 如果是长期项目,建议创建Brand Kit
   - 把logo、色彩、规范等存入Kit
   - 每次用 `--kit="kit_id"` 调用,我会自动加载
### ❌ **容易导致遗忘的情况:**
1. **新开对话但不提供上下文**
   - 新对话时我只能看到摘要,不是完整历史
   - 解决:重新上传关键图片或说明"继续上次XX项目"
2. **只用口头描述之前的图**
   - "用之前那张蓝色的图" ← 我可能找不到
   - 解决:提供具体的图片URL
3. **跨度太长的复杂项目**
   - 单个对话太长(几百轮)可能影响检索效率
   - 解决:阶段性开新对话,并在开头总结前情
## 实战示例
**场景:品牌设计项目**
```
第1轮对话:
你:"帮我提取这个logo的配色规范 [上传图片]"
我:[分析] 主色#1E40AF,辅色#F59E0B...
第2轮对话(中间问了其他问题):
你:"回到刚才的logo,用那个蓝色做海报"
我:[自动找到之前的#1E40AF] 好的,使用主色#1E40AF...
第3轮对话(新会话):
你:"继续上次的品牌项目,logo是这个 [URL],主色#1E40AF"
我:收到!基于你的品牌规范继续设计...
```
## 技术提示
如果是**长期复杂项目**,最高效的方式是:
1. 第一次对话时让我提取所有规范
2. 我会生成详细的设计文档
3. 你保存这个文档
4. 后续新对话时,直接把文档内容+关键图片URL发给我
