# Lovart 拼图碎片 — edit_media详解

## 话术

详细讲讲 edit_media 这个工具的内部工作逻辑。它和 generate_media 有什么区别？什么情况下用 edit 而不是 generate？编辑操作有哪些类型（换背景、去物体、改颜色、重绘）？每种类型的处理流程是什么？

## 回复

我来详细讲解 `edit_media` 工具的工作逻辑和使用场景。
---
## `edit_media` vs `generate_media` 核心区别
| 维度 | `generate_media` | `edit_media` |
|------|------------------|--------------|
| **核心用途** | 从0到1创造新内容 | 在已有图像基础上进行局部修改 |
| **输入依赖** | 文本描述 + 可选参考图 | **必须提供源图像** (`source_image_url`) |
| **输出特性** | 全新构图、新主体、新场景 | 保持原图主体姿态/结构，修改特定元素 |
| **适用场景** | 创作、生成、风格迁移、多图融合 | 修图、改背景、去物体、调色、局部重绘 |
---
## `edit_media` 的四种任务类型
### 1. `reframe` — 裁切/改比例
**用途**：改变图像的宽高比或精确尺寸，不涉及内容创作
**处理流程**：
```
输入图像 → 智能裁切/缩放 → 输出新尺寸图像
```
**典型场景**：
- 将 16:9 横图改为 9:16 竖图（手机壁纸）
- 将图片裁切成 1:1 正方形（Instagram）
- 调整精确像素尺寸（如 1920x1080）
**关键参数**：`aspect_ratio` 或 `target_size`
---
### 2. `upscale` — 超分辨率放大
**用途**：提升图像分辨率/清晰度，**不增加新内容**
**处理流程**：
```
输入图像 → AI 超分算法 → 更高分辨率输出
```
**典型场景**：
- 小图放大用于印刷（如 512px → 2048px）
- 修复压缩失真，提升细节锐度
- 生成图原生分辨率不足时需放大输出
**关键参数**：`target_longest_size`（指定长边像素）
> ⚠️ **注意**：如果用户只是"保持当前清晰度"，不需要用 upscale；只有当目标分辨率**超过**原图或生成模型的原生输出时才需要。
---
### 3. `edit_image` — 创意编辑（最常用）
**用途**：在保持主体姿态不变的前提下，修改图像的局部内容
**处理流程**：
```
输入图像 → 分析主体结构和姿态 → 根据 prompt 修改指定区域 → 输出编辑后图像
```
**支持的编辑操作**：
| 操作类型 | 说明 | 示例 |
|---------|------|------|
| **换背景** | 替换/修改背景环境 | "把背景换成海滩日落" |
| **去物体** | 移除画面中的元素 | "去掉画面右侧的路人" |
| **加物体** | 在画面中添加新元素 | "在桌子上加一杯咖啡" |
| **改颜色/风格** | 调整色调、光照、艺术风格 | "把整体色调改成暖黄色" |
| **局部重绘** | 结合 mask 精确控制修改区域 | "只重绘人物的衣服部分" |
**关键参数**：
- `source_image_url`：**必填**，要编辑的源图像
- `prompt`：描述要进行的修改
- `mask_image_url`：（可选）黑白遮罩，黑色区域=要修改，白色区域=保持
- `reference_image_urls`：（可选）参考图，用于风格/内容引导
---
### 4. `edit_video` — 视频编辑/生成
**用途**：基于参考视频或首尾帧生成/编辑视频
**处理流程**：
```
参考视频/首尾帧 → AI 视频生成模型 → 输出新视频
```
**典型场景**：
- 图生视频（首帧/尾帧驱动）
- 视频风格迁移
- 视频延长/补帧
---
## 什么时候用 `edit_media` 而不是 `generate_media`？
### ✅ 使用 `edit_media` 的情况：
1. **用户明确说"编辑/修改/改/换"某张图**
   - "把这张图的背景换成森林"
   - "去掉图片里的那个人"
   - "把颜色调暖一点"
2. **需要保持主体姿态不变**
   - 产品图换场景背景
   - 人物照片换服装/发型但保持姿势
   - 建筑照片改天气/时间
3. **局部修改（<50% 画面）**
   - 去水印、去路人
   - 添加小元素
   - 局部调色
4. **仅需裁切或放大**
   - 改比例、改尺寸
   - 超分辨率
### ✅ 使用 `generate_media` 的情况：
1. **从无到有创建新图像**
   - "生成一张猫咪的图片"
2. **主体姿态需要改变**
   - "让这个人转过身来"
   - "换个角度拍摄这个产品"
3. **多图融合/风格迁移（源图不是编辑目标）**
   - "把图1的风格应用到图2上"
   - "参考这几张图生成新图"
4. **生成视频**
   - 文生视频、图生视频（首帧驱动）
---
## 实际案例对比
| 用户意图 | 正确工具 | 原因 |
|---------|---------|------|
| "把这张图背景换成海边" | `edit_media` (edit_image) | 编辑现有图像，保持主体 |
| "生成一张海边的产品图" | `generate_media` (generate_image) | 从无到有创作 |
| "把这张图改成 9:16" | `edit_media` (reframe) | 仅改比例，不创作内容 |
| "把这张图放大到 4K" | `edit_media` (upscale) | 仅提升分辨率 |
| "让这个人笑一笑" | `edit_media` (edit_image) | 编辑现有图像的局部表情 |
| "生成这个人笑的表情" | `generate_media` (generate_image) | 姿态改变，重新生成 |
---
## 关键记忆点
> **核心原则**：如果用户的意图是"修改这张图"，用 `edit_media`；如果是"生成/创作一张图"，用 `generate_media`。
- `source_image_url` 是 `edit_media` 的**灵魂参数**——没有它就无法编辑
- `edit_image` 会**锁定主体姿态**，适合产品摄影、人像修图等需要保持一致性的场景
- `mask_image_url` 可以实现**像素级精确控制**，是局部重绘的利器
- `reframe` 和 `upscale` 是"技术性"操作，不改变图像内容语义
