## Seedance 视频生成流程
### 整体架构
Seedance（当前核心版本为 Seedance 2.0）是字节跳动推出的**原生多模态音视频联合生成大模型**，支持文本、图像、视频、音频四种模态的混合输入，单次生成即可输出**音画同步**的成品视频。
### 标准工作流
```
┌─────────────────────────────────────────────────────────────┐
│  1. 素材准备阶段                                               │
│     - 上传参考图像（最多9张）→ 设定视觉风格/角色/场景           │
│     - 上传参考视频（最多3段）→ 锁定运镜/动作/物理动态           │
│     - 上传参考音频（最多3段）→ 驱动口型/节奏/氛围               │
├─────────────────────────────────────────────────────────────┤
│  2. 提示词构建                                                 │
│     - 场景描述 + 主体动作 + 镜头语言 + 氛围设定                 │
├─────────────────────────────────────────────────────────────┤
│  3. 参数配置                                                   │
│     - 画幅比例（16:9 / 9:16 / 1:1 等）                        │
│     - 时长（5-15秒）                                          │
│     - 分辨率（1080p / 2K）                                    │
│     - 风格选择                                                │
├─────────────────────────────────────────────────────────────┤
│  4. 生成与后期                                                 │
│     - 原生音视频同步输出                                       │
│     - 支持视频延长、局部重绘、角色替换等编辑操作               │
└─────────────────────────────────────────────────────────────┘
```

## 文生视频 vs 图生视频的处理逻辑差异
| 维度 | 文生视频 (Text-to-Video) | 图生视频 (Image-to-Video) |
|------|-------------------------|--------------------------|
| **核心逻辑** | 深度语义解析 + 高度指令遵循 | 视觉特征锁定 + 运动延伸 |
| **输入起点** | 纯文本描述从零构建场景 | 以参考图为首帧/视觉锚点 |
| **控制重点** | 通过自然语言构建多主体交互、环境氛围、运镜轨迹 | 确保人物面部、服装、色彩、艺术风格高度一致 |
| **生成策略** | 大语言模型理解语义后生成完整视频序列 | 以图像为约束条件，赋予符合物理规律的流畅运动 |
| **适用场景** | 创意发散、概念可视化、复杂叙事 | 角色动画、产品展示、风格延续 |

## `reference_video_urls` 的解析与应用
在 Seedance 中，`reference_video_urls` 的解析逻辑遵循以下原则：
### 1. 参考类型（`reference_video_refer_type`）
| 类型 | 作用 | 使用场景 |
|------|------|---------|
| **"feature"（默认）** | 提取参考视频的特征（动作、运镜、风格）用于生成新视频 | 大多数情况下使用，如"模仿这段舞蹈动作" |
| **"base"** | 将参考视频作为基础进行直接编辑/变换 | 需要对现有视频进行局部修改时 |
### 2. 参考视频的作用域
```
reference_video_urls 输入 → 系统解析以下维度：
├── 运动特征（Motion）：主体动作轨迹、物理动态
├── 镜头语言（Camera）：推拉摇移、景别变化、运镜节奏
├── 时间结构（Timing）：动作时长、关键帧分布
└── 风格特征（Style）：光影变化、色彩流动、氛围延续
```
### 3. 多视频参考的融合逻辑
- 最多支持 **3 段参考视频**
- 系统会自动识别各视频片段的互补性
- 冲突时以**最后一段视频**或**权重最高的特征**为准

## 视频稳定性、镜头运动、角色一致性的控制机制
### 1. 视频稳定性控制
| 控制层面 | 技术实现 |
|---------|---------|
| **时序一致性** | 基于扩散模型的时序注意力机制，确保帧间连贯 |
| **物理合理性** | 内置物理引擎约束，防止物体穿模、形变 |
| **运动平滑度** | 光流估计 + 运动插值算法优化 |
### 2. 镜头运动控制
Seedance 支持**导演级运镜控制**：
```
镜头语言解析：
├── 景别：特写/近景/中景/全景/远景
├── 运动：推(Push)/拉(Pull)/摇(Pan)/移(Truck)/跟(Follow)
├── 角度：平视/俯视/仰视/倾斜
└── 速度：匀速/加速/减速/急停
```
**实现方式**：
- 通过文本提示词描述运镜指令（如"缓慢推近人物面部"）
- 通过 `reference_video_urls` 锁定具体运镜方式
### 3. 角色一致性控制
```
角色档案系统（Character Profile）：
┌────────────────────────────────────┐
│  面部特征锁定（Facial Identity）    │
│  - 五官比例、肤色、发型、表情基准    │
├────────────────────────────────────┤
│  服装道具锁定（Outfit & Props）     │
│  - 服装款式、颜色、纹理、配饰        │
├────────────────────────────────────┤
│  体态特征锁定（Body Structure）     │
│  - 身高比例、体型、姿态习惯          │
├────────────────────────────────────┤
│  跨镜头一致性（Cross-shot）         │
│  - 多镜头叙事中保持角色可识别性      │
└────────────────────────────────────┘
```
**技术实现**：
- 使用参考图像建立角色特征向量
- 在生成过程中通过注意力机制约束特征偏离
- 支持角色替换编辑（保持动作，更换角色）
---
## 总结
Seedance 的核心优势在于其**原生多模态架构**——不是简单地将文生图模型扩展到视频，而是从底层设计了音视频联合生成机制。这使得它在以下方面表现突出：
1. **真正的音画同步**：口型、环境音、音乐与画面原生对齐
2. **导演级控制**：通过多模态参考实现精确的创意控制
3. **多镜头叙事**：支持复杂的场景切换和角色一致性维护
