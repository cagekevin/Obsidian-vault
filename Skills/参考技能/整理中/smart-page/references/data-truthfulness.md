# Ground Truth Gate · 信息补全与防捏造

**核心原则**：生成的是**用户真实场景的高质量网页**，不是"看起来像汇报的占位内容"。

---

## 必做三件事

### 1) 读上下文（workbuddy 会话 + 用户上传文件）

- 当前对话历史里用户提到的所有事实（数字、日期、团队名、产品名、链接）都要**原样保留**，不得改编
- 若用户上传了 PDF / PPT / 图片 / Excel / Word，**必须先读**（Read 工具），从中抽取结论、数据、结构
- 只有在上下文真正缺失时，才从模板 mock 里借用字段结构（但**不借具体数字**）

### 2) 信息不够按「面板优先」补全

统一遵循：**面板优先、缺事实不捏造、必要时暂停补充**。

| 字段类型    | 缺失表现                 | 处理方式                                                                 |
| ----------- | ------------------------ | ------------------------------------------------------------------------ |
| 团队/作者名 | 无具体署名               | 优先在右侧面板「更多补充」收集；仍缺失则用「—」占位，不编造姓名/团队     |
| 关键数字    | 只有定性无定量           | 不用 mock 数字替代真实数字；用「—」占位，并在最终回复列出待补指标        |
| 时间窗口    | 周报无日期、立项无时间线 | 可按用户上下文原样写入；缺失则用「待补时间窗口」或「—」                  |
| 对象/受众   | 不知道给谁看             | 优先使用面板选择；缺失时按场景默认语气处理，但不虚构具体组织/人名        |
| 核心结论    | 只有素材没有观点         | 可基于用户给出的事实归纳，不新增事实；事实不足以归纳时暂停并说明需补结论 |

仅当缺失信息会导致页面不可用或用户明确要求真实正式稿时，暂停生成并请用户补充；不要在启动右侧面板前用对话追问阻塞流程。

### 3) 严禁捏造

- 不许凭空生成用户没给过的数字、姓名、公司、项目代号
- 实在不知道 → 用"—"占位并在回复里明确告知用户哪些字段待补
- 不得编造"腾讯某某业务线在 xxx 项目中的数据"这类具体化的假内容

---

## 例外：用户主动授权"自由发挥"

用户明确说"你帮我捏一份 demo / mock / 示例"，agent **可以**使用模板 mock-data 作为基础生成占位内容，并在回复里标注"本页面为示例数据，正式汇报请替换真实内容"。
