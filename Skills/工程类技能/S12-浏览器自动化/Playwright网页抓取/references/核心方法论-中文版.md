# 浏览器自动化核心方法论

## 六大原则

1. **用面向用户的定位器**（getByRole, getByText），不要 CSS/XPath
2. **永远不加手动等待** — Playwright 的自动等待机制会处理
3. **每个任务完全隔离** — 用独立的 browser context
4. **截图和 Trace 是调试命脉** — 失败必截图
5. **CI 用无头模式**，调试用有头模式
6. **反检测是猫鼠游戏** — 不更新就会被封

---

## 选择器优先级（核心方法论）

```
优先级 1: getByRole       ← 最佳选择，匹配无障碍树
    例: page.getByRole('button', { name: 'Submit' })
        page.getByRole('textbox', { name: '搜索' })

优先级 2: getByText       ← 好，匹配可见文本
    例: page.getByText('欢迎回来')
        page.getByText(/订单 #\d+/)  ← 支持正则

优先级 3: getByLabel      ← 好，匹配表单 label
    例: page.getByLabel('邮箱地址')

优先级 4: getByTestId     ← 回退方案，显式测试契约
    例: page.getByTestId('submit-button')

优先级 5: CSS/XPath       ← 最后手段，最容易碎
```

### ❌ 错误写法（容易碎）
```javascript
// CSS class 一改就废
await page.locator('.btn-primary.submit-form').click();
// 位置选择器，DOM 顺序一变就废
await page.locator('#header > div > button:nth-child(2)').click();
// 自动生成的选择器
await page.locator('[data-v-12345]').click();
```

### ✅ 正确写法
```javascript
// 最好的方式
await page.getByRole('button', { name: '提交' }).click();
await page.getByRole('link', { name: '注册' }).click();
// 按文本
await page.getByText('欢迎回来').isVisible();
// 表单标签
await page.getByLabel('邮箱').fill('user@example.com');
```

### 组合过滤
```javascript
// 找到包含"产品A"的列表项，再点其中的"加入购物车"按钮
await page.getByRole('listitem')
  .filter({ hasText: '产品A' })
  .getByRole('button', { name: '加入购物车' })
  .click();

// 排除"已售罄"的项
await page.getByRole('listitem')
  .filter({ hasNotText: '已售罄' })
  .first()
  .click();
```

## 自动等待模式

**不需要手动加等待。** Playwright 自动等待：
- 元素挂载到 DOM
- 元素可见
- 元素稳定（不在动画中）
- 元素可接收事件
- 元素已启用

### ❌ 错误（新手最常犯）
```javascript
await page.goto('/dashboard');
await page.waitForTimeout(2000);     // 别！固定等待
await page.click('.submit-button');

await page.waitForSelector('.loading');
await page.waitForTimeout(500);      // "为了保险" — 别！
```

### ✅ 正确
```javascript
// 自动等按钮可点击
await page.getByRole('button', { name: '提交' }).click();
// 自动等文本出现
await expect(page.getByText('成功！')).toBeVisible();
// 自动等导航完成
await page.goto('/dashboard');
```

### 确实需要等的情况
```javascript
// 等特定网络请求完成
const responsePromise = page.waitForResponse(
  resp => resp.url().includes('/api/data')
);
await page.getByRole('button', { name: '加载' }).click();
const response = await responsePromise;

// 等 URL 跳转
await Promise.all([
  page.waitForURL('**/dashboard'),
  page.getByRole('button', { name: '登录' }).click(),
]);
```

## 隐身模式（反爬检测）

### 爬虫被检测的原因
- `navigator.webdriver` 被设为 true（最明显）
- 浏览器指纹不一致
- 操作行为过于完美（无鼠标移动、间隔精准）
- 无头模式特征

### Puppeteer 隐身方案（最佳）
```javascript
import puppeteer from 'puppeteer-extra';
import StealthPlugin from 'puppeteer-extra-plugin-stealth';
puppeteer.use(StealthPlugin());

const browser = await puppeteer.launch({
  headless: 'new',
  args: ['--disable-blink-features=AutomationControlled'],
});
```

### Playwright 隐身方案
```javascript
import { chromium } from 'playwright-extra';
import stealth from 'puppeteer-extra-plugin-stealth';
chromium.use(stealth());
```

### 手动隐藏（最简单）
```javascript
await page.evaluateOnNewDocument(() => {
  Object.defineProperty(navigator, 'webdriver', {
    get: () => undefined,
  });
});
```

### 模拟人类行为
```javascript
// 随机延迟
const randomDelay = (min, max) =>
  new Promise(r => setTimeout(r, Math.random() * (max - min) + min));

// 鼠标移动到按钮再点击
const button = await page.$('button.submit');
const box = await button.boundingBox();
await page.mouse.move(
  box.x + box.width / 2, box.y + box.height / 2,
  { steps: 10 }  // 分步移动，模拟真人
);
await randomDelay(100, 300);
await button.click();

// 自然滚动
await page.evaluate(() => {
  window.scrollBy({
    top: 300 + Math.random() * 200,
    behavior: 'smooth'
  });
});
```

## 错误恢复模式

```javascript
async function scrapeProduct(page, url) {
  try {
    await page.goto(url, { timeout: 30000 });
    const title = await page.getByRole('heading', { level: 1 }).textContent();
    const price = await page.getByTestId('price').textContent();
    return { title, price, success: true };
  } catch (error) {
    // 保存截图
    const screenshot = await page.screenshot({
      path: `errors/${Date.now()}-error.png`,
      fullPage: true
    });
    // 保存 HTML
    const html = await page.content();
    await fs.writeFile(`errors/${Date.now()}-page.html`, html);
    return { success: false, error: error.message };
  }
}
```

### 指数退避重试
```javascript
async function withRetry(fn, maxRetries = 3, baseDelay = 1000) {
  let lastError;
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;
      if (attempt < maxRetries - 1) {
        const delay = baseDelay * Math.pow(2, attempt);
        const jitter = delay * 0.1 * Math.random();
        await new Promise(r => setTimeout(r, delay + jitter));
      }
    }
  }
  throw lastError;
}
// 使用
const result = await withRetry(() => scrapeProduct(page, url), 3, 2000);
```

## 高频错误检查清单

| 问题 | 严重程度 | 解决方案 |
|------|---------|---------|
| 用了 `waitForTimeout` | ❌ 严重 | 删除，用自动等待替代 |
| 用了 CSS class 选择器 | ⚠️ 高 | 改用 getByRole/getByText |
| 用了 `nth-child` 位置选择器 | ⚠️ 高 | DOM 顺序一变就碎，改用文本匹配 |
| 不隐藏 `webdriver` | ⚠️ 高 | 加 stealth 插件或手动隐藏 |
| 爬取循环无 try/catch | ⚠️ 警告 | 一个页面失败会拖垮全部 |
| 用了自动生成的选择器 | ⚠️ 警告 | 每次构建都会变，用 data-testid |
| CI 失败无 Trace | ⚠️ 中 | 开启 retain-on-failure |
| 无头/有头行为不一致 | ⚠️ 中 | 固定 viewport 大小 |
| 并发爬取不限速 | ⚠️ 高 | 加 p-limit 控制并发数 |
| 不处理弹窗/新窗口 | ⚠️ 中 | 用 waitForEvent('popup') |
| iframe 内容选不到 | ⚠️ 中 | 用 frameLocator 进入 iframe |

## 一句话总结

```
选元素: getByRole > getByText > getByLabel > getByTestId > CSS
等元素: 永远不手动等，Playwright 自动等
防检测: 隐藏 webdriver + 模拟人类行为
容错:   try/catch + 截图 + 指数退避重试
隔离:   每个任务独立 browser context
限速:   并发控制 + 随机延迟
```

## 测试隔离模式

每个测试/任务运行在完全隔离的环境中。

### 基本原则
```
每个任务拥有独立的：
- Browser context（Cookie、Storage 完全隔离）
- 新 Page
- 干净状态
```

### Playwright Test 示例（自动隔离）
```javascript
import { test, expect } from '@playwright/test';

// 每个测试自动获得独立的浏览器上下文
test('用户添加商品到购物车', async ({ page }) => {
  await page.goto('/products');
  await page.getByRole('button', { name: '加入购物车' }).click();
  await expect(page.getByTestId('cart-count')).toHaveText('1');
});

test('用户从购物车移除商品', async ({ page }) => {
  // 完全隔离——购物车是空的
  await page.goto('/cart');
  await expect(page.getByText('购物车是空的')).toBeVisible();
});
```

### 共享登录态的正确做法
```javascript
// 1. 先一次性登录，保存状态到文件
// setup.ts
import { test as setup } from '@playwright/test';

setup('登录', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel('邮箱').fill('user@example.com');
  await page.getByLabel('密码').fill('password');
  await page.getByRole('button', { name: '登录' }).click();
  await page.waitForURL('/dashboard');

  // 保存登录状态
  await page.context().storageState({ path: './auth.json' });
});

// 2. 其他测试复用登录态
const context = await browser.newContext({
  storageState: './auth.json'  // 加载已保存的 Cookie
});
```

### 手动隔离（不用 Playwright Test 时）
```javascript
const context = await browser.newContext();  // 全新上下文
const page = await context.newPage();
// ... 执行任务 ...
await context.close();  // 用完关闭，不污染其他任务
```

## 网络拦截模式

### 拦截无用资源（加速爬取）
```javascript
await page.route('**/*', (route) => {
  const url = route.request().url();
  const resourceType = route.request().resourceType();

  // 拦截图片、字体、媒体——只爬文本，不下载图片
  if (['image', 'font', 'media'].includes(resourceType)) {
    return route.abort();
  }

  // 拦截统计脚本
  if (url.includes('google-analytics') || url.includes('facebook.com/tr')) {
    return route.abort();
  }

  return route.continue();
});
```

### 模拟 API 响应（测试用）
```javascript
await page.route('**/api/products', async (route) => {
  await route.fulfill({
    status: 200,
    contentType: 'application/json',
    body: JSON.stringify([
      { id: 1, name: 'Mock 商品', price: 99.99 },
    ]),
  });
});
```

### 捕获真实 API 响应
```javascript
const apiResponses = [];

page.on('response', async (response) => {
  if (response.url().includes('/api/')) {
    const data = await response.json().catch(() => null);
    apiResponses.push({
      url: response.url(),
      status: response.status(),
      data,
    });
  }
});

await page.goto('/dashboard');
// apiResponses 现在包含所有 API 调用
```

## 10 大常见坑详解

### 坑 1：用 waitForTimeout 代替自动等待

**严重程度：** ❌ 严重

**场景：** 等待元素或页面状态

**症状：** 本地跑过了，CI 挂了。跑 9 次过，第 10 次挂。
"找不到元素"的报错时有时无。测试本该 3 秒跑完却要 30+ 秒。

**原因：** `waitForTimeout` 是固定延迟。页面 500ms 加载完，你也要等 2000ms；CI 慢了需要 2100ms，你又等不够。没有"正确"的值——总是要么太长要么太短。

**解决：**
```javascript
// ❌ 错误
await page.waitForTimeout(2000);
await page.click('.submit');

// ✅ 正确 - 让 Playwright 自动等
await page.getByRole('button', { name: '提交' }).click();
```

### 坑 2：CSS class 选择器

**严重程度：** ⚠️ 高

**场景：** 用 CSS class 定位元素

**症状：** 前端改了 CSS 类名，测试全挂。

**原因：** CSS class 是实现细节，不是语义。设计师把 `.btn-primary` 改成 `.button--primary`，行为没变但测试全断。

**解决：**
```javascript
// ❌ 错误 - 绑在 CSS 上
await page.locator('.btn-primary.submit-form').click();

// ✅ 正确 - 面向用户
await page.getByRole('button', { name: '提交' }).click();
```

### 坑 3：navigator.webdriver 暴露自动化

**严重程度：** ⚠️ 高

**场景：** 爬有反爬的网站

**症状：** 直接 403。出验证码。页面空白。"拒绝访问"消息。第一次成功第二次被封。

**原因：** 无头浏览器默认 `navigator.webdriver = true`，反爬检测第一眼就看到了。

**解决：**
```javascript
// 最简单 - 手动隐藏
await page.evaluateOnNewDocument(() => {
  Object.defineProperty(navigator, 'webdriver', { get: () => undefined });
});

// 最佳 - 用 stealth 插件
import puppeteer from 'puppeteer-extra';
import StealthPlugin from 'puppeteer-extra-plugin-stealth';
puppeteer.use(StealthPlugin());
```

### 坑 4：测试之间共享状态

**严重程度：** ⚠️ 高

**场景：** 多个测试依次执行

**症状：** 单独跑都过，一起跑就挂。顺序敏感——B 在 A 前面跑就过，后面跑就挂。

**原因：** 共享 browser context 意味着共享 Cookie、Storage。A 登录了，B 预期是未登录状态。

**解决：** 每个任务独立的 browser context，用完就关。

### 坑 5：CI 失败没留 Trace

**严重程度：** ⚠️ 中

**场景：** 调试 CI 上的失败

**症状：** "CI 挂了"但没有任何有用信息。本地复现不了。

**解决：**
```javascript
// playwright.config.ts
export default defineConfig({
  use: {
    trace: 'retain-on-failure',    // 保留失败 Trace
    screenshot: 'only-on-failure', // 失败自动截图
    video: 'retain-on-failure',    // 失败保留视频
  },
  retries: 2,  // 自动重试 2 次
});

// 查看 Trace
npx playwright show-trace test-results/trace.zip
```

### 坑 6：有头模式过，无头模式挂

**严重程度：** ⚠️ 中

**场景：** CI（无头）运行

**症状：** 看着浏览器跑一切正常，CI 上神秘失败。

**原因：** 无头模式没有显示器，影响 CSS 可见性计算、字体渲染、动画行为。

**解决：**
```javascript
// 固定 viewport 大小
const context = await browser.newContext({
  viewport: { width: 1280, height: 720 },
});
```

### 坑 7：爬太快被限流

**严重程度：** ⚠️ 高

**场景：** 批量爬取

**症状：** 前 50 页正常，然后全是 429。IP 被封。

**解决：**
```javascript
// 随机延迟 1-3 秒
const randomDelay = () =>
  new Promise(r => setTimeout(r, 1000 + Math.random() * 2000));

// 限制并发
import pLimit from 'p-limit';
const limit = pLimit(3);
```

### 坑 8：不处理弹窗和新窗口

**严重程度：** ⚠️ 中

**场景：** 点击 `target="_blank"` 的链接

**症状：** 点了按钮，没反应。测试卡住。操作成功但断言失败——因为你还在原页面。

**解决：** 先注册窗口事件监听，再触发点击（顺序不能反），见上方"弹窗与新窗口处理"章节。

### 坑 9：iframe 里的元素选不到

**严重程度：** ⚠️ 中

**场景：** 页面嵌入了 iframe

**症状：** 元素明明在页面上但就是"找不到"。

**原因：** iframe 是独立的文档，`page.locator` 只在主文档里搜索。

**解决：** 用 `page.frameLocator('iframe#xxx')` 进入 iframe 后再定位。

### 坑 10：爬取循环没有错误处理

**严重程度：** ⚠️ 警告

**场景：** 批量爬取

**症状：** 一个页面报错，整个爬取全崩。

**解决：** 每条数据单独 try/catch，失败记录错误继续处理下一条。
