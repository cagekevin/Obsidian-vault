# S0-开发工作流路由

<what-to-do>

## 工作流总览

一个典型的开发项目按以下阶段推进，每个阶段对应一个 S 技能：

```
[想法]
   │
   ▼
 S1 ── 需求对齐与文档 ── 新需求/新功能的第一步
   │
   ▼
 S2 ── 项目初始化与分流 ── 搭基建、配追踪器、建看板
   │
   ▼
 S3 ── 快速原型构建 ── 不确定方案时先搭原型验证
   │
   ▼
 S4 ── 闭环开发与调试 ── 写功能（TDD）或修 Bug
   │
   ├─── S5 ── 测试策略 ── 决定测什么、怎么测
   ├─── S6 ── 智能审计 ── 写完代码后审计质量
   │
   ▼
 S7 ── 代码架构优化 ── 架构有坏味道时重构
   │
   ▼
S10 ── 全局视角分析 ── 面对不熟悉的代码时鸟瞰全景
   │
   ─── S8 ── Git 版本控制 ── 贯穿所有环节的版本管理
   ─── S9 ── 部署与 CI/CD ── 构建、测试流水线、部署上线
   ─── S11 ── 基础网页设计 ── 需要设计网页/UI 时
   ─── S12 ── 浏览器自动化 ── 需要自动操作浏览器/爬数据时
```

## 路由表

根据你的需求，在这里找对应的技能入口：

| 你当前要做什么 | 路由到 | 工作流阶段 |
|---|---|---|
| "有个新想法/新功能要做"、"帮我分析这个需求" | **S1-需求对齐与文档** | 起点 |
| "初始化一个新项目"、"搭一下基建" | **S2-项目初始化与分流** | 基建 |
| "不确定方案行不行，先搭个原型试试" | **S3-快速原型构建** | 验证 |
| "写一个新功能"、"实现这个 feature" | **S4-闭环开发与调试** | 开发 |
| "有个 Bug，帮我找找原因" | **S4-闭环开发与调试** | 调试 |
| "帮我设计一下测试"、"测什么" | **S5-测试策略** | 测试 |
| "帮我审一下这段代码"、"审计这个技能" | **S6-智能审计** | 质量 |
| "代码架构很乱，帮我优化一下" | **S7-代码架构优化** | 重构 |
| "Git 冲突了"、"帮我提交代码" | **S8-Git版本控制** | 版本管理 |
| "配一下 CI"、"部署到服务器" | **S9-部署与CI-CD** | 部署 |
| "这片代码我不熟，帮我看看全局" | **S10-全局视角分析** | 分析 |
| "设计一个网页 / UI" | **S11-基础网页设计** | 前端 |
| "帮我去这个网站抓数据"、"自动填表单" | **S12-浏览器自动化** | 自动化 |

## 使用方式

1. 告诉我你**当前要做什么**（说人话就行，不需要记技能编号）
2. 我从路由表找到对应技能并自动加载执行
3. 完成后如需进入下一个阶段，我会自动告知下一步该调哪个技能
