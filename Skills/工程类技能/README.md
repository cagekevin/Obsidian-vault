# Engineering

Skills I use daily for code work.

- **[S0-开发工作流路由](./S0-开发工作流路由/SKILL.md)** — 工程类技能统一入口。根据开发任务自动分流到对应的 S 技能。不确定从哪开始时，先走 S0。
- **[S1-需求对齐与文档](./S1-需求对齐与文档/SKILL.md)** — End-to-end requirement pipeline: domain alignment + PRD generation + tracer-bullet issue breakdown.
- **[S2-项目初始化与分流](./S2-项目初始化与分流/SKILL.md)** — Project scaffold setup (issue tracker, triage labels, domain docs) and issue/kanban triage automation.
- **[S3-快速原型构建](./S3-快速原型构建/SKILL.md)** — Build a throwaway prototype to flesh out a design — either a runnable terminal app for state/business-logic questions, or several radically different UI variations toggleable from one route.
- **[S4-闭环开发与调试](./S4-闭环开发与调试/SKILL.md)** — Write features (TDD) or fix bugs (debug loop) through deterministic feedback loops + vertical tracer bullets.
- **[S5-测试策略](./S5-测试策略/SKILL.md)** — 测试金字塔分层指南、Mock 边界规范、覆盖率目标设定。
- **[S6-智能审计](./S6-智能审计/SKILL.md)** — Professional audit hub for code, prompts, and agent skills. Use when user wants to review code, audit prompts, or check skill compliance.
- **[S7-代码架构优化](./S7-代码架构优化/SKILL.md)** — Find deepening opportunities in a codebase, informed by the domain language in `CONTEXT.md` and the decisions in `docs/adr/`.
- **[S8-Git版本控制](./S8-Git版本控制/SKILL.md)** — Git 工作流标准化：Conventional Commits、分支策略、冲突解决。
- **[S9-部署与CI-CD](./S9-部署与CI-CD/SKILL.md)** — CI 设计原则、构建脚本规范、部署策略速查。
- **[S10-全局视角分析](./S10-全局视角分析/SKILL.md)** — Tell the agent to zoom out and give broader context or a higher-level perspective on an unfamiliar section of code.
- **[S12-浏览器自动化](./S12-浏览器自动化/SKILL.md)** — Browser automation complete methodology — selector priority, auto-wait, stealth, error recovery, concurrency limiting.
