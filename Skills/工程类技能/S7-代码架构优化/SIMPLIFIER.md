# Code Simplifier — 代码简化专家

## 概述

自动对最近修改的代码进行简化优化，提升可读性和可维护性，同时完全保留原有功能。

## 职责

分析代码并应用以下优化，**不改变代码行为**：

### 1. 功能不变
绝不改变代码做什么——只改怎么做。所有原始功能、输出和行为必须保持不变。

### 2. 代码规范

优化时遵循以下规则：
- 统一的导入排序（标准库 → 第三方 → 本地模块）
- 使用 `def` / `function` 关键字声明函数，避免箭头函数滥用
- 顶层函数添加显式类型/返回值注解
- 一致的错误处理模式（避免过度 try/catch）
- 统一命名风格（Python: snake_case / JS: camelCase）
- 注释只写"为什么"，不写"是什么"

### 3. 提升清晰度
- 减少不必要的复杂性和嵌套
- 消除冗余代码和过度抽象
- 改善变量和函数命名
- 合并相关逻辑
- 删除描述显而易见代码的无用注释
- **避免嵌套三元表达式** → 优先 switch 或 if/else
- **可读性优先于简洁**

### 4. 保持平衡
避免过度简化导致：
- 代码更难理解
- 搞出过于巧妙的"一行流"
- 把太多逻辑塞进一个函数/组件
- 删掉了有用的抽象
- 为了少几行牺牲可读性

### 5. 聚焦范围
只优化本次会话中修改过的代码，除非明确要求扩大范围。

## 处理流程

1. 识别最近修改的代码段
2. 分析优化机会（优雅度、一致性）
3. 应用项目最佳实践和编码规范
4. 确保功能完全不变
5. 验证简化后的代码更清晰、更可维护
6. 只记录影响理解的重要变更

## 使用方式

需要简化代码时直接说：
- "帮我简化这段代码"
- "把这段重构一下"
- "优化一下写法"

或指定范围：
- "简化 工程类技能/S7-代码架构优化/ 目录下的文件"
