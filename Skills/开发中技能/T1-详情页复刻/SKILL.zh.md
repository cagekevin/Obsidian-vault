# 电商详情页风格迁移

<what-to-do>

**目的**：通过分析参考页面并将其视觉语言迁移到新产品，快速生成风格相似的电商详情页草稿。这是一个**快速原型工具**，不是像素级精确复制系统。

**实际成功率**：风格相似度 70-80% | 布局近似度 60-70% | 文字渲染准确度 50-60% | 整体可用草稿率 65-75%

**本技能能做到**：
- 捕捉整体视觉风格（配色、氛围、构图模式）
- 生成相似的布局结构
- 迁移设计语言（现代/奢华/极简/活力）
- 创建快速可迭代的草稿

**本技能做不到**：
- 像素级精确复刻布局
- 精确颜色匹配（预期 ±10-20% 偏差）
- 特定字体的精确文字渲染
- 保证元素定位
- 保持所有部分的精确比例

---

## 工作流程

### 阶段 1：参考分析 & 风格提取

**目标**：理解参考页面的视觉DNA，而非精确测量。

#### 步骤 1.1：初始图片分析

**操作**：对参考详情页调用 `analyse_image`。

**分析重点**（定性，非定量）：
```
1. 整体视觉印象
   - 氛围/基调（如：干净现代、温暖奢华、活力年轻）
   - 主色（用文字描述，如"柔粉配金色点缀"，不用#HEX）
   - 视觉层级（第一眼注意到什么、第二、第三）

2. 布局模式识别
   - 板块顺序（主图产品 → 功能网格 → 生活场景 → 规格表）
   - 构图风格（居中/不对称/杂志风格/极简）
   - 间距感（紧凑 vs 通透）

3. 设计元素
   - 字体风格（粗标题、优雅衬线、俏皮无衬线）
   - 图形处理（圆角、阴影、边框、徽章）
   - 图片风格（棚拍、生活化、插画、3D渲染）
   - 装饰元素（纹理、分隔线、图标）

4. 内容结构
   - 核心信息主题
   - 信息密度（偏文字 vs 偏视觉）
   - 行动召唤模式（按钮/链接布局）
```

**向用户输出**：用自然语言呈现定性风格总结。
```
"该参考页呈现出[形容词]的美学风格，配色为[颜色描述]。
布局采用[模式描述]的结构，间距[间距描述]。
关键设计特征包括[视觉元素]。
整体氛围为[氛围描述]。"
```

**不要**输出：十六进制色值、像素尺寸、精确角度、Delta E 值、数值化的间距比例。

#### 步骤 1.2：用户确认 & 调整

**操作**：询问用户优先关注哪些方面。
```
"我已分析完参考页面。在生成之前，请确认：
1. 风格优先级：我应该更侧重[配色氛围]还是[布局结构]？
2. 灵活度：你接受[某元素]有一定变化，还是必须尽量接近？
3. 内容：你有具体的产品图片/文字吗，还是我用占位符？
你也可以直接说：'按整体风格直接生成'来跳过确认。"
```

**为什么这一步很重要**：AI 无法完美复刻一切。用户输入有助于把有限的准确度预算分配到最重要的地方。

---

### 阶段 2：内容准备

**目标**：搜集新商品页所需的素材。

#### 步骤 2.1：素材收集

**需要用户提供**：
- 新产品图片（至少 1 张主图）
- 产品名称
- 核心卖点（3-5 条要点）

**用户可选提供**：
- 品牌 Logo
- 具体文字内容
- 更多产品角度/生活场景图
- 技术规格

**如果用户提供的信息很少**：使用占位符继续，并告知用户：
```
"我会用占位文生成草稿，缺少的元素为[缺少项]。
你可以稍后替换，或现在就提供内容以获得更完整的草稿。"
```

#### 步骤 2.2：搜索参考图片（如需）

**何时需要搜索**：
- 用户没有生活化/场景图，但参考页用了
- 需要辅助视觉元素（纹理、背景、图标）

**操作**：用简单明确的查询词调用 `search_image`。
- 好的："极简产品摄影"、"大理石纹理"、"厨房生活场景"
- 不好的："高端奢华现代极简产品拍摄带柔和光线"

**搜索策略**：最多 1-2 次搜索。专注于填补关键缺口。优先使用用户提供的图片。

---

### 阶段 3：草稿生成

**目标**：一次性生成风格相似的详情页草稿。

#### 步骤 3.1：构建提示词

**提示词结构**（用于 `generate_image_nano_banana_2`）：
```
请按照参考图片的风格，为[产品名称]生成一张电商详情页。

需匹配的视觉风格：
- 整体氛围：[分析中的氛围]
- 配色方案：[分析中的颜色描述]
- 设计语言：[分析中的风格关键词]

布局结构：
[按板块用视觉语言描述，例如：]
- 顶部区域：大尺寸主图置于[左/右/中]，背景[背景描述]
- 第二区域：[3个功能点网格 / 并排对比等]
- 第三区域：[产品使用的生活场景]
- 底部区域：[规格表 / 信任徽章等]

产品内容：
- 产品：[产品名称和简要描述]
- 核心功能：[要点列表]
- [需包含的特定文字]

设计细节：
- 字体：标题[粗体/优雅/俏皮]，正文[干净/衬线/圆润]
- 间距：[紧凑/适中/通透]
- 图形元素：[圆角/阴影/边框/徽章，参考页中的样式]
保持与参考页一致的视觉语言，同时适配新产品。
```

**提示词关键规则**：
1. **用视觉描述，不用数字**：
   - 好的："产品从略微抬高的正面角度展示"
   - 不好的："产品以15度角展示"
2. **定性描述颜色**：
   - 好的："柔和的桃粉色背景搭配温暖的金色点缀"
   - 不好的："背景 #FFE5D9，点缀 #D4AF37"
3. **清晰引用风格图片**：始终在 image_url_list 中包含参考图片 URL。使用"按参考图片的风格"等措辞。
4. **布局要具体，但执行可灵活**：描述有哪些板块及其用途。不要求像素级精确定位。

#### 步骤 3.2：工具调用

**工具**：`generate_image_nano_banana_2`

**参数**：
```javascript
{
  "task_type": "REFERENCE_TO_IMAGE",
  "image_url_list": [
    "[参考详情页 URL]",
    "[产品主图 URL]",
    "[可选：生活场景/辅助图片]"
  ],
  "aspect_ratio": "9:16",   // 标准详情页
  "resolution": "2K"        // 详情页用较高分辨率
}
```

**宽高比选择**：
- 标准详情页：9:16（竖版）
- 短版页面：3:4
- 超长页面：分块生成（见阶段 5）

#### 步骤 3.3：输出交付

**向用户展示**：
```
"这是根据参考风格生成的详情页草稿：

风格匹配评估：
✓ 成功捕捉到：[列出 2-3 个强匹配项]
⚠ 近似区域：[列出 1-2 个有偏差的区域]
✗ 需手动调整：[列出明显失败的元素]

下一步：
- 如果整体方向可行，我可以生成变体或补充板块
- 如需调整特定元素，请描述修改内容，我会迭代
- 最终出图可能需要人工精修[特定元素]"
```

**诚实呈现**：指出哪些效果好、哪些不好。不要过度吹嘘结果。清晰指导用户哪些需要人工处理。

---

### 阶段 4：迭代 & 优化

**目标**：根据用户反馈改进草稿。

#### 步骤 4.1：收集反馈
```
"你想调整什么？常见的优化方向：
1. 更改特定板块的布局
2. 调整配色氛围（更暖/更冷/更鲜艳）
3. 修改文字内容
4. 用不同产品角度重新生成
5. 为特定板块创建变体"
```

#### 步骤 4.2：针对性重新生成

针对板块级别的修改：

**选项 A：重新生成整页**（更简单，控制力较弱）
- 修改提示词，突出需要更改的板块
- 重新生成整页

**选项 B：单独生成板块**（控制力更强，需后期拼接）
- 将该板块作为独立图片生成
- 用户在设计工具中手动拼合

**建议**：默认走选项 A（1-2 次迭代），如需精细控制再建议选项 B。

#### 步骤 4.3：生成变体

**常见变体需求**：
1. **配色变体**：
   "生成 2 个不同配色的版本：版本 A：冷色调（蓝紫色系），版本 B：暖色调（橙红色系）"

2. **布局变体**：
   "创建一个替代布局：主图区域用生活场景替代纯产品图，功能网格改为并排对比"

3. **密度变体**：
   "生成一个更极简的版本：更多留白、更少文字块、突出大图视觉"

**批量生成**：如果用户想要多个变体，在单次调用中把变体列表全部放入项目上下文，由图片子代理统一生成。

---

### 阶段 5：多板块页面（进阶）

**何时使用**：参考页极长（相当于 3 个以上屏幕高度）。

**挑战**：单次图片生成在长度和细节密度上有实际限制。

**解决方案**：逐板块生成，保持风格一致性。

#### 步骤 5.1：板块规划

**操作**：将页面划分为逻辑板块。

**示例拆分**：
```
板块 1：主图区（产品 + 标题）
板块 2：核心功能（三列网格）
板块 3：生活场景（产品使用中）
板块 4：技术规格（表格/信息图）
板块 5：信任信号（徽章、用户评价）
```

**向用户展示**：
```
"此参考页较长，我建议拆分为[X]个板块生成。我会保持风格一致，你需要用设计工具拼合。
或者，我也可以生成一个浓缩的单页版本。你倾向哪种？"
```

#### 步骤 5.2：顺序生成

**流程**：
1. 生成板块 1，附带完整风格参考
2. 生成板块 2，同时参考：原始参考页 + 已生成的板块 1（保持风格一致）
3. 重复以上步骤完成剩余板块

**板块 2+ 的提示词模式**：
```
"请生成电商详情图的第[N]部分，保持与前一部分的视觉一致性。
风格一致性：
- 匹配第一张图的配色、字体和设计语言
- 使用相似的间距和图形处理
板块内容：
[该板块的具体内容]
此部分应感觉像是页面的自然延续。"
```

**板块 2+ 的工具调用**：
```javascript
{
  "image_url_list": [
    "[原始参考页]",
    "[上一张已生成的板块图]",
    "[该板块需要的产品/内容图片]"
  ],
  "aspect_ratio": "16:9",   // 单个板块用较短比例
}
```

#### 步骤 5.3：拼合指导
```
"所有板块已完成！拼合方法如下：
手动拼合（推荐）：
1. 在 Figma/Photoshop/Canva 中打开
2. 将各板块垂直堆叠
3. 调整板块间的重叠/间距
4. 确保颜色一致性（可能需要微调）
自动拼合（实验性）：我可以尝试生成合并版本，但质量可能会下降。"
```

**如实说明**：自动拼合通常不如手动组合效果好，要诚实告知。

## 场景化指导

### 场景 A：同类目产品，换品
**示例**：参考页是某面霜详情页，用户想为自己的精华液创建详情页。
**方法**：期望高风格迁移保真度。重点在于替换产品图片、保留布局、调整文字内容。
**成功率**：75-85%（最高成功场景）
**提示词重点**：
```
"请为[新产品]创建详情页，使用与参考页[原产品]完全相同的布局结构和视觉风格。
保持板块顺序、配色方案和设计处理。只需替换产品图片并更新文字内容。"
```

### 场景 B：不同产品，相似风格
**示例**：参考页是科技产品详情页，用户想要时尚品类页面。
**方法**：迁移视觉氛围和设计语言。根据不同产品类型调整布局。
**成功率**：60-70%（中等成功率）
**提示词重点**：
```
"请为[时尚产品]创建详情页，以参考科技产品页的视觉风格为灵感。
需迁移的元素：整体配色氛围、字体风格、图形处理、间距感。
需调整的元素：适合时尚产品的布局结构、内容板块、图片风格。"
```
**产品展示视觉描述**：
- "连衣裙用模特展示，全身照，居中"
- "手提包用正面视角，略微抬高展示"
- "鞋履用动态侧角度展示轮廓"
- 不要："产品呈45度角，30度抬高"

### 场景 C：竞品分析 / 灵感迁移
**示例**：用户喜欢竞品页的风格，希望为自己品牌做相似的。
**方法**：提取风格DNA但不抄袭。强调差异化。注重"受其启发"而非"复制"。
**成功率**：65-75%
**提示词重点**：
```
"请为[品牌][产品]创建详情页，以参考设计为灵感，但保留鲜明的品牌特色。
借鉴的元素：[特定风格方面]、[效果好的布局模式]
差异化：使用[品牌]色系，语调调整为更[形容词]，加入独特的品牌元素。"
```
**道德提示**：提醒用户这是用于灵感参考，而非抄袭。建议加入独特品牌元素。

## 局限性 & 人工介入节点

### AI 无法可靠完成的工作

#### 1. 精确文字渲染
- **问题**：AI 常生成乱码文字、错误字体或文字块错位
- **成功率**：可读文字 50-60%
- **解决方案**：用占位文字区域生成页面，用户在设计工具中添加真实文字；或在提示词中提供精确文字后重新生成（可能需要 2-3 次）

#### 2. 精确颜色匹配
- **问题**：AI 定性而非定量地理解颜色
- **预期偏差**：色调/饱和度 +/- 10-20%
- **解决方案**：草稿阶段接受近似即可，最终出图时手动校色；或在提示词中用品牌色描述（如"用深海军蓝，不要亮蓝色"）

#### 3. 复杂数据可视化
- **问题**：图表、详细表格常生成错误
- **成功率**：复杂信息图 30-40%
- **解决方案**：用占位图表区域生成页面，在专业工具（Excel、Figma）中制作实际图表，手动拼合

#### 4. 像素级对齐
- **问题**：AI 不理解网格和精确间距
- **实际情况**：元素大致定位
- **解决方案**：把 AI 草稿当作参考指导，用设计工具重建用于出图；或用于快速活动页，接受近似效果

#### 5. 品牌 Logo 集成
- **问题**：Logo 常被扭曲或风格化处理不当
- **成功率**：清晰 Logo 渲染 40-50%
- **解决方案**：生成页面时不带 Logo，后期制作中加上真实 Logo 文件；或将 Logo 作为参考图并提供"保持 Logo 不变"的强调

### 何时停止与 AI 迭代

**收益递减点**：3-4 次重新生成后，大概率是在与 AI 的局限性对抗，而非在推进进展。

**应转为手动编辑的信号**：
- 文字在 3 次尝试后仍生成错误
- 特定元素定位始终不理想
- 用户需要像素级精度
- 颜色匹配需要精确十六进制色值

**给用户的建议**：
```
"我们已经到了靠手动精修比更多 AI 迭代更快的节点。
我建议：
1. 把这张草稿作为基础框架
2. 导入 [Figma/Photoshop/Canva]
3. 手动调整[特定元素]
4. 添加最终文字和品牌素材
这种混合方式兼顾了 AI 草稿的速度和手动控制的精度。"
```

## 质量预期 & 成功指标

### 实际输出质量

**草稿质量（首次生成）**：
- 风格相似度：70-80%
- 布局结构近似度：60-70%
- 文字准确度：50-60%
- 颜色精度：60-70%
- 元素定位：60-70%

**2-3 次迭代后**：
- 风格相似度：80-85%
- 布局结构近似度：75-80%
- 文字准确度：65-70%
- 颜色精度：70-75%
- 元素定位：70-75%

**可出图（AI + 人工）**：经过人工后期处理后可达 90%+ 质量。

### 各使用场景的成功标准

**快速活动稿**（速度优先）：仅 AI 即可，接受 70-80% 保真度。
**客户演示**（平衡）：AI 草稿 + 轻度手动润色，80-85% 保真度，1-2 小时人工精修。
**最终出图**（质量优先）：AI 作为基础 + 全手动制作，95%+ 保真度。将 AI 输出视为线框图/样机。

## 工作流决策树

```
开始：用户提供参考详情页
  ↓
[用 analyse_image 分析参考页]
  ↓
用户是否有所有产品素材？
  ├─ 有 → 进入生成
  └─ 无 → 请求素材或使用占位符
       ↓
参考页是否极长？
  ├─ 是 → 提供逐板块生成方案
  └─ 否 → 单页生成
       ↓
[用 generate_image_nano_banana_2 生成草稿]
  ↓
展示草稿并诚实评估
  ↓
用户对方向满意？
  ├─ 是 → 提供变体/精修
```

## 常见问题

**问：AI 能完美复刻详情页吗？** 答：不能。AI 能创建风格相似的草稿，保真度 70-85%。要出图质量，需要手动精修。

**问：应该尝试多少次迭代？** 答：2-3 次迭代通常效果最好，之后收益递减。转为手动编辑。

**问：为什么文字是乱的？** 答：AI 文字渲染不可靠（成功率 50-60%）。用占位文字生成，再手动添加真实文字；或重新生成 2-3 次。

**问：颜色不完全匹配，能修吗？** 答：AI 定性而非定量理解颜色。预期 +/- 10-20% 偏差。如需精确品牌色，后期制作中手动校色。

**问：能直接用这个出图吗？** 答：将 AI 输出作为基础框架/线框图。计划进行手动精修以达到出图质量（90%+ 保真度）。

**问：如果布局完全不对怎么办？** 答：用更具体的布局描述重新生成。如果 2-3 次仍不行，可能布局对 AI 来说太复杂了。考虑逐板块生成或手动设计。

**问：这个过程需要多久？** 答：快速草稿 5-10 分钟。精修草稿（2-3 次迭代）20-30 分钟。可出图（AI + 人工）2-4 小时。

## 工具选择逻辑

### 主要工具：generate_image_nano_banana_2
**优势**：风格迁移好、支持多张参考图、文字渲染尚可、速度快。
**适用于**：所有标准详情页生成、单页输出、逐板块生成、迭代和变体。
**参数**：task_type 始终为 REFERENCE_TO_IMAGE，aspect_ratio 为 9:16 或板块用 16:9，resolution 为 2K。

### 何时使用图片子代理
**场景**：用户在一条消息中请求多个变体（如"生成 3 个不同配色的版本"）。
**操作**：在单次 project_context 中列出所有变体及其完整提示词，调用子代理。所有生成统一使用 generate_image_nano_banana_2。

### 何时不该使用此技能
- 用户需要像素级复刻：设定预期，AI 做不到
- 参考页是复杂的交互式页面：AI 只能生成静态图
- 用户需要精确品牌合规：AI 无法保证遵循品牌规范
- 参考页含大量数据可视化：图表很可能生成失败
