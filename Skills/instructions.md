# Skills/instructions.md

## 这是我的工作方法论

这里存放的是我多年积累的自动化流程和最佳实践。每个技能都是我对某一类问题的解法——AI 学会了这些，就知道我怎么做事、我喜欢什么流程、我踩过什么坑。

**Skills 是这套系统的灵魂，把整个系统串起来。**

## 文件结构

技能按大类分，每个大类下是具体技能子文件夹：
- 工作效率类/（W0-W9）
- 工程类技能/（S0-S12）
- 个人专用类/
- baoyu/
- 杂项工具类/
- 编码准则类/
- 整理中/

## 命名规则

- **大类文件夹**：中文名，如 `工作效率类/`、`工程类技能/`
- **技能文件夹**：`编号-中文名/`，如 `W6-视频创作/`、`S1-需求对齐与文档/`
- **技能定义文件**：统一用 `SKILL.md`
- **配套脚本**：保持原文件名（.py / .js / .sh 等）

## AI 的角色

- Kevin 需要 AI 帮忙做任何事的时候，AI 来这里找对应技能
- 进入任何技能子文件夹前，先读该文件夹的 SKILL.md
- 如果找不到合适技能，告诉 Kevin，问是否需要创建新技能

## 依赖管理

Python 和 Node 统一用版本管理工具安装，详见 `CLAUDE.md` 的"依赖安装"章节。

各项目自己的依赖：
- Python 项目：在项目目录建 `venv` 虚拟环境，`pip install -r requirements.txt`
- Node 项目：在项目目录 `pnpm install`

## 关于 Tools/（实用脚本）

`Tools/` 目录下是独立的实用脚本（不绑定到某个技能），通过"兔子"、"tools"等关键词触发。
进入 Tools/ 前先读该目录的 instructions.md。

## 技能迭代循环

Skills 的质量取决于参考文档的质量，而参考文档的质量取决于笔记库的质量。这是一个循环：
1. 你做项目时积累的经验 → 更新到 Projects/ 或 Reading/ 的笔记里
2. 这些笔记积累到一定程度 → 提炼成 Skills 的参考文档
3. Skills 质量提升 → 下次做类似项目输出更好
4. 输出更好 → 记更多笔记 → 回到第 1 步

**随着日记月累，这些 Skill 也需要根据我的个人风格不断去优化。**
