# 通用原理：从模型机制推导策略

> 这些原理不依赖具体模型——不管下一个生图模型是什么架构，扩散去噪的物理过程、训练数据分布的统计特性、编码器的语义映射规律不会变。

---

## 原理1：扩散去噪过程——提示词是方向指令

**机制**：扩散模型从纯噪声开始，逐步去噪生成图像。提示词控制的是"往什么方向去噪"——它在每一步采样中提供一个条件梯度，引导去噪轨迹朝提示词描述的图像分布收敛。

**推导链**：
- 去噪是逐步的 → 早期步骤决定全局结构，后期步骤只做细节调整
- 提示词提供方向 → 冗余信号使方向模糊，矛盾信号使方向震荡
- 去噪终点是分布中的一个点 → 同一提示词每次采样可能落到分布的不同位置（随机性来源）

**策略**：
- **重要信息前置**：主体/构图/风格放前面，细节修饰放后面——早期去噪步先"读"到的是前面的token
- **减法优于加法**：只写控制内容、风格、构图的词，去掉不变词和同义词——每条多余的词都在拉扯去噪方向
- **一次一个意图**：不要在同一提示词中同时要求"冷色调"和"暖色调"——矛盾梯度导致震荡

**边界**：这个原理适用于所有基于扩散的模型（SD/Flux/MJ/Imagen等）。自回归模型（如Parti）的生成机制不同，但"冗余信号稀释有效信号"的结论仍然成立——因为文本编码器的注意力机制是共通的。

## 原理2：训练数据分布——Generic是默认值

**机制**：模型的"默认输出"是训练数据的高概率区域。LAION等大规模数据集中，"高质量"图像高度集中在几种主流审美——高饱和、完美对称、均匀照明、正面构图。这不是模型"喜欢"这些，而是统计学上这些特征在训练分布中密度最高。

**推导链**：
- 训练数据有偏 → 模型的先验分布偏向主流审美
- 提示词越模糊 → 去噪方向越自由 → 越容易收敛到高概率区域
- 高概率区域 = "平均好图" = Generic

**策略**：
- **特异性描述**：不说"漂亮眼睛"，说"琥珀色瞳孔带金色放射纹"——特异性词将去噪方向从高概率分布拉向低概率但更符合意图的区域
- **反常规组合**：不同功能×不同风格交叉——"蒸汽朋克医疗室"、"水彩风的机甲"——打破训练数据中的常规配对
- **风格锚定**：指定具体艺术媒介/年代/流派——"1970年代丝网印刷"比"复古风格"编码更精确，因为训练数据中"1970s silkscreen"与特定视觉特征共现频率高

**边界**：特异性有上限——如果描述太罕见（训练数据中几乎不存在），模型无法生成，会退回到最近的已知分布。策略是"朝低概率方向推一步"，不是"推到分布之外"。

## 原理3：注意力分配——零和博弈

**机制**：Transformer自注意力中，每个token对其他token的注意力权重经过softmax归一化——总和为1。这意味着注意力是零和资源：给A多一点，B就少一点。位置编码使靠前的token在注意力计算中获得更多权重。

**推导链**：
- softmax归一化 → token越多，每个token的平均注意力越少
- 位置编码效应 → 靠前token权重更大
- 100词的提示词中每个词影响力只有50词的一半

**策略**：
- **删不变词**：去掉"high quality, best quality, masterpiece"——如果去掉它输出没变化，说明它没在起作用，只是稀释了有效词的注意力
- **去同义词**：只保留一个——"beautiful, gorgeous, stunning"三个词和写一个效果一样，但占三倍注意力
- **主体前置**：最重要的内容放最前面——位置效应使靠前token先建立"锚点"
- **SD 75-token分组**：CLIP每75个token独立编码，跨组无注意力交互。超过75词时用BREAK主动控制分组边界，把相关概念放在同一组

**边界**：T5-XXL（Flux的编码器）支持512 token且无分组，注意力稀释问题大幅缓解但不消失。GPT-4o编码器支持极长上下文，但过长提示词仍会引入噪声。原则不变：每条词都应该有存在的理由。

## 原理4：编码器架构——为什么提示词不能跨模型迁移

**机制**：同一个词在不同编码器中被映射为不同的语义向量。CLIP将"best quality"编码为"高质量图像"方向的向量（在SD中确实有用）；T5-XXL将"best quality"编码为自然语言中该短语的语义——可能关联到广告语、产品评价等非视觉内容（在Flux中无效或有害）。

**推导链**：
- 编码器不同 → 同一文本→不同语义向量→不同图像分布
- 编码器训练数据不同 → 同一词的"视觉锚点"不同
- 因此：SD的逗号词组写法在Flux中不工作，Flux的自然语言写法在SD中效果差

**跨模型翻译规则**：

| SD → Flux | 变化 |
|-----------|------|
| 逗号词组 | → 自然语言长句 |
| "best quality, masterpiece" | → 不需要，删除 |
| 负向词 | → 不需要，删除 |
| "(word:1.3)" 权重 | → 不支持，用自然语言强调 |
| "BREAK" 分组 | → 不需要，T5无75-token限制 |
| "one girl" | → "girl"（T5将"one girl"关联动漫） |

| SD → MJ | 变化 |
|---------|------|
| 逗号词组 | → 描述性语言 + 参数 |
| 负向词 | → `--no` 参数 |
| 权重语法 | → `::数值` 多概念权重比 |
| 构图词 | → `--ar` 宽高比更直接 |

| 任意模型 → Ideogram | 变化 |
|---------------------|------|
| 自然语言 | → JSON结构化格式更精确 |
| 文字内嵌描述 | → 独立text元素+坐标位置 |

**边界**：跨模型翻译不是逐词替换——是理解"这条提示词在原模型中实现了什么视觉意图"，再用新模型的格式重新表达这个意图。关键变量是编码器架构：CLIP→词组，T5→自然语言，Qwen-VL→结构化，GPT-4o→对话式。

## 原理5：条件生成——你不写的比写的更重要

**机制**：在条件生成中，提示词提供条件信号，模型在条件约束下采样。未提及的维度，模型从无条件先验（训练数据分布）中自行填充。这意味着：**你不显式控制的每个维度，都在被模型的默认偏好接管。**

**推导链**：
- 条件信号只覆盖提示词提及的维度
- 未提及维度 → 无条件先验 → 训练分布的统计平均
- 如果你没写光影 → 模型用最常见的光照（正面均匀照明）
- 如果你没写构图 → 模型用最常见的构图（正面居中）

**策略**：
- **显式声明关键维度**：主体、风格、构图、光影、配色——这5个维度至少各写一句
- **SD中省略=不可控**：CLIP会对未提及元素做条件概率推断（"地板→房间→窗户"），构图不可控
- **Flux中省略=隐式控制**：T5-XXL对未提及元素不做概率补充，"只写什么出什么"——利用信息不对称控制构图（想俯拍只写地面元素）
- **MJ中省略=审美接管**：MJ对未提及维度做高级审美加工，结果好看但不可控

**边界**：显式声明不等于堆砌——每条声明都应该有明确目的。"写着玩"和"显式控制"的区别是：你知道为什么要写这个词，以及去掉它会变什么。

## 原理6：光照描述的编码精度梯度

**机制**：文本编码器对光照术语的编码精度，取决于该术语在训练数据中与特定视觉模式的共现频率。"Rembrandt lighting"在训练数据中与"面颊三角形光斑"高度共现→编码精确→模型响应直接。"dramatic lighting"对应的光照模式千差万别→编码模糊→模型取平均→"有点戏剧性但说不上来哪种"。

**推导链**：
- 共现频率高 → 语义向量指向窄分布 → 编码精确 → 可控
- 共现频率低/含义宽泛 → 语义向量指向宽分布 → 编码模糊 → 不可控
- 同理适用于：构图词、风格词、材质词

**光照描述三要素**（跨模型通用）：
1. **光源类型**：key light / rim light / backlight / volumetric light / softbox
2. **方向**：from upper left / from behind / overhead / 45-degree angle
3. **色温/质感**：warm 3200K / cool 6500K / golden hour / harsh specular

模糊词→精确词的替换表：

| 模糊词 | 精确替代 |
|--------|---------|
| dramatic lighting | Rembrandt lighting from upper left |
| beautiful light | soft diffused key light from overhead, warm 3200K |
| good lighting | three-point studio lighting, key from 45° left |
| dark lighting | low-key single source from behind, rim light only |
| moody atmosphere | chiaroscuro, deep shadow on 80% of frame, single warm key |

**扩展**：这个"共现频率→编码精度"的规律不只适用于光照，也适用于构图词（"close-up"比"rule of thirds"共现频率高→更可靠）、材质词（"brushed aluminum"比"metallic"精确）、风格词（"1970s silkscreen poster"比"retro style"精确）。

## 原理7：一致性的本质——锚定在风格空间中

**机制**：AI生成的一致性问题根源是"每次生成都在风格空间中做独立采样"——不同批次的生成会漂移到风格空间的不同区域。一致性=在风格空间中选定锚点，将所有生成都约束在锚点附近。

**推导链**：
- 每次独立采样 → 风格漂移
- 固定种子+参数 → 固定采样起点 → 缩小漂移范围
- 图生图+低重绘幅度 → 以锚点图为条件 → 约束去噪方向
- 风格LoRA → 修改模型权重 → 将整个输出分布朝特定风格偏移

**一致性控制强度梯度**（从弱到强）：
1. 固定风格提示词（弱——仍有随机漂移）
2. 固定种子+参数（中——相同输入→相同输出）
3. 图生图+低重绘幅度0.3-0.5（强——以参考图为锚）
4. ControlNet（很强——像素级约束）
5. 风格LoRA/微调（最强——修改模型本身）

**场景应用**：
- 角色一致性：先出一张满意的角色图→图生图+低重绘幅度出其他角度/场景
- 图标一致性：固定种子+统一风格词，只换主体词
- 游戏美术一致性：先生成10-20张风格探索图→选3-5张作"美术圣经"→后续全部基于圣经参数
- 产品摄影一致性：固定构图+光影+背景模板，只换产品

**边界**：锚定和创意是矛盾的——锚定越强，自由度越低。批量生产用强锚定，创意探索用弱锚定或无锚定。

## 原理8：文字渲染的编码瓶颈

**机制**：CLIP等视觉-语言模型的文本编码器设计目标是将语言映射到视觉语义空间——"猫"映射到"猫的视觉概念"。但拼写（"C-A-T"→"猫"）和排版（字母的空间位置关系）不在设计目标中，因此CLIP"理解猫的概念但不理解C-A-T的拼写"。

**推导链**：
- CLIP不理解拼写 → SD文字乱码
- 需要拼写理解 → 要么用专门训练的模型（Ideogram的Qwen-VL），要么用多模态理解（GPT Image 2的GPT-4o）
- 文字渲染能力 = 编码器的拼写理解能力

**跨模型文字渲染能力梯度**：
- GPT Image 2：英文99%+，中文高准确率
- Ideogram v3：英文90-95%，JSON格式精确控制位置
- Seedream 5.0：中文强，英文一般
- Flux 2 Pro：约65%
- MJ V7：40%以下
- SD 1.5/XL：几乎不可渲染

**策略**：
- 需要精确文字 → 选文字渲染强的模型，文字内容用引号包裹
- SD中需要文字 → 用ControlNet+预排版文字图
- 多语言文字 → 中文用Seedream/GPT Image 2，英文用Ideogram/GPT Image 2

**边界**：即使文字渲染强的模型，复杂排版（弧形文字、多行对齐、混合字体）仍需后处理。AI文字渲染适合"文字作为画面元素"，不适合"文字作为信息载体需要精确排版"。
