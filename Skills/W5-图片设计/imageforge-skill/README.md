# imageforge-skill

> AI生图决策引擎——不是给人读的知识库，是给Agent的可执行决策链。

## 这是什么

ImageForge是一个AI生图的决策引擎Skill——Agent收到生图请求后自动解析、路由、组装、质检、诊断。当用户向Agent发送生图请求时，Agent按5步流程自动处理：

1. **请求解析** — 提取场景/主体/风格/参考图/约束5个slot
2. **场景路由** — 按信号关键词决策加载哪个场景文件
3. **Prompt组装** — 用模板+填充规则+示例，而非自由写作
4. **质量门控** — 生成前必检8项，不通过回Step 3
5. **失败诊断** — 按场景分类的诊断路由表+12条反模式触发信号

## 为什么是"决策引擎"而不是"知识库"

人类读知识库→理解→自己写prompt。Agent的链路不同：收到请求→解析意图→路由→组装→质检→诊断失败。ImageForge按Agent链路设计，每一步都是可执行决策。

- `rules.md`：19条if-then可执行规则，不是教育原理
- 场景文件：模板+填充规则+示例，不是"核心挑战→控制维度→提示词模式"
- 反模式：带触发条件的决策信号，不是静态列表

## 设计原则

1. **场景经验 > 模型经验**：模型会跌落神坛，场景经验万变不离其宗
2. **决策规则 > 教育原理**：if-then比"理解为什么"更适合Agent执行
3. **模板填充 > 自由写作**：Agent需要slot和填充规则，不是灵感
4. **诊断路由 > 修复建议**：按场景分类的失败→修复映射，带规则引用
