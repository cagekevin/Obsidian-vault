## 1. 平台规格与画布系统

### 画布尺寸

| 平台 | 尺寸 | 比例 | CSS Class |
|---|---|---|---|
| 小红书图文 | 1080 × 1440 px | 3:4 | `.poster.xhs` |
| 微信方封面 | 1080 × 1080 px | 1:1 | `.poster.square` |
| 微信主封面 | 2100 × 900 px | 21:9 | `.poster.wide` |

### 安全区

| 方向 | 3:4 | 1:1 | 21:9 |
|---|---|---|---|
| 左右内边距 | 88px | 88px | 120px |
| 上下内边距 | 96px | 88px | 88px |
| issue-strip 底部位置 | left:88px / right:88px / bottom:56px | 同左 | left:120px / right:120px |

### 输出规范

- 导出格式：PNG（文字多时首选），纯图片场景可用 JPEG
- 缩放因子：2x（Playwright 截图时 deviceScaleFactor=2）
- 安全区规则：所有文字和关键视觉元素必须在安全区内
- 推荐页面数：1封面 + 4~8内容页 + 1总结/清单页

### 2.6 Midnight Ink（午夜墨·暗色——唯一官方暗色调）

> 仅用于游戏关键美术、夜景摄影、电影感封面。不得自创其他暗色主题。

```css
--paper:       #0e0d0c;
--paper-2:     #1a1714;
--ink:         #ece2cf;
--muted:       #9a8c75;
--line:        rgba(236,226,207,.22);
--accent:      #d4a04a;
--accent-soft: #3a2a14;
--ink-rgb:     236,226,207;
--paper-rgb:   14,13,12;
--accent-rgb:  212,160,74;
```

Midnight Ink 必须额外覆盖两个背景层：

```css
[data-theme="midnight-ink"] .grain {
  opacity: .26;
  mix-blend-mode: screen;
  background-image: radial-gradient(rgba(255,244,214,.10) 1px, transparent 1px);
}
[data-theme="midnight-ink"] .paper-wash {
  background:
    radial-gradient(80% 50% at 28% 16%, rgba(212,160,74,.12), transparent 64%),
    radial-gradient(70% 60% at 80% 86%, rgba(60,40,20,.20), transparent 72%),
    linear-gradient(180deg, rgba(236,226,207,.02), rgba(0,0,0,.32));
}
[data-theme="midnight-ink"] .frame-img {
  background: #18120f;
  box-shadow: 0 0 0 1px rgba(236,226,207,.10);
}
```

### 颜色使用规则

- `--paper` 是主底色，`--ink` 是主文字色
- `--accent` 要节省使用：分段标记、页码、拉取引用分割线、或一个高亮短语
- `--paper-2` 用于图片容器底色、issue-strip 区域、checklist 带
- 浅色主题（前五个）：确保有真实对比度，不要变成"米色叠米色"
- Midnight Ink：不要叠不透明卡片或填充块。深色杂志依赖图片出血 + 暖金色强调来建立层次，不是用背景块

### Editorial 没有卡片类

**Editorial 模板不提供任何 `.card-*` 类**。杂志风通过字重、分割线、ledger 行、列结构来建立层次——不是卡片底色。

```css
/* ✗ 错误：在 Editorial 中用卡片填充块 */
<div class="card-fill">...</div>

/* ✓ 正确：用 ledger 行或列结构 */
<div class="ledger">
  <div class="ledger-row">...</div>
</div>
```

如果发现自己想在 Editorial 页面上加一个卡片背景 → **风格选错了**，应该切换 Swiss。

## 3. 文字字体系统

### 3.1 字体栈

| CSS 变量 | 字体 | 用途 |
|---|---|---|
| `--serif-zh` | `"Noto Serif SC", "Songti SC", "STSong", serif` | **所有**显示标题、正文、引文 |
| `--serif-en` | `"Playfair Display", "Noto Serif SC", serif` | 英文斜体副标题、拉取引用 |
| `--sans-zh` | `"Noto Sans SC", -apple-system, "PingFang SC", "Microsoft YaHei UI", sans-serif` | 工具性文字、后备(极少使用) |
| `--sans-en` | `"Inter", -apple-system, BlinkMacSystemFont, sans-serif` | 中英文混排中的拉丁正文 |
| `--mono` | `"IBM Plex Mono", ui-monospace, "SF Mono", Consolas, monospace` | 标签、页码、分类、图注、issue-strip |

核心原则：**正文必须用 serif-zh，正文不能用 sans**。

### 3.2 字体层级（3:4 默认值）

**硬规则："越大越轻"**。显示标题字重 **500**，宽字距。小字（kicker/meta/label）是唯一可以用 mono 大写 + 宽字距的地方。

| 角色 | Class | 字号 | 字重 | 字距 | 字体族 | 说明 |
|---|---|---|---|---|---|---|
| 大标题 | `.h-display` | 124px | 500 | +.04em | serif-zh | 主封面、章节标题 |
| 章节标题 | `.h-xl` | 88px | 500 | +.03em | serif-zh | 内容页标题 |
| 中标题 | `.h-md` | 56px | 500 | +.02em | serif-zh | 次级标题 |
| 副标题 | `.h-sub` | 36px | 400 it | normal | serif-en | 英文斜体，仅作副标题 |
| 拉取引用 | `.pullquote` | 64px | 500 it | normal | serif-zh | 斜体引文 |
| 导语 | `.lead` | 28px | 400 | normal | **serif-zh** | 段落导语 |
| 正文 | `.body` | 24px | 400 | normal | **serif-zh** | 段落正文 |
| 分类标签 | `.kicker` | 21px | 500 | +.22em | mono | 全大写 |
| 元数据 | `.meta` | 18px | 500 | +.20em | mono | 全大写 |
| 标签 | `.label` | 18px | 500 | +.20em | mono | 全大写 |

### 3.3 各画布尺寸的字体缩放

```css
/* Square (1:1) */
.poster.square .h-display { font-size: 116px; }
.poster.square .h-xl      { font-size: 78px; }
.poster.square .h-md      { font-size: 48px; }
.poster.square .lead      { font-size: 28px; }
.poster.square .body      { font-size: 26px; }

/* Wide (21:9) */
.poster.wide .h-display   { font-size: 128px; }
.poster.wide .h-xl        { font-size: 96px; }
.poster.wide .lead        { font-size: 34px; }
```

### 3.4 中文标题长度表

先根据长度选字号，不可在不缩短文案的情况下缩小字号。

| 标题形态 | `.h-display` | `.h-xl` |
|---|---|---|
| 1行 ≤6 字 | 124px (默认) | 88px (默认) |
| 1行 7-10 字 | 108px | 88px→手动调小 |
| 2行 每行 ≤8 字 | 96px | 80px |
| 2行 任一行 9-12 字 | 84px | 72px |
| 3行(罕见) | 72px | 64px |

### 3.5 `.h-xl` 各画布硬限制

Editorial `.h-xl` 在每种画布上的最大容量——超出则下方内容溢出画布：

| 画布 | 默认字号 | 最多行 | 最多字/行 | 超限后果 |
|---|---|---|---|---|
| `.poster.xhs` (1080×1440) | 88px | 2 | 9 | 3行会把边栏/ledger/pipeline推出底部条 |
| `.poster.square` (1080×1080) | 78px | 2 | 8 | 底部卡片或 meta strip 被裁剪 |
| `.poster.wide` (2100×900) | 96px | 1 | 16 | 换行后挤占右侧边栏列 |

**Per-recipe nuance**（已验证的极限数据）：
- **M11（边栏随笔）**：2行标题 + 3段正文（每段3-4句）+ 5-7行边栏，正好填满1440px
- **M14（纵向流水线）**：2行标题 + **5步**（每步 step-title 42px serif + step-desc 26px sans，28px gap）正好触底——加第6步需要把标题缩到1行
- **M07（结尾页）**：2行标题 + **5条 ledger**（每条 ledger-title + ledger-note ≈ 100-120px行高）正好填满。要6条需标题缩为1行

### 3.6 最小可读字号（手机安全）

1080px 宽的 PNG 在手机上通常缩放为 360-420 逻辑像素宽。下方为最小值，**不可低于**。

| 角色 | 最小值 | 备注 |
|---|---|---|
| 正文/段落 | 28px | Editorial、Swiss 通用 |
| 导语 | 30px | 1.5x 正文保护 |
| 图注/kicker | 20px | 不可低于 18px |
| 标签/meta strip | 20px | mono 大写 |
| 网格单元格标题 | 24px | Matrix/card |
| 数字标注 | 22px | Stat-card/ledger |

## 4. 图片系统

### 4.1 图片容器

所有图片必须用 `.frame-img` 包裹 + 一个标准比例 class：

| Class | 比例 | 用途 |
|---|---|---|
| `.r-3x4` | 3:4 | 人像封面、田野笔记照片 |
| `.r-1x1` | 1:1 | 方形人物、产品、平衡网格 |
| `.r-4x3` | 4:3 | 经典编辑照片、全出血顶部区域 |
| `.r-3x2` | 3:2 | 杂志内文插图 |
| `.r-16x9` | 16:9 | 横图、信息图 |
| `.r-16x10` | 16:10 | 左文右图的默认分割 |
| `.r-21x9` | 21:9 | 微信主封面大片 |

```html
<!-- 标准用法 -->
<figure class="frame-img r-3x4">
  <img src="assets/photo.jpg" alt="描述"
       style="object-position:center 55%">
</figure>
<figcaption class="img-cap">图注文字</figcaption>
```

### 4.2 裁切控制（强制要求）

**每张图片必须手动设置 `object-position`，不可留默认值 `center 50%`**。

| 主体在源图中的位置 | inline `object-position` |
|---|---|
| 主体靠上（天空为主 landscape、脸部在上方） | `center 25-35%` |
| 主体居中（默认） | `center 50%`（可不写） |
| 主体中段（徒步者、手部、中景） | `center 55-65%` |
| 主体靠下（前景装备、脚部） | `center 70-80%` |

**特殊场景**——全出血封面(M16)的 object-position：

| 人脸/焦点在源图位置 | recommended for 3:4 |
|---|---|
| 人脸/焦点在顶部1/3 | `center 25%`（抬起脸部，底部留标题空间） |
| 人脸/焦点在中部1/3 | `center center`（默认，文字去顶部+底部） |
| 人脸/焦点在底部1/3 | `center 70%`（降下脸部，顶部留标题空间） |
| 宽幅风景、无单一主体 | `center 35%`（稍微上移保留地平线） |
| 竖图全身照 | `center top` |

### 4.3 图注 class（易错点）

Editorial 模板的图注 class 是 `.img-cap`（18px mono，全大写）：

```html
<figcaption class="img-cap">图注 · 地理位置 · 参数</figcaption>
```

**⚠️ 不要写成 `.cap`**——`.cap` 不是 Editorial 的 class，会回退到浏览器默认16px，触发最小字号检查失败。

### 4.4 截图容器

```html
<figure class="frame-shot r-16x9 corners-sm shadow-soft bg-paper-2 inset-sub">
  <img src="assets/screenshot.png" alt="截图">
</figure>
```

Style-locked 默认值（不要改）：

| 属性 | 默认值 |
|---|---|
| 圆角 | `.corners-sm`（6px） |
| 阴影 | `.shadow-soft` |
| 底色调 | `.bg-paper-2` |
| 内边距 | `.inset-sub`（24px） |

其他可用色调：`.bg-paper`、`.bg-ink`、`.bg-grid`（网格背景）、`.bg-dot`（点阵背景）

### 4.4 设备外壳

```html
<!-- 浏览器外壳 -->
<div class="device-browser">
  <div class="frame-shot r-16x9">
    <img src="assets/webpage.png" alt="网页">
  </div>
</div>

<!-- 手机外壳 -->
<div class="device-phone">
  <img src="assets/app.png" alt="App截图">
</div>
```

### 4.5 M16 全出血封面图片规范

当图片覆盖整个画布时，必须通过两项测试，**不可用遮罩补救**：

**静区测试**：图片中有 ≥30% 画布的区域是低细节/低对比/均匀的（失焦背景、深阴影、雾、平静水面、天空、虚化草地）。这是标题要放的地方。

**光线测试**：图片承载氛围感/克制的光线（阴天、晨雾、黄金时刻、林下、胶片柔和、黄昏剪影）。拒绝高饱和正午、机顶闪光灯、游客照。

两项测试都通过 → 才可选用 M16。任一失败 → 换 M01（拆分布局，图片放框内）。

### 4.6 全出血页面的页面序列约定

当用户有多张好照片（旅行/户外/家居）时，不要一次性把所有大片都放前面：

```
P1     M16 Image-Led Cover     ← 最强一张照片全出血，短标题
P2     M05 / M10 / M11          ← 文字密集型：清单/证据/边栏——眼睛休息
P3     M02 田野笔记照片         ← 第二好照片，带框+图注
P4-N   M02 / M11               ← 更多田野笔记或边栏随笔
P_last M07 / M04               ← 结尾页或拉取引用
```

**硬规则：不可连续两张 M16**。全出血后眼睛需要靠文字落定才能接受下一张大片。连续两张 M16 = "选不出用哪张照片"。

## 5. 排版系统（布局基元）

### 5.1 基础布局

| Class | 属性 | 用途 |
|---|---|---|
| `.stack` | `flex-direction: column` | 默认垂直流 |
| `.row` | `flex-direction: row` | 水平流 |
| `.grow` | `flex: 1` | 填充剩余空间 |
| `.center` | `text-align: center` | 居中 |
| `.pad-y-md` | `padding-top/bottom: 36px` | 中等垂直间距 |
| `.rule` | `height:1px, background:var(--line)` | 细分割线 |
| `.rule-accent` | `height:2px, background:var(--accent), width:96px` | 强调分割线 |

### 5.2 间隙系统

| Class | 值 | 典型用途 |
|---|---|---|
| `.gap-1` | 12px | 紧密间距 |
| `.gap-2` | 24px | 小间距 |
| `.gap-3` | 36px | 默认段间距 |
| `.gap-4` | 48px | 大块内容间距 |
| `.gap-5` | 64px | 主要内容断点 |

### 5.3 列系统

| Class | 属性 | 用途 |
|---|---|---|
| `.col-2` | `grid-template-columns: 1fr 1fr; gap:48px` | 两等分 |
| `.col-3` | `grid-template-columns: repeat(3, 1fr); gap:36px` | 三等分 |
| `.col-2-7-5` | `grid-template-columns: 7fr 5fr; gap:48px` | 左文右图的引文+图片 |
| `.col-2-8-4` | `grid-template-columns: 8fr 4fr; gap:48px` | 左重文+右侧栏 |

### 5.4 边栏组件（Marginalia）

```html
<div class="marginalia">
  <div>
    <!-- 主内容 -->
  </div>
  <div class="mg-col">
    <p>关键词·引用</p>
    <p>脚注·补充</p>
  </div>
</div>
```

CSS：左侧 1fr，右侧 220px 固定宽度列，中间 48px gap，右侧列左侧有 `border-left:1px solid var(--line)`。

### 5.5 账本/清单组件（Ledger）

```html
<div class="ledger">
  <div class="ledger-row">
    <span class="ledger-nb">01</span>
    <span class="ledger-title">产品名</span>
    <span class="ledger-note">一句话说明</span>
  </div>
  <!-- 继续 4-6 行 -->
</div>
```

CSS：每行 `grid-template-columns: 96px 1fr auto`，`gap:24px`，`padding:28px 0`，底部分割线。

### 5.6 纵向流水线组件（Pipeline-v）

```html
<div class="pipeline-v">
  <div class="step">
    <div class="step-nb">01</div>
    <div>
      <h3 class="step-title">步骤标题</h3>
      <p class="step-desc">步骤描述</p>
    </div>
  </div>
  <!-- 3-5 步 -->
</div>
```

CSS：每步 `grid-template-columns: 80px 1fr; gap:28px`，底部分割线。最多 5 步。

### 5.7 前后对比组件（Before/After）


## 6. 装饰系统（背景层）

### 6.1 层模型

杂志风背景由 **3-5个层** 叠成（不得使用纯色平铺）：

1. **纸张底色**：`--paper`（来自主题色盘）
2. **纸纹颗粒**：`.grain`
3. **墨迹/轮廓场**：`.paper-wash` 或 WebGL canvas（`.mag-bg`）
4. **可选墨迹晕影**：`.ink-vignette`——增强封面/氛围页的深度
5. **可选分割线**：内容分隔用 hairline（不是整页网格）
6. **内容层**：`.content`

封面氛围页还可通过 `.poster.hero` class 增强 WebGL 可见度（opacity 从 .34 升到 .62）。

### 6.4 WebGL 背景

使用 `assets/magazine-bg-webgl.js` 提供的确定性墨流背景：

```html
<canvas class="mag-bg" data-bg="ink-flow"></canvas>
<script src="assets/magazine-bg-webgl.js"></script>
<script>
  (function () {
    function cssVarRgb(name, fallback) {
      var raw = getComputedStyle(document.documentElement).getPropertyValue(name).trim();
      if (!raw) return fallback;
      var parts = raw.split(',').map(function (x) { return parseInt(x.trim(), 10); });
      return parts.length === 3 && parts.every(function (n) { return !isNaN(n); }) ? parts : fallback;
    }
    function mountAll() {
      if (!window.MagazineBg || !window.MagazineBg.mount) return;
      var ink    = cssVarRgb('--ink-rgb',    [10, 10, 11]);
      var paper  = cssVarRgb('--paper-rgb',  [243, 240, 232]);
      var accent = cssVarRgb('--accent-rgb', [17, 17, 17]);
      document.querySelectorAll('canvas.mag-bg').forEach(function (c) {
        window.MagazineBg.mount(c, {
          ink: ink,
          paper: paper,
          accent: accent,
          strength: 0.32,
          frozenTime: 12.5
        });
      });
    }
    if (document.readyState === 'complete') mountAll();
    else window.addEventListener('load', mountAll);
  })();
</script>
```

### 6.5 何时加强/减弱背景可见度

**强背景（WebGL 明显可见）**：封面、章节分割页、拉取引用、稀疏论点页、结尾页。

**弱背景（WebGL 很淡）**：截图页、密集的账本、清单、产品证据页。

### 6.6 WebGL 后备方案（2D Canvas）

如果 WebGL 不可用，创建 2D canvas 模拟墨迹氛围：

- 大径向墨迹渐变色块
- 数条半透明轮廓弧
- 柔和纸纹噪声或不均匀墨迹洗涤
- 极低透明度的主题色

### 6.7 禁止

- ❌ 亮色渐变
- ❌ 整页网格、点阵、方格纸
- ❌ 装饰性的斑点/圆圈（与布局无关）
- ❌ 正文后面放强背景标记
- ❌ WebGL 遮挡截图或小号图注

## 9. 全局规范与硬规则

### 9.1 风格身份测试

有三条全部满足才是真正的杂志风：

1. 背景至少有一层氛围层：纸纹、墨迹、冻结 WebGL 画布或轮廓场。**纯色平铺 `#f3f0e8` 不是 Editoria**l。
2. 显示标题使用 serif-zh / serif-en 字体族。
3. 页面上至少有一个：大图井、serif 拉取引用（带英文斜体）、左右边栏列、或有真正杂志行层次的 ledger。

如果页面有 serif 标题、mono 标签到处用、带结构化的 pill 网格、平铺纸色背景 → 那是 "有 serif 的 Swiss"，不是 Editorial。

### 9.2 反模式

**反模式 A：杂志无氛围**
- 单纯色纸背景，无纸纹、无墨迹、无 WebGL
- mono 标签（IBM Plex Mono）在每个 kicker/foot/pill 用
- 一个 serif 标题孤零零地漂浮，没有大图、拉取引用或边栏列

修复：加纸纹+墨迹背景，或把 mono 标签替换为 serif/italic 微文字，或引入大图/serif 拉取引用。

**反模式 B：底部重叠**
`.foot` / `.issue-strip` 用 `position:absolute; bottom:...` 时，上方内容可能涨破底部。安全模式：flex column `.stack` 中 `.foot` 用 `margin-top:auto`。

**反模式 C：全出血封面标题穿过主体**
先检查图片是否通过静区+光线测试，再映射主题位置，最后设置 `object-position`。

### 9.3 硬规则

1. 一个套图选择一个风格模式，**不可混用 Editorial 和 Swiss**。
2. 一个套图选择一个主题色盘。
3. ❌ 禁止使用 emoji。用 Lucide 图标或克制文字。
4. ❌ 禁止假数据、假百分比、`Lorem` 文本。
5. ❌ 禁止 `border-radius`、`box-shadow`、`linear-gradient`（Editorial 允许框影用于截图容器，但不可用于卡片背景）。
6. ❌ 杂志风禁止纯一色背景——must 有 paper grain + 可选 WebGL。
7. 每张 poster 必须有固定的导出尺寸。不可用 `vw`/`vh`。
8. 每张图片必须用 `.frame-img` + 标准比例 class 包裹。
9. 尊重字号层级和最小尺寸。先砍文案，不可缩小字号。
10. 正文必须用 serif-zh。
11. "越大越轻"：显示标题字重 ≤500。
12. 3:4 卡内容必须覆盖 ≥75% 画布高度。
13. 每张 poster 最多一个 issue-row 或一个 issue-strip，不可同时存在。

### 9.4 人像填充规则

3:4 画布的纵向分区：

| 分区 | 典型高度 | 角色 |
|---|---|---|
| 头部/meta | 56-90px | issue 标签、分类、页码 |
| 标题/导语 | 240-420px | hook 和主张 |
| 证据/正文 | 520-760px | 图片、ledger、图表、对比 |
| 底部带 | 90-180px | 图注、清单、摘要 |
| 页脚 | 52-86px | issue/页元数据 |

检查气味：
- ❌ ledger/表格占画布 <45%
- ❌ 页面底部 25% 空
- ❌ 4 条列表项写成短行，大量空余在下面
- ❌ 截图太小
- ❌ 正文居中在中间，无上下节奏

### 9.5 图片覆盖文字规则

在任何文字叠加在图片上的页面上，必须执行以下顺序：

**第1步 · 选片（主要杠杆）**：通过静区测试 + 光线测试。任一项失败 → 换图或换 M01。

**第2步 · 先尝试无遮罩**：把标题放在图片的静区内。大多数编辑图片在这一步就能通过。

**第3步 · 局部色调遮罩（仅后备）**：只有第2步对比度失败才用。三条规则：
1. **局部**：仅在标题区域。用 `radial-gradient` 中心在标题块上。
2. **色调取自图片**：从图片里取一个深色。森林 → `#1a2818`。黄昏 → `#4a2638`。**不能用纯黑 `#000`**。
3. **柔和**：峰值 α 0.15-0.30。需要 >0.40 才能读 → 图片失败了，回第1步。

**遮罩禁止项**（image-overlay.md 中列出的每条都需遵守）：
- ❌ 统一全画布垂直渐变衰减（"暗角"全局遮罩）——那是游戏主视觉处理，不是杂志
- ❌ 纯黑 `#000` 遮罩色——必须取自图片色调
- ❌ 标题背后放纯黑/白矩形块——读起来像截图画标注
- ❌ `mix-blend-mode: difference` 提高对比度——破坏肤色、打印时出错
- ❌ `img { opacity: .6 }` — 杀死照片的层次感
- ❌ 不先尝试 Step 2（无遮罩）就直接跳到 Step 3

**第4步 · 缩略图对比度检查（始终）**：缩放到 360px 宽检查标题是否可读。

**主题发现**：在 hero block 旁边写 HTML 注释记录主体位置：
```html
<!-- subject map:
     face/skull: 50% x 45% y, occupies ~30% of frame
     safe text zone: top band (0-15% y) and bottom band (65-100% y)
-->
```

### 9.6 页面变化节奏

7 页小红书套图，至少用 5 种不同形态：
- 封面 / 特色图
- 短文拆分
- 高账本
- 证据截图
- 拉取引用或氛围论题
- 清单/对比
- 结尾页

**避免重复"标题+导语+3 行"超过两次。**

## 附：生成流程参考

### 步骤 0 — 初始化

创建项目目录（按项目所在目录的规范命名，如 `Projects/HKH品牌/YYYY-MM-DD-项目名/`）
目录结构：
```
├── index.html           ← 复制种子模板
├── assets/              ← 图片素材
└── output/              ← 导出文件
```

### 步骤 1 — 选风格 + 主题

- 在 `<html>` 上设置 `data-theme` 选择 6 个主题色盘之一
- 在 `<head>` 加载 Google Fonts（Noto Serif SC、Noto Sans SC、Playfair Display、Inter、IBM Plex Mono）

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@400;500;700;900&family=Noto+Sans+SC:wght@300;400;500;700&family=Playfair+Display:ital,wght@0,400;0,700;1,400&family=Inter:wght@300;400;500;700&family=IBM+Plex+Mono:wght@400;500&display=swap">
```

### 步骤 2 — 填充海报

在 `<!-- POSTERS_HERE -->` 区域填入多组 `<section class="poster xhs">`（M01-M16 选择对应 recipe 的 HTML 结构）。

### 步骤 3 — 渲染

```bash
# 截图（输出 PNG + alpha 透明版 + PSD 图层碎片）
node <skill-path>/assets/render.cjs <project-dir>

# 合成 PSD
node <skill-path>/assets/psd-composer.cjs <project-dir>
```
