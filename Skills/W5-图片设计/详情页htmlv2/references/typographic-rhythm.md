## 一、本规范要解决的 3 个反复出现的问题

| 问题 | 现象 | 根源 |
|---|---|---|
| **断行孤儿字** | 主标第 2 行只剩 1-2 字 | 字数估错 / 未做断行预设 |
| **940px 溢出 / 空洞** | 高度 > 940 或 < 600 | 预算用 clamp **上限**算（实际渲染是**下限**） |
| **没层次** | 主标/子标/正文视觉等同 | 字号阶梯过小（< 2.5x）、子标违规抢戏 |

## 二、实际渲染字号表（**写屏前必查**）

> 容器宽 790px / 内容宽 694px（padding 48×2）/ Fraunces + Noto Serif SC

| class | clamp 范围 | **实际渲染（@ 790px）** | 单行可容纳字符 | 单行可容纳中文 |
|---|---|---|---|---|
| `.display-xxl` | clamp(48px, 7vw, 72px) | **55px**（7vw=55.3px）| ~10 字符 | **5-6 字** |
| `.display-xl` | clamp(44px, 5.5vw, 60px) | **44px**（5.5vw=43.5px→下限）| ~16 字符 | **8-9 字** |
| `.display-lg` | clamp(32px, 4vw, 48px) | **32px**（4vw=31.6px→下限）| ~22 字符 | **11-12 字** |
| `.h1-sm` | 36px 固定 | **36px** | ~20 字符 | **10-11 字** |
| `.deco-en` | 18px 固定 | 18px | ~38 字符 | 不用中文 |
| `.body` | 14px 固定 | 14px | ~50 字符 | **20-22 字**（换行 1 行 ≈ 50px 高）|
| `.meta` | 12px 固定 | 12px | ~58 字符 | 25-28 字 |
| `.lead` | 16px 固定 | 16px | ~44 字符 | 17-18 字 |
| `.kicker` | 11px 固定 | 11px | ~64 字符 | 不用中文 |

**关键发现**（v1 草案）：`.display-xl` 在 790px 视口下命中 **clamp 下限 44px**（不是预算时以为的 60px 上限）—— **预算时按实际渲染值算，不要按上限**。

## 三、主标字数与断行铁律（**硬约束**）

### 3.1 字数上限（写之前数好）

| class | **1 行排** | **2 行排** | **3 行排**（禁止）|
|---|---|---|---|
| `.display-xxl` 55px | ≤ 6 字 | 7-12 字 | > 12 字 |
| `.display-xl` 44px | ≤ 9 字 | 10-18 字 | > 18 字 |
| `.display-lg` 32px | ≤ 12 字 | 13-24 字 | > 24 字 |
| `.h1-sm` 36px | ≤ 11 字 | 12-22 字 | > 22 字 |

### 3.2 断行铁律（写时必用 `<br>` 强制断）

> **核心铁律**：断行**不是禁止**，是**每行都不能只剩 1-2 字**。

| 规则 | 说明 | 反例 |
|---|---|---|
| **重头在上** | 第 1 行字数 ≥ 第 2 行 | ❌ "肌肤的悄悄话 / 35 岁之后" |
| **每行 ≥ 3 字**（无孤儿）| **每行**最少 3 字（含末行 / 第 2 行 / 第 3 行）| ❌ "肌肤都找到了 / 逆龄的 / 你"（末行 1 字）<br>❌ "肌肤的悄悄 / 话。"（第 2 行 1 字）|
| **标点同行** | 句号 / 逗号不单独行 | ❌ "肌肤的悄悄话 / 。" |
| **中英不断** | 英文单词不拆到 2 行 | ❌ "Plant / Retinol" |
| **em 不孤立** | `<em>` 标签内 ≥ 2 字 | ❌ "肌肤的<em>悄悄</em>话"（如果 em 只 2 字） |

**用户原话（v0.5.1）**：
> "不是说不能断行，是说你断行之后，不能第二行只有一个字或者两个字。"

**精确化**：断行后**每一行**最少 3 字。**不止第 2 行**——第 3 行、末行也算。

**自检 grep**（HTML 文件）：
```bash
# 找"<br>"后只跟 1-2 字的模式（中文 + 中文标点）
grep -nE '<br>.{1,2}[，。！？、]' *.html
# 或更宽松：找<br> 后的纯短串
grep -nE '<br>[^<]{1,4}<' *.html
```
**0 命中为 PASS**。

### 3.2.1 强制断句符（**v0.5.2 新增 · 防 LLM Token 盲区**）

> **核心问题**：LLM 通过 Token 而非"字"理解文本，复杂数学公式（每行 ≥ 3 字 / 第 1 行 ÷ 总字数 ≥ 0.45）在长上下文中**极易产生幻觉**。
>
> **优化策略**：把"AI 算断行"换成"用户给断行，AI 替换分隔符为 `<br>`"。

#### 断句符协议

用户提供文案时，**主标题如需换行**，在断行处加 `|`（竖线）作为**强制断句符**。

**AI 行为**：扫描标题中的 `|`，**1:1 替换为 `<br>`**，**不擅自增删 / 调整位置**。

#### 协议对照

| 用户输入 | AI 输出 | 说明 |
|---|---|---|
| `时间，从来不是对手。` | `<p class="display-xxl">时间，从来不是对手。</p>` | 无 `|` = 不换行 |
| `时间，从来 \| 不是对手。` | `<p class="display-xxl">时间，从来<br>不是对手。</p>` | 1 个 `|` = 1 个 `<br>` |
| `她们都找到了 \| 逆龄的开关，<br>下一个就是你。` | `<p class="display-xl">她们都找到了<br>逆龄的开关，<br>下一个就是你。</p>` | 用户混用 `\|` + `<br>` 都允许，**1:1 替换** |
| `配合【HK 逆龄提拉术】，\| 让每一滴都精准 \| 作用于轮廓。` | `<p class="display-xl">配合【HK 逆龄提拉术】，<br>让每一滴都精准<br>作用于轮廓。</p>` | 多 `\|` = 多 `<br>` |

#### 反例（AI 擅自换行）

| 错误做法 | 原因 |
|---|---|
| 用户输入 `时间，从来 \| 不是对手。` | **❌** AI 改成 `时间，从<br>来不是对手。`（位置错）|
| 用户输入 `时间，从来 \| 不是对手。` | **❌** AI 删了 `\|` 直接 `时间，从来不是对手。`（不换行）|
| 用户输入 `时间，从来 \| 不是对手。` | **❌** AI 加额外 `<br>` 成 3 行（无中生有）|

#### 何时提示用户用 `\|`

- Step 3 选排版模板时，**主动问**用户："这段主标需要换行吗？如需要，请在断行处加 `\|`"
- 用户给了整段无 `\|` 的标题，**不能擅自加 `<br>`**——只能：
  1. 整体 1 行排（如字数允许）
  2. 降字号（§3.6）
  3. **回问用户**："这段太长，请提供带 `\|` 的断行版本"

#### 自检（v0.5.2 自动化）

`scripts/dp2-self-check.cjs` 加 #10 项：
- 检查 HTML 中**所有 `<br>` 都是用户提供的**（通过 grep `\|` 在源文案中次数 vs HTML 中 `<br>` 次数——需要更复杂工具链，留 v0.6 写）
- v0.5.2 暂时人工对照

### 3.3 断行预设（写时直接落 `<br>`，不靠浏览器自动断）

**正确写法**（带 `<br>`）：
```html
<p class="display-xl">35 岁之后，<br><em>肌肤的悄悄话</em>。</p>
```

**错误写法**（让浏览器断）：
```html
<p class="display-xl">35 岁之后，肌肤的悄悄话。</p>
```

### 3.4 断行后视觉平衡公式

```
第 1 行字数 × 1.0 + 第 2 行字数 × 1.0 = 总字数
目标：第 1 行 ÷ 总字数 ≥ 0.45（重头在上）
反例：6/14=0.43（勉强）/ 7/14=0.5（理想）/ 9/14=0.64（重）
```

### 3.5 内容宽与 padding 折算（**断行真正的杀手**）

> **核心认知**：断行不是"字数算错"，是**没扣 padding**。
> 屏宽 790px / 内容宽 = 790 - padding × 2。

#### 3.5.1 各 class 的 padding 速查

| class | padding-left/right | 实际内容宽 | 单行可容纳中文字（按 display-xl 44px）|
|---|---|---|---|
| `.mod-top-c` | 48px × 2 | **694px** | 8-9 字 |
| `.mod-body` | 48px × 2 | **694px** | 8-9 字 |
| `.callout-box` | 24px × 2 | 内部 grid-3 各分 ≈ 215px | 4-5 字 |
| **裸屏**（无 padding）| 0 | 790px | 10-11 字 |

#### 3.5.2 难断字警示

**易换行的字**：`廊` `裹` `量` `露` `镶` `酿` `腾` `赢` —— 这些字实际渲染宽度比平均字多 2-4px。

**写时规则**：
- 写完后**手动数最后一行的字数**，**不要**假设浏览器会断得均匀
- 含以上字的句子，**多预留 1-2 字的 buffer**

#### 3.5.3 写屏前必查（断行 4 步）

```
1. 数总字数（含中文标点）
2. 选 class（display-xxl / xl / lg / h1-sm）
3. 查 §3.1 表 → 几行排
4. 查 §3.5.1 实际内容宽 → 单行可容字数
5. 写 <br> 强制断行（不让浏览器自动断）
6. 验证 §3.2 全部 5 条规则
```

**不勾完不落盘**。

### 3.6 大标题字号降级决策树（**写之前先答 4 问**）

> **核心规则**：标题太长 → **不禁止**，可以**降字号**。
> 给用户**两种字号方案**选，**不擅自动**。

#### 3.6.1 4 问（写之前必答）

| 问 | 答案指向 |
|---|---|
| Q1 总字数（含标点）= ？| 查 §3.1 表 |
| Q2 你希望几行排？| 1 行 / 2 行 / 3 行 |
| Q3 这是屏 1 Hero 还是其他屏？| 屏 1 = display-xxl 必选；其他 = display-xl 默认 |
| Q4 是否含品牌前缀？| 是 → 必须 2 行排 + 品牌前缀单独一行 |

#### 3.6.2 决策树

```
Q: 总字数 + 目标行数 + 是否含品牌前缀
│
├─ 屏 1 Hero
│   ├─ ≤ 6 字 → A 方案：display-xxl 72px（1 行排）
│   └─ 7-12 字 → A 方案：display-xxl 60px（2 行排）
│
└─ 屏 2-12（其他）
    ├─ ≤ 9 字 + 1 行 → A 方案：display-xl 44px
    ├─ 10-18 字 + 2 行 → A 方案：display-xl 44px
    ├─ 19-24 字 + 2 行 → A 方案：display-lg 32px（降字号）
    └─ 25+ 字 → 拒绝（必须改文案 / 拆屏 / 改子标）
```

**两套方案对照**（**必须给用户两选**）：

| 场景 | A 方案 | B 方案 |
|---|---|---|
| "她们都找到了逆龄的开关，下一个就是你。"（18 字）| display-xl 44px（2 行排）| display-lg 32px（1 行排，更保险）|
| "衰老应当是优雅的，而非必然。"（12 字）| display-xl 44px（1 行排）| display-xxl 60px（2 行排，更戏剧）|
| "配合【HK 逆龄提拉术】，让每一滴都精准作用于轮廓。"（22 字）| display-xl 44px（2 行排）| display-lg 32px（2 行排，更安全）|

**给用户时**说：
> "这段标题 18 字。**A 方案 display-xl 44px 2 行排**——更戏剧但断行风险高；**B 方案 display-lg 32px 1 行排**——更安全但视觉重音弱。选哪个？"

**不擅自决定字号**。

#### 3.6.4 Few-Shot 断行示例（**v0.5.2 新增 · 防 Token 盲区**）

> **核心洞察**：LLM 通过模仿具体示例的成功率远高于数学计算。**给 3-5 组 Good/Bad 对比**，比纯公式更可靠。

**示例 1**（12 字 · 单行排）：

```html
<!-- ✅ GOOD · display-xl 44px 1 行排 -->
<p class="display-xl">衰老应当是<em>优雅</em>的，而非必然。</p>

<!-- ❌ BAD · 拆 2 行后第 2 行只剩 2 字"然。" -->
<p class="display-xl">衰老应当是<em>优雅</em>的，<br>非必然。</p>
```

**示例 2**（18 字 · 双行排，强制断句符）：

```html
<!-- ✅ GOOD · 用户给 | → AI 替换 <br> -->
<!-- 用户输入：她们都找到了 | 逆龄的开关，下一个就是你。 -->
<p class="display-xl">她们都找到了<br>逆龄的开关，下一个就是你。</p>

<!-- ❌ BAD · AI 擅自加 <br> 导致末行"你" 1 字 -->
<p class="display-xl">她们都找到了<br>逆龄的开关，<br>下一个就是你。</p>
```

**示例 3**（22 字 · 含品牌前缀 + display-lg 降字号）：

```html
<!-- ✅ GOOD · 22 字用 display-lg 32px + <br> 强制断 -->
<p class="display-lg">配合【HK 逆龄提拉术】，<br>让每一滴都精准作用于<em>轮廓</em>。</p>

<!-- ❌ BAD · 22 字硬塞 display-xl → 末行"廊" 1 字 -->
<p class="display-xl">配合【HK 逆龄提拉术】，<br>让每一滴都精准作用于廊。</p>
```

**示例 4**（25+ 字 · 拒绝 + 拆屏）：

```
# 用户给 25+ 字长标题
# AI 不能擅自降字号 / 加 <br>，必须回问：

"这段 28 字超过 display-lg 32px 单行 12 字的限制。3 种处理方案：
 A. 拆成 2 屏（屏 N 主张 / 屏 N+1 论证）
 B. 删字缩到 18 字以内（您给精简版）
 C. 用更小的 .h1-sm 36px（重音减弱）
 选哪个？"
```

**示例 5**（难断字"廊 / 廓 / 裹"）：

```html
<!-- ✅ GOOD · 难断字预留 buffer，降字号到 display-lg -->
<p class="display-lg">精准作用于<em>轮廓</em>，不再下垂。</p>

<!-- ❌ BAD · display-xl 44px 硬塞，难断字"廓"挤成末行 1 字 -->
<p class="display-xl">精准作用于<em>轮廓</em>，<br>不再下垂。</p>
```

#### 3.6.3 含品牌前缀的标题修法

**反例**（V3 常见）：`配合【HK 逆龄提拉术】，让每一滴都精准作用于轮廓。` 22 字塞进 display-xl 44px → 第 3 行只剩"廓"或"廊" 1 字。

**修法 A**（推荐）：拆 2 行，用 br 强制断
**修法 B**（更安全）：降字号到 display-lg
**修法 C**（最安全）：品牌前缀单独 kicker 标签

### 3.7 Emoji 铁律（v0.5 新增 · 用户点出）

> **铁律**：页面 HTML 中**禁止出现任何 emoji**（🎯 🔥 💡 等 Unicode emoji 字符）。

**原因**：
- emoji 渲染依赖系统字体（macOS / Windows / Android 不一致）
- 详情页要"高奢" = "克制"，emoji 廉价感强
- 真实资产优先时，emoji 与产品图视觉冲突

**修法**：
- 用户原文含 emoji（如 `🔴 笑起来法令纹可以夹住一支笔`）→ **删 emoji，保留正文**（"笑起来法令纹可以夹住一支笔"）
- 原文无 emoji → 写时**严禁加 emoji 装饰**
- 用 .icon-circle 替代 emoji 的视觉占位（PRD 原有 38 BEM）
- 例外：**纯装饰性 emoji**（如品牌 logo 旁边 ❀）可保留（**需用户确认**）

**自检**：grep `[\u{1F300}-\u{1FAFF}]` HTML 文件 → 0 命中。

## 四、940px 预算公式（**用实际渲染值，不用 clamp 上限**）

### 4.1 屏 940px 切分模板

```
总高 940px
├── 标题区 mod-top-c（padding-top 80 + 内文 + padding-bottom 24）
├── 自由区 mod-body（flex:1 自动撑满）
└── 屏底区 mod-img（如有，含 margin-bottom 48）
```

### 4.2 标题区高度速算表（**实际渲染 44px display-xl**）

| 屏类型 | kicker 11 | 16 | deco-en 18 | 16 | 主标 N 行 | deco-line 1 | 24 | body 14 (1-2 行) | 24 | **标题区总高** |
|---|---|---|---|---|---|---|---|---|---|---|
| 屏 1 Hero 2 行 | 11 | 16 | 18 | 16 | 55×2=110 | 1+48 | 24 | 50 | 24 | **318** |
| 屏 2 大主张 2 行 | 11 | 16 | 18 | 16 | 55×2=110 | 1+48 | 24 | 50 | 24 | **318** |
| 屏 3 痛点 1 行 | 11 | 16 | 18 | 16 | 44 | 1+48 | 24 | 50 | 24 | **252** |
| 屏 3 痛点 2 行 | 11 | 16 | 18 | 16 | 44×2=88 | 1+48 | 24 | 50 | 24 | **296** |
| 屏 N 1 行+无 body | 11 | 16 | 18 | 16 | 44 | 0 | 0 | 0 | 24 | **129** |

> 公式：标题区 = 80 (padding-top) + 各项实际渲染值 + 24 (padding-bottom)
> **必须用表查**，不能用 clamp 上限估算（实测误差 30-60px）

### 4.3 自由区高度（自动撑满）

```
自由区 = 940 - 标题区 - 屏底 mod-img - 屏底 margin-bottom 48
```

### 4.4 屏底图高度规则

| 屏类型 | mod-img 高度 | 用法 |
|---|---|---|
| Hero 首屏 | `flex:1` 撑满（min-height 360）| 用 `.mod-img.mod-img--hero` |
| 一般屏 | **明确像素 200-280** | 用 `.mod-img`（不用 --hero）|
| 文字屏 | 0（无图）| 不放 mod-img |

### 4.5 填充率硬约束

- 填充率 = (标题区 + 自由区 + 屏底图) / 940
- **≥ 75% 必达**（PRD §2.5 R5）
- **禁止 < 75%** —— 即"标题区 + 自由区 < 705px" 视为空洞违规

## 五、痛点 / grid 块的留白铁律

### 5.1 块与图的关系

```
痛点块底 → 至少 24px 留白 → 屏底图顶
（避免贴边，PRD R7 + 视觉层次）
```

### 5.2 块内层级

```
icon 88 + 16 + 子标（h1-sm 36 或 .lead 16） + 8-16 + body 14
合计 ≈ 142-156px（垂直）
```

### 5.3 块与块的间距

- grid-3 内部 gap = 32px（PRD `.grid-3` 规定）
- **禁止**用更大 gap 拉散（失去分组感）
- **禁止**用更小 gap 挤压（失去呼吸）

## 六、配色与对比铁律

### 6.1 主标与子标层级（避免"没层次"）

| 层级 | class | 字号 | 颜色 |
|---|---|---|---|
| 主标 | `.display-xl/xl` | 44-55px | `var(--text-main)` |
| **金斜体重音** | `<em>` 内 | 同主标 | `var(--accent)` |
| 子标 | `.h1-sm` 36px 或 **`.body` 14px 加粗** | 36 或 14 | `var(--text-main)` |
| 正文 | `.body` | 14px | `var(--text-muted)` |

**反例**（v3 屏 3 痛点子标违规）：子标用 `.lead` 16px + 上面主标 44px，**差 28px 视觉不拉开** → 改用 `.body` 14px 加粗或 `.h1-sm` 36px。

### 6.2 em 标签使用

- **每屏主标 em 长度 ≤ 主标总字数 1/2**
- 反例：主标 14 字，em 占 7 字 → em 抢戏，**整个主标视觉重音混乱**
- 正确：主标 14 字，em 占 3-5 字 → em 是"重点"而非"主角"

## 七、写屏前 checklist（强制 7 步）

## 八、常见错误（写屏时禁止）

| 错误 | 反例 | 修法 |
|---|---|---|
| 用 clamp 上限算预算 | "display-xl 60px" | 用 §2 实际渲染 44px |
| 不断行让浏览器自动断 | `<p>35 岁之后，肌肤的悄悄话。</p>` | 强制 `<br>` 断行 |
| 第 2 行 ≤ 2 字 | "肌肤的 / 悄悄话" | 加 / 减字让第 2 行 ≥ 3 字 |
| 子标用 .lead 16 | 痛点子标 16px | 用 .h1-sm 36 或 .body 14 加粗 |
| 痛点块贴图 | 块底 0px 间距 | 加 mod-body padding-bottom 24-32 |
| em 占主标 1/2+ | "肌肤的<em>悄悄话</em>" 6 字全 em | em 限 3-5 字 |
| 屏 1 标题区 88px | "display-xxl 88px" | 实际渲染 55px（用 §2 表） |
