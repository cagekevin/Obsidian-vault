> 适用于各类图片生成任务（产品图、海报、社媒图、场景图等）。根据用户需求灵活调整比例、风格和图片数量。

## [Module A] 核心生成流水线

### Phase 1. 信息收集与分析

#### 1.1 接收用户需求
- 产品名称、类型、核心卖点
- 品牌调性、目标受众
- 特殊要求（平台、比例、风格等）

#### 1.2 产品视觉解码
- 使用 analyse_image 工具分析用户提供的产品图片
- 提取关键视觉特征：
  - 产品形态（形状、尺寸、材质）
  - 色彩信息（主色、辅色、点缀色）
  - 包装设计（容器、标签、盖子）
  - 独特细节（纹理、光泽、透明度）
- ⚠️ 分析完成后，后续生成提示词中不要额外描述产品外观，只说 "product from reference images" 即可——参考图已经包含了所有视觉信息，文字描述反而可能造成冲突

如果用户提供多张产品图：
- 询问用户哪张是主图（用于提取主要视觉特征）
- 其他图片作为辅助参考（用于了解产品的不同角度/状态）
- 在分析报告中分别说明：
  - 主图特征：[用于后续生成的主要参考]
  - 辅助图特征：[补充信息，如侧面、细节等]

#### 1.3 行业调研（条件触发）

何时需要：
- 用户未提供参考风格
- 产品属于新兴品类，缺少视觉参考
- 用户要求"参考行业最佳实践"

何时可跳过：
- 用户已提供完整的参考图和风格要求
- 产品属于成熟品类，AI 已有足够知识储备
- 用户明确要求"不要参考竞品"

执行方式：
- 使用 search_web 或 search_image 搜索同类产品、行业趋势
- 了解竞品视觉风格、常用设计元素
- 为色彩方案和视觉风格提供参考
- ⚠️ 聚焦用户最核心的 3-5 个关键词即可，搜索结果过多时不要全部堆砌到提示词中——信息过载反而让 AI 抓不住重点

如果跳过：
- 在 Phase 3.1 设计全局令牌时，完全基于用户提供的信息和产品图
- 在 Phase 5 向用户确认时，标注"未进行行业调研"

### Phase 2. 前置校验与预处理

校验前提：用户已提供产品图和完整文案。

#### 2.1 产品图质量检查
- 清晰度检查：是否模糊、过曝、欠曝
- 角度检查：是否有正面、侧面等必要角度
- ⚠️ 如不符合要求：提示用户重新提供（这是硬性要求，无法自动修复）

#### 2.2 文案智能分层

触发条件：单个卖点文案 >30 词

分层标准：

| 层级 | 长度 | 提取规则 | 关键词 |
|------|------|----------|--------|
| 主标题 (Primary Title) | 3-8 词 | 产品名 + 核心卖点 | 产品类型、核心功效、独特技术 |
| 次级标题 (Secondary Title) | 3-6 词 | 专利技术、核心成分 | patented, advanced, clinically proven |
| 正文 (Body Text) | 15-25 词 | 功效描述、适用人群 | 简洁、避免冗长 |

自动处理流程示例：

原始文案："Our advanced ceramide retinol capsules use patented time-release technology to deliver powerful anti-aging benefits while minimizing irritation, suitable for all skin types including sensitive skin."（40 词）

↓ AI 自动分层 ↓

| 层级 | 提取结果 |
|------|----------|
| 主标题 | "Advanced Ceramide Retinol Capsules" |
| 次级标题 | "Patented Time-Release Technology" |
| 正文 | "Delivers powerful anti-aging benefits while minimizing irritation. Suitable for all skin types including sensitive skin." |

向用户展示并等待确认：
```
检测到长文案（40 词），已自动分层：

主标题：Advanced Ceramide Retinol Capsules
次级标题：Patented Time-Release Technology
正文：Delivers powerful anti-aging benefits...

是否接受此分层？
[ 接受 ] [ 手动调整 ]
```

边界情况处理：
- 25-30 词：提示用户"文案略长，建议分层，是否需要？"
- 用户拒绝分层：记录"用户拒绝分层"，在 Phase 4.4 文字量冲突处理中，建议更换适合长文案的骨架（如 Benefit Explanation）
- 用户坚持不分层且不换骨架：
  - 在 Phase 6.5 校验时标注"文字量风险"
  - 生成时在提示词中强调"ensure all text is clearly visible and readable"

#### 2.3 骨架匹配度评估与动态扩展

触发条件：所有标准骨架匹配度 <60%

评分维度（每项 0-100 分）：

内容匹配度（权重 40%）
- 卖点类型与骨架用途是否对应？（0-100）
- 示例：功效说明卖点 vs Benefit Explanation 骨架 = 90 分

视觉需求匹配度（权重 30%）
- 卖点需要的视觉元素，骨架能否提供？（0-100）
- 示例：需要对比图，Before/After 骨架 = 95 分

文字量匹配度（权重 20%）
- 卖点文案长度与骨架文字容量是否匹配？（0-100）
- 示例：30 词文案 vs Hero Shot（适合 <10 词）= 40 分

风格匹配度（权重 10%）
- 卖点调性与骨架视觉风格是否一致？（0-100）
- 示例：科技感卖点 vs Technology/Patent 骨架 = 90 分

最终匹配度计算：

最终匹配度 = (内容匹配度 × 0.4) + (视觉需求匹配度 × 0.3) + (文字量匹配度 × 0.2) + (风格匹配度 × 0.1)

评级标准：

| 匹配度 | 评级 | 处理方式 |
|--------|------|----------|
| 90-100% | 完美匹配 | 直接使用 |
| 70-89% | 良好匹配 | 直接使用 |
| 50-69% | 可用但不理想 | 使用但标注"匹配度中等，可能需要迭代" |
| <50% | 不建议使用 | 触发"新骨架建议"流程 |

新骨架建议流程：

1. AI 分析卖点特征
   - 卖点类型（功效/场景/技术/认证等）
   - 核心信息（主要传达什么）
   - 视觉需求（需要什么视觉元素）
2. 生成新骨架定义
   - 骨架名称
   - 用途说明
   - 视觉元素
   - 文字结构
   - 参考骨架（基于哪个标准骨架改进）
3. 向用户展示建议方案
4. 用户确认后，新骨架加入"项目骨架库"（不污染标准库）

示例：

用户卖点："可以在飞机上使用，符合航空安全规定"

AI 分析：
- 卖点类型：便携性 + 合规性
- 核心信息：合规认证 + 使用场景
- 视觉需求：场景照片 + 认证标志

匹配度评估：
- Usage Scenario：内容 50 + 视觉 60 + 文字 80 + 风格 70 = 58%
- Certification/Award：内容 40 + 视觉 50 + 文字 70 + 风格 60 = 48%
- Size/Dimension：内容 30 + 视觉 40 + 文字 80 + 风格 50 = 42%

结论：所有骨架 <60%，触发新骨架建议

新骨架定义：
- 骨架名称：Travel-Friendly / Portability
- 用途说明：展示产品的便携性和旅行适用性
- 视觉元素：飞机/行李箱/尺寸对比/合规标志/TSA 认证
- 文字结构：主标题（便携特性）+ 正文（合规说明）
- 参考骨架：Usage Scenario + Certification/Award 的混合

#### 2.4 色彩预处理（从产品图提取色彩信息）
- 从产品图提取主色、辅色、点缀色
- 记录产品色彩特征，供 Phase 3 设计色彩方案时参考
- ⚠️ 此处仅提取色彩信息，不执行冲突检查（色彩冲突检查在 Phase 3 选定色彩方案后，由 Phase 4.2 执行）

### Phase 3. 方案设计

#### 3.1 设计全局令牌（Global Tokens）

色彩系统：
- 📸 从产品图提取基础色彩（用 analyse_image）
- 📊 搜索调研：用 search_web 或 search_image 搜索同类产品、行业趋势（如果 Phase 1.3 已执行，则复用结果）
- 🎨 基于产品色彩 + 调研结果，设计多套色彩方案（通常 2-3 套）供用户选择
- 每套方案包含：
  - 主色（标题）
  - 辅色（副标题）
  - 点缀色（装饰元素）
  - 数据高亮色
  - 正文色
  - 背景色

字体系统：
- 根据品牌调性选择字体风格（衬线/无衬线/手写/装饰）
- 定义字体层级（主标题/副标题/正文/标注）
- 确保字体与品牌调性一致

视觉风格：
- 根据产品类型和品牌调性确定整体视觉风格
- 常见风格：简约现代、奢华高端、自然有机、科技未来、复古怀旧、活力年轻等
- 定义光影氛围、材质质感、构图风格

特殊元素视觉风格（条件触发）

触发条件：当卖点包含以下类型元素时，需要设计专门的视觉风格：
- 成分/配方（如：神经酰胺、透明质酸、活性肽）
- 材质/质地（如：凝胶、乳霜、粉末）
- 技术原理（如：缓释技术、微囊包裹、离子导入）
- 抽象概念（如：保护屏障、深层渗透、持久锁水）

识别方法：
- 检查卖点文案中是否包含专业术语/科学名词
- 判断该元素是否难以用常规摄影/3D 渲染直观表达
- 如果是，则触发特殊元素视觉风格设计流程

处理流程：
1. 使用 search_image 搜索该元素的常见视觉表达方式
2. 基于搜索结果，设计 2-3 套风格方案
3. 每套方案包含：
   - 风格名称
   - 视觉元素说明
   - 提示词关键词
   - 适合场景
4. 向用户展示并等待选择

示例：

卖点："含有神经酰胺成分"

识别：专业术语 ✓ / 难以直观表达 ✓ → 触发流程

搜索结果分析：常见表达方式有分子结构图、微观粒子、保护屏障示意图

设计方案：

方案 A：科学可视化
- 视觉元素：分子结构、微观粒子、科技感光效
- 关键词：molecular structure, scientific visualization, glowing particles
- 适合场景：强调科技感、专业性的产品

方案 B：功效隐喻
- 视觉元素：保护屏障、水分锁定、皮肤层次
- 关键词：protective barrier, moisture lock, skin layers
- 适合场景：强调功效、易于理解的产品

方案 C：自然提取
- 视觉元素：天然原料、提取过程、纯净质感
- 关键词：natural ingredients, extraction, pure essence
- 适合场景：强调天然、有机的产品

#### 3.2 配图逻辑设计（核心创新）

> 详见 [[利益视觉化原则]]，本节为 skill 执行流程中的简要说明。

⚠️ 配图的本质 = 利益视觉化

核心原则：
- 图片要展示的是消费者获得的利益，而不是获得利益的过程或方法
- 利益是主角，产品是配角，渠道/方法是装饰

视觉权重层级：
```
主角（利益元素）：视觉焦点，最突出，占据画面中心或前景
    ↓
配角（产品）：支持性元素，清晰可见但不抢镜，位于中景或侧面
    ↓
装饰（场景/渠道）：背景或边缘，弱化处理，暗示而非强调
```

⚠️ 重要说明：
- 以上是"视觉权重"关系，而非精确的"画面占比"
- AI 工具无法精确控制各元素占画面的百分比
- 应通过提示词中的描述详细程度和空间位置词来控制视觉权重

如何在提示词中体现视觉权重：

| 视觉权重 | 提示词策略 | 关键词 | 字数占比 |
|----------|------------|--------|----------|
| 主角（利益元素） | 详细描述 + 突出位置 | huge, eye-catching, prominently displayed, in the foreground, as the focal point | 40-50% |
| 配角（产品） | 简洁描述 + 支持位置 | supporting element, in the mid-ground, positioned beside/behind, clearly visible but secondary | 25-35% |
| 装饰（场景/渠道） | 极简描述 + 背景位置 | subtle, in the background, softly blurred, as a hint, small icon/badge | 5-15% |
| 氛围（光线/色彩） | 简短描述 | soft lighting, warm tones, gradient background | 5-10% |

示例对比：

❌ 错误示例（使用精确比例，AI 无法执行）：
```
Visual: 20% OFF tag occupies 45% of the canvas, product bottle occupies 30%, 
background occupies 25%
```

✅ 正确示例（使用视觉权重描述）：
```
Visual: A huge, eye-catching 20% OFF discount tag in deep red color, prominently 
displayed in the foreground as the focal point. The product bottle from reference 
image is positioned as a supporting element in the mid-ground, clearly visible but 
secondary to the discount tag. Soft peach gradient background with subtle glow.
```

视觉权重体现在：
- 主角（折扣标签）：huge, eye-catching, prominently, foreground, focal point（5 个强调词）
- 配角（产品）：supporting element, mid-ground, secondary（3 个定位词）
- 装饰（背景）：soft, subtle（2 个弱化词）

配图思维链（必须暴露）：

何时生成：
- Phase 3.3 为每个卖点设计配图方案时
- 在选择设计骨架之前生成（思维链帮助选择合适的骨架）

何时展示给用户：
- Phase 5 方案确认时，作为"每个卖点的配图方案"的一部分展示
- 用户可以看到 AI 的推理过程，提出修改意见

何时用于内部检查：
- Phase 6.5 出图前校验（方向性校验）：快速检查配图逻辑是否正确
- Phase 7.2 出图前校验（细节校验）：全面检查配图逻辑是否正确
- Phase 8.4 配图逻辑修正：重新生成思维链，标注问题所在

展示格式：
- 向用户展示时：使用完整的思维链模板（包含 4 个部分）
- 内部检查时：只检查关键点（主角是否是利益元素、产品是否过度抢镜等）

配图思维链模板：

每个卖点配图前，必须展示以下思维链：

```
【配图思维链】
1. 卖点分析：
   - 原始卖点：[用户提供的文案]
   - 核心利益：[消费者得到什么]
   - 获取方式：[消费者需要做什么]

2. 视觉角色分配：
   - 主角（利益元素）：[具体视觉元素，如：巨大的 20% 折扣标签]
   - 配角（产品）：[产品在画面中的角色]
   - 装饰（渠道/方法）：[如何暗示获取方式，如：小型账户图标]

3. 构图逻辑：
   - 视觉焦点：[主角占据画面中心/前景]
   - 空间层次：[主角在前景 > 配角在中景 > 装饰在背景]
   - 引导路径：[视觉如何引导用户从利益 → 产品 → 方法]

4. 检查清单：
   ✓ 利益元素是否是视觉焦点？
   ✓ 产品是否清晰可见但不抢镜？
   ✓ 渠道提示是否弱化处理？
   ✓ 画面是否直观易懂？
```

配图具体化原则：
- ❌ 抽象概念（如"注册"、"上传"、"独家"）
- ✅ 具体利益（如"20% OFF 折扣标签"、"金币和现金"、"FREE 标签"）

常见错误示例：

| 卖点 | ❌ 错误配图逻辑 | ✅ 正确配图逻辑 |
|------|----------------|----------------|
| 注册送 20% 折扣 | 主角：注册界面<br>配角：产品 | 主角：巨大的 20% 折扣标签<br>配角：产品<br>装饰：小型账户图标 |
| 上传收据得现金 | 主角：手机上传界面<br>配角：收据 | 主角：大量金币和现金<br>配角：产品<br>装饰：小型 WhatsApp 图标 |

### Phase 4. 冲突检测与自动处理

#### 4.1 骨架冲突检测

触发条件：多个卖点选择了相同骨架

处理方案：
1. 分析每个卖点与该骨架的匹配度
2. 保留匹配度最高的卖点使用该骨架
3. 为其他卖点重新选择次优骨架
4. 向用户展示调整方案并说明原因

#### 4.2 色彩冲突检测与自动调整

检测标准：
- 对比度不足：文字与背景对比度 <4.5:1
- 色相冲突：产品色与背景色色相差 <30°（在色环上过于接近）
- 情绪不一致：色彩情绪与品牌调性不符

自动调整方案：

场景 1：对比度不足
- 检测：文字色与背景色对比度 <4.5:1
- 调整：
  - 方案 A：加深文字颜色（降低明度 20-30%）
  - 方案 B：调整背景颜色（提高或降低明度，确保对比度 ≥4.5:1）
  - 方案 C：为文字添加描边或阴影

场景 2：产品色与背景色过于接近
- 检测：产品主色与背景色色相差 <30°
- 调整：
  - 方案 A：调整背景色色相（偏移 ±60°）
  - 方案 B：调整背景色饱和度（降低 30-50%，使其更中性）
  - 方案 C：使用渐变背景（产品区域用对比色，边缘用协调色）

场景 3：色彩情绪不一致
- 检测：色彩情绪与品牌调性矛盾（如：奢华品牌用荧光色）
- 调整：
  - 重新设计色彩方案，确保与品牌调性一致
  - 向用户展示调整前后对比并说明原因

场景 4：无法自动调整
- 检测：产品色与背景色色相差 <30°，且产品色不可更改，背景色调整后对比度仍不足
- 处理方案：
  - 向用户说明冲突原因
  - 提供 3 种解决方案供选择：
    - 方案 A：使用渐变背景（产品区域用对比色，其他区域用协调色）
    - 方案 B：增加视觉辅助（产品周围加白色描边/阴影/光晕）
    - 方案 C：调整产品位置（放在背景中对比度较高的区域）
  - 等待用户选择后继续

#### 4.3 风格冲突检测

触发条件：多个卖点的视觉风格不统一

处理方案：
1. 识别主导风格（出现频率最高或用户明确偏好的风格）
2. 将其他卖点的风格调整为主导风格
3. 向用户展示调整方案

#### 4.4 文字量冲突检测

触发条件：
- 文案长度超过所选骨架的文字容量
- 且该问题在 Phase 2.2 未被解决（用户拒绝分层或分层后仍超出容量）

处理方案：
1. 检查 Phase 2.2 的处理记录：
   - 如果用户已接受分层且文案符合骨架容量 → 跳过此步骤
   - 如果用户拒绝分层 → 执行下面的流程
   - 如果分层后仍超出容量 → 执行下面的流程
2. 向用户说明问题："当前骨架 [骨架名称] 适合 [X-Y] 词的文案，但您的文案有 [Z] 词"
3. 提供解决方案：
   - 优先方案：更换为适合长文案的骨架（如 Benefit Explanation）
   - 备选方案：精简文案或进一步分层
   - 最后方案：保持原骨架，但在提示词中强调"ensure all text is clearly visible and readable"，并在 Phase 6.5 校验时标注"文字量风险"
4. 等待用户选择后继续

提示词关键用词规范：

|------|--------|
| 强调全局一致性 | ALL, consistent across all frames, unified |
| 描述动态过程 | PARTIALLY, in the process of, transitioning from...to |
| 颜色协调 | matches the background palette, harmonize with |
| 风格统一 | Pixar-style 3D rendering for ALL elements |

提示词比重分配（视觉描述中各元素的字数要和视觉权重成正比）：

⚠️ 配图逻辑决定比重分配：

标准比重（适用于大多数卖点）：
- 利益元素（主角）：40-50% 字数
  - 详细描述外观、颜色、位置
  - 使用强调词：huge, eye-catching, prominently, focal point
- 产品（配角）：25-35% 字数
  - 简洁描述类型和位置，不写细节
  - 使用定位词：supporting element, mid-ground, secondary
- 场景/背景：15-20% 字数
  - 极简描述场景类型
  - 使用弱化词：soft, subtle, gentle
- 光线/氛围：5-10% 字数
- 装饰元素（渠道提示）：5-10% 字数
  - 能不写就不写，避免喧宾夺主
  - 使用弱化词：small, subtle, as a hint

特殊情况：
- Hero Shot 骨架（产品为主角）：产品 50-60% 字数，其他元素相应减少
- Lifestyle Integration 骨架（场景为主角）：场景 40-50% 字数，产品 20-30% 字数

示例：

❌ 错误示例（产品描述过多，利益元素描述不足）：
```
Visual: A golden capsule bottle with elegant label and metallic cap, placed on 
a marble surface, with a small 20% discount tag in the corner.
```
- 产品描述：60% 字数
- 利益元素：10% 字数
- 场景：30% 字数

✅ 正确示例（利益元素为主角，视觉权重清晰）：
```
Visual: A huge, eye-catching 20% OFF discount tag in deep red color, prominently 
displayed in the foreground as the focal point. The product bottle from reference 
image is positioned as a supporting element in the mid-ground, clearly visible but 
secondary to the discount tag. Soft peach gradient background with subtle glow.
```
- 利益元素：50% 字数（huge, eye-catching, prominently, foreground, focal point）
- 产品：30% 字数（supporting element, mid-ground, secondary）
- 场景：20% 字数（soft, subtle）

⚠️ 关键理解：
- 字数比重 = 提示词中的描述详细程度（可控）
- 画面占比 = 最终生成图中的实际面积（不可控）
- 通过控制字数比重 + 空间位置词，间接影响视觉权重

     - 方案 B：更换设计骨架
     - 方案 C：调整配图思维链（重新定义主角/配角）

## [Module B] 资产库

### Skeleton_Library（标准骨架库）

按使用频率分类：

**高频骨架（80% 的卖点适用）：**

1. **Hero Shot（英雄主图）**
   - 用途：突出产品本身，展示产品的整体形象
   - 视觉元素：产品居中，简洁背景，柔和光线

   - 适合卖点：品牌形象、产品发布、主打产品
   - 文字容量：低（<10 词）

2. **Benefit Explanation（功效说明）**
   - 用途：详细说明产品的核心功效和优势
   - 视觉元素：产品 + 功效可视化（如图标、示意图）
   - 文字结构：主标题 + 功效列表/段落（15-30 词）
   - 适合卖点：功效说明、成分介绍、技术优势
   - 文字容量：高（15-30 词）

3. **Before/After Comparison（前后对比）**
   - 用途：展示使用产品前后的效果对比
   - 视觉元素：左右或上下分屏，清晰的对比标识
   - 文字结构：Before/After 标签 + 简短说明
   - 适合卖点：效果展示、问题解决、改善对比
