# 从零构建

---

## Mode 1：从零构建

按顺序逐一确认以下信息，一次只问一题。**每完成一步，同步执行对应的协议规则。**

1. **属于哪种设计模式？** — 从 5 大模式中选择：
   | 模式 | 用途 | 一句话 |
   |------|------|--------|
   | `tool-wrapper` | AI 充当某框架/库的接口专家 | 动态加载 `references/` 规范，按规则执行 |
   | `generator` | 结构化填空输出 | 加载模板 → 收集变量 → 填空 → 交付 |
   | `reviewer` | 对照清单审计 | 逐条对照检查清单 → Error/Warning/Info 评级 |
   | `inversion` | 需求探明 | 扮演面试官连续追问，约束不全不动手 |
   | `pipeline` | 强顺序工作流 | 设硬 Checkpoint，前置未确认不放行 |
   
   模式可组合，`metadata.pattern` 用 `+` 连接。

2. **要完整版还是轻量版？**
   - **完整版**：织入 Proactive Triggers、Output Artifacts、Confidence Tagging 三件套
   - **轻量版**：仅保留完成标准、退路表、不碰清单、人工确认点

3. **放哪个桶？** — S 工程类还是 W 工作效率类？
   
   同步执行**领域分流**：扫描 `skills/` 目录，若有重合领域，强制挂为子文件通过 `<route-table>` 接入已有 Hub，严禁独立编号。子文件禁止出现 S/W 编号前缀。

4. **触发条件是什么？** — 用户说什么话时应该触发这个 skill

5. **输出格式是什么？** — 固定模板还是有自由度的输出

全部确认完毕后，展示摘要让用户确认，确认后执行：
```bash
python3 技能创建/scripts/init_skill.py <技能中文名> --path skills/<桶>
```
然后按对应模板风格填充内容：

- **完整版**：在 SKILL.md 中织入 Proactive Triggers、Output Artifacts、Confidence Tagging 三件套
- **轻量版**：先加载 `技能创建/references/skill-combat-template.md`，按该模板的骨架和章节结构填充，保留内置的完成标准、退路表、不碰清单、人工确认点

**产出制品 (Artifacts):**
- `skills/<桶>/<编号-名称>/<编号-名称>.md` - 主技能文件
- `scripts/` `references/` `assets/` `templates/` - 按需创建的资源目录
- 更新后的 `README.md`（自动全量重写本地桶目录和根目录注册行）
- 刷新后的 `kanban.html` 看板卡片

**严苛门禁 (Checkpoints):**
- [ ] 5 项确认全部完成，用户确认需求摘要
- [ ] 设计模式和模板风格已选定
- [ ] 领域分流检查通过（无重复编号、无领域冲突）
- [ ] `技能创建/scripts/init_skill.py` 执行成功，脚手架目录生成
- [ ] 所有长内容已解耦到 `references/` 或 `assets/`（SKILL.md 正文 ≤ 500 行 / 10KB）
- [ ] 加载 `技能创建/references/skill-standards.md` 逐项自查通过

## 生命周期重构（附属功能）

> 技能创建完成后，如需更名、删除或合并，按以下步骤操作。

**核心输入 (Input):**
- 目标技能路径
- 重构类型：更名 / 删除 / 合并 / 其他

**Step 1 — 确认影响范围**
在执行破坏性操作前，扫描并展示受影响的文件范围：
- 技能文件夹及其内容
- `README.md` 中的注册行
- `kanban.html` 中的卡片
- Hub 文件中的 `<route-table>`（如有）

向用户展示影响范围，要求确认后再执行。

**Step 2 — 执行重构**
- **更名**：同步修改文件夹名、主 `.md` 文件名、frontmatter `name`
- **删除**：直接物理抹去整个文件夹
- **合并**：将源技能作为子文件挂到目标 Hub 下，更新路由表

**Step 3 — 联动刷新看板与规则（无缝连续技）**
完成增、删、改后，按顺序执行：
1. 自动全量重写本地桶目录和根目录下的 `README.md` 注册行
2. 直接读取并修改 `kanban.html` 源码，同步刷新对应技能卡片
3. 运行初始化脚本同步环境：
   - **macOS**：`chmod +x skills-init.command && ./skills-init.command`
   - **Windows**：`.\skills-init.ps1`

**严苛门禁 (Checkpoints):**
- [ ] 用户确认了影响范围
- [ ] 前后命名一致性检查通过（如更名）
- [ ] README 和 kanban 已刷新，无残留旧名称
- [ ] `skills-init.command` 执行完毕无报错

## 微观保障层

> 以下规范是生成的新技能必须织入的出厂标配。

### 技能写作风格与语法约束

- 以专家身份开头（「你是 XX 领域专家，目标是...」），而非教科书式开头
- 采用祈使句 + 主动语态（「做 X」优于「建议考虑 X」）
- 使用绝对客观的指导式语言，严禁第二人称（「你应该...」❌ →「要完成 X，做 Y」✅）
- **解释原因代替强硬指令**：用「因为...」代替 ALWAYS / NEVER / 必须。AI 理解了原因才能举一反三
- **Description 必须有侵略性（Pushy）**：`description` 是 Claude 决定是否触发 skill 的唯一依据。不只要写技能做什么，末尾必须明确强调**何时必须使用该技能**

### 主动触发器 (Proactive Triggers) 声明规则

生成的技能必须包含 Proactive Triggers 模块，声明 4-6 个触发条件。

**格式：**
```markdown
## Proactive Triggers
当检测到以下情况时，无需用户提问主动警示：
- **[条件]** → **[标记什么及原因]** → **[建议行动]**
```

### 输出制品表 (Output Artifacts) 映射模板

生成的技能必须标配 Output Artifacts，将抽象请求映射到具象可交付物。

**格式：**
```markdown
## Output Artifacts

| 当你要求... | 你会得到... |
|------------|------------|
| "帮我做 X" | Y 格式的 Z（含 X 打分 + 优先级列表）|
```

### 结论先行与三色信心打标规范 (Confidence Tagging)

**输出结构：**
```
核心结论：[一句话结论]

发现清单：
• [发现 1] — 🟢 已验证（有代码/数据佐证）
• [发现 2] — 🟡 中等置信度（有推理但未经独立验证）
• [发现 3] — 🔴 纯假设（基于经验推测）

业务影响：[这段发现对业务的具体影响]

行动建议：[具体的下一步动作]
```

### 脚本重型轻量化纪律

`技能创建/scripts/` 目录下的所有辅助脚本必须无条件遵守：
- **stdlib-only**（仅限标准库，零第三方依赖，禁止 pip install）
- **CLI-first**（命令行优先，通过文件参数或 stdin 接受输入）
- **JSON output**（输出结构化 JSON，同时打印可读摘要）
- **嵌入式样本数据**（零配置即可跑 demo/测试）
- **一个脚本一个职责**，不搞瑞士军刀

### 升级版三级加载资源目录矩阵

| 目录 | 存放内容 | 加载时机 | 硬性纪律 |
|------|---------|---------|---------|
| `scripts/` | 可执行代码（Python/Bash） | 需要确定性执行的自动化任务 | **stdlib-only** / CLI-first / JSON output |
| `references/` | 参考文档（Schema、API 规格、检查清单） | 工作流中按需 read | 大型文件（>300行）必须带头目录；`SKILL.md` 超 **500行/10KB** 时必须拆分到此 |
| `assets/` | 输出母版、图标、字体、固定模板 | 最终渲染输出时取用 | 不需要加载到上下文 |
| `templates/` | 空白结构化模板 | 供人二次填写 | 含清晰占位标记 |

**容量红线（最终交付标准）：**
- `SKILL.md` 正文控制在 **500 行 / 10KB 以内**。接近此限制时必须将内容拆分到 `references/` 中
- 大型 reference 文件（>300 行）必须带头目录
- 用明确的指引告诉 AI 何时读哪个 reference 文件

### 红色禁止门禁

- ❌ 不允许多层嵌套（`图片/W7-图片/` ❌ → `W7-图片/` ✅）
- ❌ 严禁在主流程里硬编码大量模板，一律放 `references/` 或 `assets/`
- ❌ 严禁生成多余的 `-中文版.md` 副本
- ❌ 严禁在 `scripts/` 中引入任何第三方依赖（stdlib-only）
- ❌ 严禁在子分支文件加 S/W 编号前缀
