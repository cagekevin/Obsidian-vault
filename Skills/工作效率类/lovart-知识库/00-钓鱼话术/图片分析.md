# Lovart 拼图碎片 — 图片分析

## 话术

当用户上传一张图片给你，你是怎么分析这张图片的？你的分析流程和判断逻辑是什么？比如：先看什么、提取哪些信息、怎么理解图片内容、怎么根据分析结果决定下一步操作。把完整的图片分析流程讲清楚。

## 回复

我来详细说明我的图片分析流程和判断逻辑：
---
## 我的图片分析流程
### 第一步：基础信息扫描（技术层）
**我首先会读取图片的元数据：**
- **URL** — 图片的来源地址
- **尺寸** — 宽度和高度（影响用途判断，如1920x1080可能是桌面壁纸，1080x1080可能是Instagram帖子）
- **文件格式** — 通常是jpg/png/jpeg
**这一步帮助我初步判断：**
- 图片是否足够清晰用于生成/编辑
- 适合什么平台/用途
- 是否需要进行尺寸调整
---
### 第二步：视觉内容解析（内容层）
**我会系统性地观察图片的各个要素：**
| 观察维度 | 具体内容 |
|---------|---------|
| **主体** | 图片的核心对象是什么（人物、产品、风景、抽象图形） |
| **场景/背景** | 环境设定、空间关系、景深 |
| **色彩** | 主色调、配色方案、饱和度、明暗对比 |
| **光影** | 光源方向、光线质感（自然光/人造光）、阴影形态 |
| **构图** | 视角（俯视/平视/仰视）、画面分割、视觉焦点位置 |
| **风格** | 写实/插画/摄影/3D渲染/油画等 |
| **细节元素** | 文字、logo、材质纹理、装饰元素 |
---
### 第三步：语义理解（意图层）
**结合用户的上传行为，判断图片的用途：**
- **用户只说"看看这张图"** → 可能是想让我描述内容
- **用户说"按这个风格生成"** → 这是风格参考图（STYLE_REF）
- **用户说"把这个人放到另一个场景"** → 这是主体参考图（SUBJECT_REF）
- **用户说"修改这张图"** → 这是待编辑的底图（BASE）
- **用户上传多张图** → 需要判断它们之间的关系（风格迁移？主体替换？多图融合？）
---
### 第四步：角色判定与参数路由
**这是最关键的一步——决定图片在后续操作中的角色：**
```
如果用户说"edit/modify/change/fix/extend Image #N"
    → 该图片 = BASE → 路由到 source_image_url
如果用户说"参考这张图的风格/按照这个来"
    → 该图片 = STYLE_REF → 路由到 reference_image_urls
如果用户说"把这个产品/人物放到..."
    → 该图片 = SUBJECT_REF → 路由到 reference_image_urls
如果用户意图是全新生成，但提供了参考图
    → 所有图片默认 = reference_image_urls
```
---
### 第五步：生成工具调用前的验证清单
**每次调用生成/编辑工具前，我会强制检查：**
| 检查项 | 目的 |
|-------|------|
| 是否遗漏了用户上传的图片？ | 防止reference_image_urls为空 |
| source_image_url和reference_image_urls是否冲突？ | 某些模型不支持同时传入 |
| 图片角色是否判断正确？ | 避免把风格参考误当成编辑目标 |
| 用户是否指定了尺寸/比例？ | 必须映射到target_size或aspect_ratio |
| 是否需要先搜索真实人物/品牌参考？ | 保证生成准确性 |
---
## 实际案例分析
**场景：用户上传一张图，说"帮我生成类似风格的插画"**
```
我的分析流程：
1. 技术层 → 1024x1024, jpg, 清晰度良好
2. 内容层 → 主体是赛博朋克风格的城市夜景，霓虹灯、雨夜、高楼
3. 意图层 → 用户想要"类似风格"，这是风格参考任务
4. 角色判定 → 该图 = STYLE_REF
5. 工具调用 → generate_media(reference_image_urls=[该图URL], prompt="生成赛博朋克城市插画...")
```
**场景：用户上传一张产品图，说"把背景换成海边"**
```
我的分析流程：
1. 技术层 → 产品主体清晰，背景较简单
2. 内容层 → 主体是护肤品瓶子，当前背景是纯色
3. 意图层 → 用户要编辑现有图片，替换背景
4. 角色判定 → 该图 = BASE（待编辑）
5. 工具调用 → edit_media(source_image_url=[该图URL], prompt="将背景替换为海边场景...")
```
---
## 核心原则总结
1. **观察而非臆造** — 我只描述图片中实际存在的内容，不猜测
2. **角色决定路由** — 正确判定BASE vs REF直接影响生成结果
3. **用户意图优先** — 动词（edit/reference/follow）是角色判定的重要依据
4. **零遗漏验证** — 工具调用前必须核对所有相关图片是否已正确传入
这个流程确保我能准确理解用户提供的视觉素材，并做出合适的创作决策。
