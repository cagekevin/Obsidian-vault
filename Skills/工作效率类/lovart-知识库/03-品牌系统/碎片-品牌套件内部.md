## 品牌 Kit 机制完整演示

### 1. 品牌 Kit 的数据结构

```
Brand Kit: "HKH"
├── [LOGO]         - 品牌标志（主Logo、副Logo、图标版本）
├── [COLOR]        - 品牌色彩系统（主色、辅色、渐变色值）
├── [FONT]         - 品牌字体规范（标题字体、正文字体）
├── [CHARACTER]    - 品牌IP/吉祥物形象
├── [PRODUCT]      - 产品图片/包装样式
├── [STYLE]        - 风格参考图（摄影风格、插画风格）
└── [GUIDELINE]    - 品牌使用规范文档
```

### 2. Kit 的创建与存储

品牌 Kit 的创建通常通过以下方式：
- **后台管理系统**：品牌管理员上传资产并配置
- **API 接口**：程序化创建和更新
- **对话中声明**：用户在对话中指定 `--kit="HKH"` 时，系统会自动创建空 Kit 并关联

### 3. 引用 Kit 生成图片的完整流程

```
用户输入: "生成一张 HKH 品牌的社交媒体海报 --kit=HKH"

        ↓

系统调用 asset_factory(asset_id="HKH", asset_type="brand_kit")
        ↓

返回 Kit 内容:
{
  "brand_name": "HKH",
  "assets": {
    "logo": "https://.../hkh_logo.png",
    "colors": {
      "primary": "#1E3A8A",
      "secondary": "#F59E0B",
      "accent": "#10B981"
    },
    "fonts": {
      "heading": "Montserrat Bold",
      "body": "Open Sans"
    },
    "product_images": ["https://.../product1.jpg"],
    "style_references": ["https://.../style1.jpg"]
  }
}
        ↓

系统将 Kit 资产注入生成参数:
- reference_image_urls = [logo_url, product1_url, style1_url]
- prompt 追加: "使用 HKH 品牌蓝色(#1E3A8A)为主色调..."

        ↓

调用 generate_media() 生成符合品牌规范的图片
```

### 4. Kit 的优势

| 场景 | 不使用 Kit | 使用 Kit |
|------|-----------|----------|
| 品牌一致性 | 每次手动描述颜色和Logo位置 | 自动注入品牌规范 |
| 效率 | 重复输入品牌信息 | 一次配置，多次复用 |
| 协作 | 团队成员标准不统一 | 共享统一品牌资产 |
| 更新 | 需修改所有历史提示词 | 更新Kit即可影响所有后续生成 |
