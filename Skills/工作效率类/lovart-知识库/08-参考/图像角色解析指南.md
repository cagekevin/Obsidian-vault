我来详细解释这些图像角色概念，以及如何在实际工作中使用它们。
---
## 图像角色定义
| 角色 | 含义 | 使用场景 |
|------|------|----------|
| **BASE** | 被直接编辑/修改的图像 | 用户说"修改这张图"、"把背景换掉" |
| **STYLE_REF** | 风格参考（色彩、光影、质感） | 用户说"按照这张图的风格"、"参考这个色调" |
| **LAYOUT_REF** | 构图/布局参考 | 用户说"保持这个构图"、"参考这个排版" |
| **SUBJECT_REF** | 主体内容参考（人物/产品/物体） | 用户说"把这个产品放进去"、"用这个人物" |
---
## 实际案例演示
### 案例 1：单图编辑（BASE 单独使用）
**用户需求**："帮我把这张产品图的背景换成森林场景"
```
Image #1: 产品白底图
```
**角色判断**：
- Image #1 = **BASE**（用户明确要修改这张图）
**工具调用**：
- `source_image_url`: Image #1
- `prompt`: "将背景替换为森林场景，保持产品主体不变"
---
### 案例 2：风格迁移（BASE + STYLE_REF）
**用户需求**："把这张照片改成油画风格，参考梵高的星空"
```
Image #1: 用户上传的照片
Image #2: 梵高星空图（用户说"参考这个风格"）
```
**角色判断**：
- Image #1 = **BASE**（被修改的主体）
- Image #2 = **STYLE_REF**（提供风格参考）
**工具调用**：
- `source_image_url`: Image #1
- `reference_image_urls`: Image #2
- `prompt`: "将第一张照片转换成第二幅图的梵高油画风格"
---
### 案例 3：主体合成（BASE + SUBJECT_REF）
**用户需求**："把这张沙发放到这个客厅场景里"
```
Image #1: 单人沙发产品图
Image #2: 客厅场景图（用户说"放到这个场景"）
```
**角色判断**：
- Image #2 = **BASE**（作为场景的底图）
- Image #1 = **SUBJECT_REF**（要放入的主体）
**工具调用**：
- `source_image_url`: Image #2
- `reference_image_urls`: Image #1
- `prompt`: "将第一张图中的沙发放入第二张图的客厅场景中"
---
### 案例 4：多参考生成（无 BASE，纯参考）
**用户需求**："生成一张图，构图参考第一张，色彩参考第二张"
```
Image #1: 构图参考图
Image #2: 色彩参考图
```
**角色判断**：
- 无 BASE（用户要生成新图，不是编辑现有图）
- Image #1 = **LAYOUT_REF**
- Image #2 = **STYLE_REF**
**工具调用**：
- `reference_image_urls`: Image #1, Image #2
- `prompt`: "生成一张新图像，参考第一张图的构图布局，采用第二张图的色彩风格"
---
### 案例 5：复杂组合（BASE + STYLE_REF + SUBJECT_REF）
**用户需求**："把这张照片里的我换成穿西装的形象，风格参考这张商务海报"
```
Image #1: 用户照片
Image #2: 西装模特参考图
Image #3: 商务海报风格参考
```
**角色判断**：
- Image #1 = **BASE**（被修改的照片）
- Image #2 = **SUBJECT_REF**（西装形象参考）
- Image #3 = **STYLE_REF**（商务风格参考）
**工具调用**：
- `source_image_url`: Image #1
- `reference_image_urls`: Image #2, Image #3
- `prompt`: "修改第一张照片中的人物着装，参考第二张图的西装形象，整体风格参考第三张商务海报"
---
## 关键判断口诀
| 用户说... | 对应角色 |
|-----------|----------|
| "修改/编辑/换背景/去掉..." | **BASE** |
| "风格像/参考这个色调/质感..." | **STYLE_REF** |
| "构图参考/排版像..." | **LAYOUT_REF** |
| "用这个产品/人物/物体..." | **SUBJECT_REF** |
---
## 参数映射规则
| 角色 | 对应参数 | 说明 |
|------|----------|------|
| BASE | `source_image_url` | 只能有一个 |
| STYLE_REF / LAYOUT_REF / SUBJECT_REF | `reference_image_urls` | 可以有多个，逗号分隔 |
