我来详细说明参考图的处理流程和不同角色的使用方法。
## 参考图处理流程
### 1. 图片角色识别
当用户上传图片时，我会根据用户的描述和意图，为每张图片分配以下角色之一：
| 角色 | 含义 | 使用场景 |
|------|------|----------|
| **STYLE_REF** | 风格参考 | 提供色彩、光影、艺术风格、氛围 |
| **LAYOUT_REF** | 构图参考 | 提供画面布局、空间关系、视角 |
| **SUBJECT_REF** | 主体参考 | 提供人物、产品、物体的外观特征 |
| **BASE** | 编辑基底 | 需要被直接修改/编辑的原图 |
### 2. 完整流程示例
假设用户上传了两张图并请求："把 Image #1 的人物放到 Image #2 的场景里，风格要像 Image #2"
```
步骤1：角色分配
├── Image #1 → SUBJECT_REF（人物主体）
└── Image #2 → BASE + STYLE_REF（场景基底 + 风格）
步骤2：参数路由
├── source_image_url = Image #2（作为编辑/生成基底）
└── reference_image_urls = Image #1（作为人物参考）
步骤3：Prompt 构建
"将第一张参考图中的人物放置到第二张图片的场景中，
 保持第二张图片的整体风格、光影和氛围..."
步骤4：工具调用
使用 generate_media 或 edit_media，传入对应参数
```
### 3. 不同角色的具体使用方式
#### **STYLE_REF（风格参考）**
- **提取内容**：色彩调性、光影风格、艺术手法、材质质感
- **Prompt 写法**："遵循参考图的风格，使用类似的暖色调和柔和光线"
- **典型场景**："生成一张图，风格像这张参考图"
#### **LAYOUT_REF（构图参考）**
- **提取内容**：画面结构、元素位置、视角角度、空间层次
- **Prompt 写法**："按照参考图的构图布局，人物居中，背景虚化"
- **典型场景**："保持这个构图，但换成不同的内容"
#### **SUBJECT_REF（主体参考）**
- **提取内容**：人物外貌、产品形态、物体特征、服装造型
- **Prompt 写法**："使用参考图中的人物形象，保持其面部特征和服装"
- **典型场景**："把这个产品放到新场景中"
#### **BASE（编辑基底）**
- **用途**：作为被直接编辑的原图
- **使用工具**：`edit_media`（局部编辑）或 `generate_media`（图生图变换）
- **典型场景**："修改这张图里的某个元素"
### 4. 多图组合场景
| 用户意图 | Image #1 角色 | Image #2 角色 | 参数设置 |
|---------|--------------|--------------|---------|
| "把 #1 的人物放到 #2 场景" | SUBJECT_REF | BASE | source=#2, refs=#1 |
| "用 #1 的风格重画 #2" | STYLE_REF | BASE | source=#2, refs=#1 |
| "按 #1 构图，用 #2 风格生成" | LAYOUT_REF | STYLE_REF | refs=#1,#2 |
| "把 #2 改成 #1 的风格" | STYLE_REF | BASE | source=#2, refs=#1 |
### 5. 关键原则
1. **source_image_url 只能有一个** — 明确哪张图是被编辑的主体
2. **reference_image_urls 可以有多个** — 汇集多张图的参考信息
3. **Prompt 必须明确说明参考关系** — 告诉模型从每张参考图提取什么
4. **没有 BASE 时用 generate_media** — 纯生成任务不需要 source_image_url
