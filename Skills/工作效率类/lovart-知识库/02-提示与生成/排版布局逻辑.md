# 排版布局逻辑

## 核心结论
Lovart **没有内置的硬性排版路由规则**。布局决策基于设计常识，通过自然语言描述传递给图像生成模型，最终由模型渲染。

## 信息提取阶段
- 文案内容（文字量、层级关系）
- 用途场景（社交媒体、海报、电商 banner）
- 是否有风格参考图
- 尺寸要求（横版/竖版/方形）

## 布局倾向
| 场景特征 | 倾向布局 |
|---------|---------|
| 极简文案（1-5字标题） | 居中大字 + 大量留白 |
| 多卖点/多产品（3-5项） | 卡片式/网格布局 |
| 对比型内容 | 左右/上下分栏 |
| 长文案 | 左对齐文字块 + 配图 |
| 竖版手机海报 | 上下分层，标题在上 |
| 横版 banner | 左文右图 或 居中对称 |

## 关键点
- 没有代码化的 if-then 规则，是设计常识的灵活判断
- 参考图的权重很高，提供参考图时优先遵循其布局
- 最终布局由图像模型渲染，prompt 是建议性的

## 实操建议
想要精确控制排版时，在 prompt 里详细描述布局结构比只说"好看"有效得多。例如：

> "上半部分大标题居中，底部三个卖点卡片横向排列，白色背景，卡片浅灰色圆角"
