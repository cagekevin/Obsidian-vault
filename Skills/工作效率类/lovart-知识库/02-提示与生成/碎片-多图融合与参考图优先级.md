# Lovart 拼图碎片 — 多图融合与参考图优先级

## 话术

当我上传多张参考图时，Lovart 内部是怎么处理这些图的？风格参考、角色参考、场景参考各起什么作用？如果多张参考图的信息冲突了，优先级怎么决定？融合逻辑是什么样的？

## 回复

我来解释一下 Lovart 处理多张参考图的内部机制：
## 参考图的角色分类
当你上传多张图片时，系统会根据你的描述自动识别每张图的作用：
| 角色 | 作用 | 典型使用场景 |
|------|------|-------------|
| **风格参考 (STYLE_REF)** | 提供色彩、质感、艺术风格 | "按照这张图的风格画" |
| **角色参考 (SUBJECT_REF)** | 提供人物/产品的外观特征 | "让这个角色做某个动作" |
| **布局参考 (LAYOUT_REF)** | 提供构图、空间安排 | "参考这张图的构图" |
| **基础图 (BASE)** | 被直接编辑/修改的原图 | "修改这张图" |
## 融合逻辑与优先级
**1. 显式指令优先**
- 如果你明确说"按 Image #1 的风格，用 Image #2 的角色"，系统会严格遵循
**2. 冲突时的默认优先级**
```
BASE（被编辑的图）> 角色参考 > 风格参考 > 布局参考
```
**3. 融合方式**
- **风格+角色融合**：提取角色的外观特征，套用风格的色彩/质感
- **多风格融合**：系统会尝试找到风格间的平衡点，或按描述权重分配
- **角色一致性**：多张角色图会合并特征（如服装来自图A，发型来自图B）
## 实际建议
| 场景 | 推荐做法 |
|------|---------|
| 想要特定风格 | 明确说"参考这张图的风格" |
| 想要特定角色 | 说"使用这个角色"并提供清晰角色图 |
| 风格+角色都要 | 分别说明每张图的作用 |
| 避免冲突 | 尽量让参考图在关键特征上一致 |
