## 5 种视频生成模式

| 模式 | 输入 | 输出 |
|------|------|------|
| A：文生视频 | 文本 prompt | 视频 |
| B：首帧驱动 | 文本 + 起始帧图 | 视频 |
| C：首尾帧驱动 | 文本 + 起始帧 + 结束帧图 | 视频 |
| D：多图转视频 | 文本 + 多张参考图 | 视频 |
| E：参考视频 | 文本 + 参考视频 | 视频 |

## 模型路由决策树

```
用户有指定模型？→ 是 → 检查模型是否支持该模式 → 是 → 使用
                                    → 否 → 推荐支持该模式的模型
用户无指定 → 特征检测：
  ├─ 多张参考图 → Kling Omni
  ├─ 多视频参考 → Kling Omni
  ├─ 有首尾帧 → Kling V3
  ├─ 要求 4K → Kling V3（4K模式）
  └─ 要求快速 → Kling 2.5 Turbo
  └─ 无特殊 → generate_video_kling_v3 (mode=pro)
```

## 视频 vs 图片关键差异

| 环节 | 图片 | 视频 |
|------|------|------|
| 同步/异步 | 同步 | 异步（必须轮询） |
| 生成时间 | 秒级 | 分钟级 |
| 模型选择 | 相对简单 | 复杂（模式-模型兼容） |
| 后处理 | 简单上传 | 编码/音频/缩略图流水线 |
| 失败成本 | 低 | 高 |
| 资源调度 | 单 GPU | 分布式集群 |

## 异步轮询流程

```
提交 → job_id → 每 2 秒 GET /status/{job_id}
  → {status: "processing", progress: 45%}
  → 完成 → video_url + thumbnail_url + duration
```
