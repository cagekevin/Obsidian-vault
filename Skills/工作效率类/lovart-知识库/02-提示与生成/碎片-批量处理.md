# Lovart 拼图碎片 — 批量处理

## 话术

当用户需要生成多张图或多段视频时，你的批量处理逻辑是什么？你是怎么规划顺序、管理上下文、确保风格一致的？多图生成和单图生成在流程上有什么不同？

## 回复

我来详细说明多图/多视频生成的批量处理逻辑：
---
## 批量处理核心原则
### 1. 顺序规划策略
| 场景 | 处理顺序 | 原因 |
|:---|:---|:---|
| **系列海报/同一主题多图** | 先生成1张主视觉 → 以此为参考生成其余 | 建立风格基准，确保一致性 |
| **多角度产品图** | 先生成标准视角 → 再生成特写/场景图 | 保持产品特征统一 |
| **故事板/分镜** | 按叙事顺序生成 | 便于检查叙事连贯性 |
| **A/B测试变体** | 先生成基准版 → 再生成变体 | 控制变量，明确差异点 |
### 2. 上下文管理
```
第一轮生成：
- 使用用户提供的所有参考图（品牌kit、风格参考、产品图）
- 生成第1张锚点图
第二轮及以后：
- 保留原始参考图
- 新增：上一轮生成的图作为 [STYLE_REF] 或 [CONSISTENCY_REF]
- prompt中明确说明"保持与参考图一致的..."
```
### 3. 风格一致性保障
- **视觉锚点**：第一张生成的图作为后续所有图的基准
- **Prompt复用**：核心描述词、风格关键词保持不变
- **参考图叠加**：将已生成的图传入 `reference_image_urls`
- **关键属性锁定**：色彩方案、光影方向、材质表现等明确写入prompt
---
## 单图 vs 多图流程对比
| 维度 | 单图生成 | 多图生成 |
|:---|:---|:---|
| **参考图使用** | 直接使用用户提供的参考 | 首轮用用户参考，后续叠加已生成图 |
| **Prompt策略** | 一次性完整描述 | 核心描述固定，场景/角度变化 |
| **生成调用** | 单次调用 | 循环调用，传递上下文 |
| **一致性控制** | 依赖单次生成质量 | 依赖锚点图+参考链传递 |
| **失败处理** | 重新生成单张 | 锚点失败需重走首轮，后续失败可单独重试 |
| **用户确认点** | 生成后直接交付 | 锚点图需确认后再继续 |

## 实际执行示例
假设用户要生成"咖啡品牌4张场景图"：
```
Step 1: 生成锚点图（厨房场景）
  - 参考：品牌kit + 风格参考图
  - 输出：Image_A
Step 2: 生成咖啡馆场景
  - 参考：品牌kit + 风格参考图 + Image_A（作为风格一致性参考）
  - prompt: "...保持与参考图一致的光影风格和色调..."
Step 3: 生成户外场景
  - 参考：品牌kit + 风格参考图 + Image_A
Step 4: 生成办公场景
  - 参考：品牌kit + 风格参考图 + Image_A
```
