## 目录角色

| 目录 | 角色 | 谁写入 | 导出到 NotebookLM |
|------|------|--------|-------------------|

## 工具速查

```bash
# 自动钓鱼管线：从 fish_tasks.json 读任务 → 套话 → 存 00-钓鱼话术/
python scripts/lovart.py

# 单次提问：自动分类/指定分类 → 写入对应目录
python scripts/ask-lovart.py "小红书封面图设计规范"
python scripts/ask-lovart.py "亚马逊A+模板" --category "06-业务落地"
python scripts/ask-lovart.py --batch tasks.json       # 批量处理
python scripts/ask-lovart.py --list                    # 查看待处理任务

# 合并导出 → 桌面全量文件 → 拖入 NotebookLM
python scripts/export-for-notebook.py

# 刷新本 README 的目录索引
python scripts/generate-index.py
python scripts/generate-index.py --print   # 仅打印，不改文件

# 检查 00-钓鱼话术/ 的晋升状态
python scripts/promote.py --check
python scripts/promote.py --check -v       # 详细输出（含已晋升的）

# 晋升单个 raw → 分类目录
python scripts/promote.py --from "设计原则与审美规则" --to "02-提示与生成"

# 一键晋升所有未晋升的
python scripts/promote.py --batch

# 晋升后自动清理原始文件（确认无误再用）
python scripts/promote.py --batch --delete

# 一键全流程：钓鱼 → 晋升 → 重建索引
python scripts/run-pipeline.py
python scripts/run-pipeline.py --no-delete   # 不删原始文件
```

## KBRetriever 兼容

每个目录均包含 `data_structure.md`（导航索引），供 AI agent 通过分层索引渐进检索：

```
lovart-知识库/
├── data_structure.md                    ← 顶级索引
├── 02-提示与生成/
│   ├── data_structure.md               ← 本级索引
│   └── 图片生成完整流程.md               ← 实际内容
└── ...
```

Agent 搜索流程：读根 `data_structure.md` → 选相关子目录 → 读子 `data_structure.md` → grep + 窗口读取 → 迭代收敛答案。

`data_structure.md` 不参与 NotebookLM 导出，仅用于 agent 导航。
