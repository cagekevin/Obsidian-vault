我来详细讲解 `generate_slides`（PPT生成）工具的内部工作逻辑。
---
## 一、什么情况下调用 `generate_slides`？
**必须同时满足以下条件：**
| 条件 | 说明 |
|------|------|
| 用户**明确提到**幻灯片/PPT/演示文稿 | 如"做一个PPT"、"生成幻灯片"、"presentation" |
| 内容是**多页结构** | 需要分页展示不同主题或章节 |
| 不是单张图片/信息图/流程图 | 这些应走 `generate_media` 或 `edit_media` |
**不调用的情况：**
- 用户要"一张海报" → 走 `generate_media`
- 用户要"流程图/思维导图" → 走 `generate_media`
- 用户只说"网页/landing page"但没提HTML → 走 `generate_media`
---
## 二、必需参数
| 参数 | 类型 | 说明 |
|------|------|------|
| `current_task_context` | string | **完整用户输入**，包括主题、要点、子要点、期望页数 |
| `num_of_slides` | integer | 幻灯片页数，根据用户需求确定 |
| `slides_language` | string | 内容语言，如 "Chinese"、"English" |
**可选参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| `context_file_url_list` | string[] | 参考文档URL（.txt/.pdf/.md/.html），工具会分析并提取内容 |
| `content_asset_image_url_list` | string[] | 内容素材图（如产品图、团队照片、Logo），LLM决定放在哪页 |
| `style_reference_image_url_list` | string[] | 风格参考图（配色、纹理、艺术风格），直接作为样式参考 |
---
## 三、内部工作流程（从入参到输出）
```
┌─────────────────────────────────────────────────────────────────┐
│  STEP 1: 接收并解析输入                                          │
├─────────────────────────────────────────────────────────────────┤
│  • 提取主题/核心内容                                              │
│  • 识别关键点和子要点                                              │
│  • 确定页数 num_of_slides                                         │
│  • 如有 context_file_url_list，读取并分析文档内容                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 2: 内容规划与分页逻辑                                        │
├─────────────────────────────────────────────────────────────────┤
│  内部LLM将内容映射到每一页：                                        │
│                                                                 │
│  第1页: 封面 (Cover)                                              │
│        - 标题 + 副标题 + 视觉背景                                  │
│                                                                 │
│  第2~N-1页: 内容页 (Content Slides)                                │
│        - 每页1-2个核心要点                                         │
│        - 自动选择布局类型：                                         │
│          • 标题+正文 (Title + Body)                               │
│          • 标题+双栏 (Two Column)                                  │
│          • 标题+图片 (Title + Image)                               │
│          • 数据展示 (Data/Chart)                                   │
│          • 列表页 (Bullet Points)                                  │
│                                                                 │
│  第N页: 结尾页 (Closing)                                          │
│        - 总结/感谢/联系方式                                        │
│                                                                 │
│  如有 content_asset_image_url_list，在此步骤决定图片分配到哪页        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 3: 风格与视觉设计                                            │
├─────────────────────────────────────────────────────────────────┤
│  • 如有 style_reference_image_url_list，提取配色、字体、纹理风格      │
│  • 无风格参考时，根据主题自动选择合适风格：                          │
│    - 商务主题 → 简洁专业风                                          │
│    - 创意主题 → 活泼现代风                                          │
│    - 学术主题 → 清晰严谨风                                          │
│                                                                 │
│  确定每页的视觉元素：                                               │
│  - 背景色/渐变                                                     │
│  - 主色+辅色                                                       │
│  - 标题字体 + 正文字体                                              │
│  - 装饰元素（几何图形、线条、图标风格）                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 4: 逐页生成幻灯片图片                                        │
├─────────────────────────────────────────────────────────────────┤
│  对每一页调用图像生成：                                              │
│                                                                 │
│  生成参数包括：                                                    │
│  - 该页完整文本内容（标题+正文）                                    │
│  - 布局类型指令                                                    │
│  - 风格参数（颜色、字体、装饰）                                     │
│  - 分配的素材图URL（如有）                                          │
│                                                                 │
│  输出: 每页一张 1920×1080 的PNG图片                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 5: 打包输出                                                  │
├─────────────────────────────────────────────────────────────────┤
│  • 将所有页面图片打包为可下载格式                                    │
│  • 返回给用户：                                                    │
│    - 每页图片的URL列表                                             │
│    - 整体PPT预览/下载链接                                          │
│    - 每页内容摘要                                                  │
└─────────────────────────────────────────────────────────────────┘
```
---
## 四、排版决策逻辑（内部如何决定每页布局）
工具内部LLM根据**内容类型**自动选择布局：
| 内容特征 | 选择的布局 |
|----------|-----------|
| 只有一个大标题 + 简短副标题 | 封面布局（居中，大字体，背景图） |
| 一个主题 + 3-5个要点列表 | 标题+列表布局（左标题右列表，或上下结构） |
| 一个概念 + 需要配图说明 | 标题+图片布局（图文并排或上下） |
| 两个对比内容/优缺点 | 双栏对比布局 |
| 有时间线/步骤流程 | 时间轴/步骤布局 |
| 有数据/百分比 | 图表/数据可视化布局 |
| 结尾总结 | 简洁居中布局 |

## 五、完整示例调用
```python
generate_slides(
    current_task_context="""
    主题：人工智能在医疗领域的应用
    要点：
    1. AI医疗市场概况（市场规模、增长率）
    2. 主要应用场景（影像诊断、药物研发、个性化治疗）
    3. 成功案例（具体医院/公司案例）
    4. 面临的挑战（数据隐私、监管、伦理）
    5. 未来趋势展望
    期望页数：6页
    """,
    num_of_slides=6,
    slides_language="Chinese",
    context_file_url_list=["https://example.com/ai_medical_report.pdf"],
    content_asset_image_url_list=[
        "https://example.com/medical_ai_product.jpg",
        "https://example.com/company_logo.png"
    ],
    style_reference_image_url_list=[
        "https://example.com/blue_tech_style.jpg"
    ]
)
```

## 六、关键设计原则
1. **内容优先**：先确定每页说什么，再决定怎么排
2. **视觉一致性**：所有页面保持统一的色彩、字体、风格
3. **信息密度控制**：每页不超过3个核心要点，避免拥挤
4. **素材智能分配**：自动判断哪页适合放哪张图
5. **渐进式呈现**：封面→引入→展开→总结，符合叙事逻辑
这就是 `generate_slides` 工具的完整内部工作流程。
