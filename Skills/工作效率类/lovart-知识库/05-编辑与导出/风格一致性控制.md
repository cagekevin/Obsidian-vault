## 我的风格一致性策略
### 1. 核心原则：视觉锚点复用机制
**核心逻辑**：用已生成的图片作为后续生成的"视觉锚点"，形成风格传递链。
```
第一轮：文本 → 图片A（建立基准风格）
第二轮：图片A + 新提示 → 图片B（继承A的风格）
第三轮：图片A/B + 新提示 → 图片C（持续累积一致性）
```
### 2. 技术手段
#### **A. Reference Image 传递**
- 将已生成的图片 URL 传入 `reference_image_urls` 参数
- 在 prompt 中明确说明："保持与参考图一致的色调/光影/构图风格"
- 多图参考时，指定每张图的作用（如"参考图1的色调，参考图2的构图"）
#### **B. 角色一致性控制**
- 角色图作为固定参考传入所有相关生成
- 使用 `source_image_url` 进行角色姿态变换（保持长相不变，改变动作/场景）
#### **C. 风格关键词固化**
- 从第一轮提取有效的风格描述词
- 在后续 prompt 中原样复用（如"电影感打光、青橙色调、35mm胶片质感"）
### 3. 具体执行流程
| 步骤 | 操作 | 目的 |
|------|------|------|
| 1 | 生成第一张"风格锚点图" | 建立基准视觉语言 |
| 2 | 分析锚点图的关键特征 | 提取可复用的风格要素 |
| 3 | 将锚点图传入后续生成作为 reference | 强制风格继承 |
| 4 | 累积多张图后可多图参考 | 增强一致性约束 |
| 5 | 必要时使用 edit_media 进行局部调整 | 保持主体不变，修改环境 |
### 4. 实际案例演示
假设你要生成一个咖啡品牌的4张场景图：
```
图1（锚点）: 厨房大理石台面，晨光透过窗户，暖色调，柔和阴影
    ↓ reference
图2: 咖啡厅木桌，拿铁咖啡 → "保持图1的暖色调和光影质感"
    ↓ reference
图3: 户外草地野餐 → "参考图1-2的色调和光线风格"
    ↓ reference
图4: 办公室桌面 → "延续前三张图的视觉风格，统一品牌调性"
```
### 5. 限制与应对
| 挑战 | 解决方案 |
|------|----------|
| 模型对参考图的"理解"有偏差 | 在 prompt 中具体描述要继承什么（色调/光影/构图/材质） |
| 多张参考图权重不均 | 明确指定每张参考图的作用 |
| 复杂场景一致性难维持 | 分阶段生成：先生成角色，再合成场景 |
---
## 总结
我的风格控制逻辑是**"锚点建立 → 特征提取 → 参考传递 → 持续累积"**。通过将已生成的图片作为后续生成的参考输入，形成风格传递链，而不是每次都从零开始生成。
