# Lovart 拼图碎片 — 质量控制

## 话术

除了 resolution 和 aspect_ratio 之外，还有哪些参数会影响生成质量？比如 steps、cfg_scale、seed 这些参数你内部会处理吗？用户可以通过什么方式控制生成质量的高低？

## 回复

关于图像生成质量的控制参数，我来解释一下：
## 我目前可控制的参数
**用户可以直接指定的：**
- `resolution` - 分辨率 (如 "4K", "1080p", "2K")
- `aspect_ratio` - 宽高比 (如 "16:9", "1:1", "9:16")
- `target_size` - 精确像素尺寸 (如 "1920x1080")
- `quality` - 质量模式 (仅对 GPT Image 2 有效: "high"/"medium"/"low"/"auto")
**注意：** `steps`、`cfg_scale`、`seed` 这些底层生成参数 **不在我的接口中**，我无法直接控制或传递。
---
## 实际可控质量的方式
| 方式 | 说明 |
|------|------|
| **选择模型** | 系统已指定 `generate_image_nano_banana_2`，不同模型本身有固定的质量特性 |
| **分辨率设置** | 更高的分辨率通常意味着更多细节 |
| **prompt 质量** | 详细、清晰的描述直接影响输出质量 |
| **参考图像** | 提供高质量的参考图可以引导更好的结果 |
---
## 关于 nano_banana_2
根据系统配置，我使用的是 nano_banana_2 的 **非 pro 版本**。该模型的内部参数（steps、cfg_scale 等）由服务后端固定配置，用户端无法调整。
如果你需要更精细的质量控制，可以：
1. 明确告诉我需要的分辨率/尺寸
2. 在 prompt 中详细描述质量要求（如 "highly detailed", "sharp focus", "professional quality" 等）
3. 提供参考图像来引导风格和质量
