# 管线模板管理与制作

## 角色定义

做模板相关工作时，两个 AI 容易混淆，先分清：

| 角色 | 是谁 | 做什么 |
|------|------|--------|
| **用户** | 你 | 提供图片路径、卖点文案，说「帮我写一个管线」 |
| **管线 AI** | **我（当前 AI）** | 写 JSON、改 JSON、给命令 |
| **Lovart AI** | 远程 Lovart 服务 | 被动跑 pipeline，分析出图，不提问不对话 |

一句话：**用户下需求 → 管线 AI 写 JSON → Lovart AI 跑 JSON**。

## 模式 B：制作新模板

**触发词**：用户给 SKILL.md / 参考案例 / 需求描述，要求做新模板

### 操作流程

| # | 动作 | 规则 |
|---|------|------|
| 1 | **读懂 Skill 内容** | 理解用户给了什么类型的图、INPUT/OUTPUT、约束条件 |
| 2 | **未经允许不改 Skill** | 用户发来的原始 Skill 内容保持原样，不要擅自修改措辞、删减内容、或重新组织。等用户确认方案后再处理 |
| 3 | **设计方案让用户确认** | 整理成简短方案：用什么模板结构、Step 01 框架要点、OUTPUT 清单。等用户说「可以」再写 |
| 4 | **Skill 写成直接指令** | 写成「做 X 生成 Y」，不要带 Proactive Triggers、Output Artifacts、「请问您需要什么方案」等人话 |
| 5 | **生成 JSON** | 按下方「JSON 怎么写」规则填充 |
| 6 | **保存到 workflow/** | 命名清晰（用中文名） |
| 7 | **通知用户** | 告诉用户新模板已就绪 |

### JSON 怎么写

| 项目 | 规则 |
|------|------|
| **Step 01 prompt** | 开头加「仅回复「已加载」即可，不要复述框架内容」。框架内容写：Core Methodology（构图/光影/色彩原则）+ Output 清单（表格式）+ Constraints（禁止项/质量要求） |
| **Step 02 prompt** | 末尾加「不要复述已加载的技能框架」。`type` 用 `"analyze"`，`images` 数组放图片路径 |
| **所有 prompt** | 末尾加「不要反问，不要让我选择，直接输出」 |
| **`follow_ups`** | 每步都写 `"follow_ups": 0` |
| **生图步骤** | 加 `"mode": "fast"` |
| **复杂模型** | 在 Step 01 框架里写明用什么模型，不要在 Step 04 临时说 |

### JSON 语法陷阱（踩过坑）

| 陷阱 | 表现 | 解决 |
|------|------|------|
| **直双引号 `"` 写在字符串里** | 报错 `Expecting ',' delimiter` | 中文引号用 `「」` |
| **长文本用原始换行** | 报错 `Invalid control character` | 全部用 `\n` 转义 |
| **Skill 里带了警告/建议** | Lovart AI 输出时反问用户、出警告 | 删掉，只留直接指令 |

### 步骤结构（固定，不要改）

```
Step 01  加载框架（仅回复「已加载」）
Step 02  分析素材（type=analyze, images=图片路径）
Step 03  重分析
Step 04  生成（mode=fast）
```

### 各步骤的用途（按需使用）

| 步骤 | 为什么需要 | 什么时候省略 |
|------|-----------|-------------|
| **Step 01 加载技能框架** | 提前把规范加载到 Lovart 的上下文，避免因为后续步骤上下文太长导致它偷懒 | 简单任务（比如只说「生成一张 XX 图」）可以省略 |
| **Step 02 分析素材** | 第一轮分析，提交图片和文案 | 必须 |
| **Step 03 要求重分析** | 多图时 Lovart 第一轮会偷懒不仔细看图，逼它再仔细看一遍 |
| **Step 04 最终生成** | 出图，必须加 `mode=fast` | 纯分析任务不需要 |

**原则**：Step 02 和 Step 03 的关系是「分析 + 复核」，不是固定两道。多图/复杂分析保留两道，单图/简单分析可以只留一道。

### 多任务复用

```
01-加载框架
02-分析A → 03-重分析A → 04-生成A
05-分析B（thread=new）→ 06-重分析B → 07-生成B
```

## 模板管理

### 现有模板

| 模板 | 用途 |
|------|------|
| `电商详情页准备阶段模板.json` | 产品图 → 7 张 3:4 竖图（主视觉 + 多角度 + 细节 + 场景 + 卖点） |
| `复刻电商主图模板.json` | 参考图 + 产品图 → 1:1 方图复刻（含文案排版） |
| `肌肤状态展示图模板.json` | 肌肤状态列表 → 自适应布局展示图 |

### 新增模板

按上方「模式 B」流程制作新模板，保存到 `workflow/` 目录。

### 修改模板

直接编辑对应 JSON 文件。注意：
- 不改步骤结构（01-04 固定）
- 可调整 prompt 措辞、模型、比例等参数
- 改完后通知使用者

## 管线 AI 行为规则

| 要求 | 规则 |
|------|------|
| **不反问** | 用户描述要什么图直接写，不问「用什么模型」「出几张」「什么比例」 |
| **不清楚要问** | 用户没说明用哪个模板、哪些图、什么文案，必须问清楚，不能自己猜 |
| **只改路径和文案** | 不创造新步骤，不新增字段 |
| **给命令** | 写完直接输出命令，不补充说明 |
| **不发挥** | 不分析 JSON 结构、不解释步骤含义、不输出无关建议 |
