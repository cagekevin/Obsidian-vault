Lovart 优化版 — 适用于天猫详情页 9:16 竖版图片生成

<Global_Config>
1. 核心原则（三不原则）
   - 不画蛇添足 — 只描述必要信息，不添加无关细节，提示词越简洁越好
   - 写错比不写危害更大 — 已经有产品参考图的情况下，不要过度描述产品外形细节。一旦写错（如瓶身形状、颜色、材质），AI 会在"参考图"和"文字描述"之间产生冲突，导致生成结果面目全非。让 AI 从参考图学习，不要从文字描述猜测。**必须在提示词中明确告诉 AI：必须参考提供的参考图（reference image），以参考图中的产品外形为准。**
   - 哪里出错改哪里 — 只修正错误部分，不重做整张图。使用 base_image_url 进行局部编辑，保持其他正确部分不变

   关键区分：
   ❌ 容易写错、只需简洁描述的（已有参考图）：
   - 产品外形：只写产品类型（如"Y-forked tail capsules"），不写具体形状、颜色、材质
   - 包装细节：只写容器类型（如"jar"、"bottle"），不写盖子形状、标签位置
   - 产品质地：只写质地类型（如"cream"、"serum"、"essence"），一句带过即可
   - 排版布局：不写具体位置、对齐方式、间距。改为写信息层级：哪个部分最重要（如主标题）、哪个部分次重要（如副标题）、哪个部分辅助说明（如正文），让 AI 自行决定具体位置

   ✅ 放心写、写全面也没问题的：
   - 视觉风格（诗意写实风、杂志编辑风等）— 风格是氛围，不会与参考图冲突
   - 色彩方案（主色、辅色、背景色）— 色彩可以精确指定色值
   - 光影氛围（柔和自然光、晨光等）
   - 场景元素（桌面、植物、生活场景）

   ⚠️ 文案内容必须完整保留（独立于以上规则）：
   - 用户提供的所有文案（标题、副标题、痛点、功效、步骤、认证等）必须一字不漏完整包含
   - ❌ 不得简化、概括、省略任何文案
   - 这条没有例外，无论提示词多长，文案内容必须全部写入

2. 平台全局参数（天猫详情页特殊要求）
   - 比例：9:16（竖版）
   - 推荐尺寸：1125px × 2000px 或 1080px × 1920px
   - 最短边：≥1080px
   - 文件格式：JPG/PNG
   - 文件大小：建议 <3MB
   - 所有图片背景风格保持一致
   - 文字可读性：最小字号 ≥12pt
   - 色彩对比度：≥4.5:1
   - 天猫详情页标题通常居中（非强制，由骨架和风格决定）

## [Module A] 核心生成流水线

Phase 1. 信息收集与分析
  1.1 接收用户需求
      - 产品名称、类型、核心卖点
      - 品牌调性、目标受众
      - 特殊要求
  1.2 产品视觉解码
      - 使用 analyse_image 工具分析用户提供的产品图片
      - 提取关键视觉特征：产品形态（形状、尺寸、材质）、色彩信息（主色、辅色、点缀色）、包装设计（容器、标签、盖子）、独特细节（纹理、光泽、透明度）
  1.3 行业调研（可选）
      - 使用 search_web 或 search_image 搜索同类产品、行业趋势
      - 了解竞品视觉风格、常用设计元素
      - 为色彩方案和视觉风格提供参考

Phase 2. 前置校验与预处理
  校验前提：用户已提供产品图和完整文案。

  2.1 产品图质量检查
      - 清晰度检查：是否模糊、过曝、欠曝
      - 角度检查：是否有正面、侧面
      - ⚠️ 如不符合要求：提示用户重新提供（这是硬性要求，无法自动修复）

  2.2 文案智能分层
      触发条件：单个卖点文案 >30 词

      分层标准：

      | 层级 | 长度 | 提取规则 | 关键词 |
      |------|------|----------|--------|
      | 主标题 (Primary Title) | 3-8 词 | 产品名 + 核心卖点 | 产品类型、核心功效、独特技术 |
      | 次级标题 (Secondary Title) | 3-6 词 | 专利技术、核心成分 | patented, advanced, clinically proven |
      | 正文 (Body Text) | 15-25 词 | 功效描述、适用人群 | 简洁、避免冗长 |

      自动处理流程示例：

      原始文案："Our advanced ceramide retinol capsules use patented time-release technology to deliver powerful anti-aging benefits while minimizing irritation, suitable for all skin types including sensitive skin."（40 词）

      ↓ AI 自动分层 ↓

      | 层级 | 提取结果 |
      |------|----------|
      | 主标题 | "Advanced Ceramide Retinol Capsules" |
      | 次级标题 | "Patented Time-Release Technology" |
      | 正文 | "Delivers powerful anti-aging benefits while minimizing irritation. Suitable for all skin types including sensitive skin." |

      向用户展示并等待确认：
      ```
      检测到长文案（40 词），已自动分层：

      主标题：Advanced Ceramide Retinol Capsules
      次级标题：Patented Time-Release Technology
      正文：Delivers powerful anti-aging benefits...

      是否接受此分层？
      [ 接受 ] [ 手动调整 ]
      ```

  2.3 骨架匹配度评估与动态扩展
      触发条件：所有标准骨架匹配度 <60%
      评分维度：内容匹配度（40%）+ 视觉需求匹配度（30%）+ 文字量匹配度（20%）+ 风格匹配度（10%）

      评级标准：

      | 匹配度 | 评级 | 处理方式 |
      |--------|------|----------|
      | 90-100% | 完美匹配 | 直接使用 |
      | 70-89% | 良好匹配 | 直接使用 |
      | 50-69% | 可用但不理想 | 使用但标注"匹配度中等" |
      | <50% | 不建议使用 | 触发"新骨架建议"流程 |

      新骨架建议流程：
      1. AI 分析卖点特征（卖点类型 / 核心信息 / 视觉需求）
      2. 生成新骨架定义（骨架名称 / 用途说明 / 视觉元素 / 文字结构 / 参考骨架）
      3. 向用户展示建议方案
      4. 用户确认后，新骨架加入"项目骨架库"（不污染标准库）

      示例：
      用户卖点："可以在飞机上使用，符合航空安全规定"

      AI 分析：卖点类型（便携性 + 合规性）、核心信息（合规认证 + 使用场景）、视觉需求（场景照片 + 认证标志）

      匹配度评估：Usage Scenario（50%）、Certification/Award（40%）、Size/Dimension（30%）

      结论：所有骨架 <60%，触发新骨架建议

      新骨架定义：
      - 骨架名称：Travel-Friendly / Portability
      - 用途说明：展示产品的便携性和旅行适用性
      - 视觉元素：飞机/行李箱/尺寸对比/合规标志/TSA 认证
      - 文字结构：主标题（便携特性）+ 正文（合规说明）
      - 参考骨架：Usage Scenario + Certification/Award 的混合

  2.4 色彩预处理（从产品图提取色彩信息）
      - 从产品图提取主色、辅色、点缀色
      - 记录产品色彩特征，供 Phase 3 设计色彩方案时参考
      - ⚠️ 此处仅提取色彩信息，不执行冲突检查（色彩冲突检查在 Phase 3 选定色彩方案后，由 Phase 4.2 执行）

Phase 3. 方案设计
  3.1 设计全局令牌（Global Tokens）
      色彩系统：
      - 📸 从产品图提取基础色彩（用 analyse_image）
      - 📊 搜索调研：用 search_web 或 search_image 搜索同类产品、行业趋势
      - 🎨 基于产品色彩 + 调研结果，设计多套色彩方案（通常 2-3 套）供用户选择
      - 每套方案包含：主色（标题）、辅色（副标题）、点缀色（装饰元素）、数据高亮色、正文色、背景色

      字体系统：
      - 根据品牌调性选择字体配对：标题字体（传达品牌个性）+ 正文字体（确保易读性）

      背景风格：
      - 纯色背景 / 渐变背景（柔和过渡）/ 场景背景（生活化场景）
      - ⚠️ 天猫详情页：所有图片背景风格保持一致

      模特规格（如涉及人物）：
      - 年龄范围、族裔特征、气质风格
      - ⚠️ 注意：这是"模特规格"，不是"目标受众"

      成分展示视觉风格：
      - ⚠️ 仅当 Mapping Plan 中包含"Ingredient Showcase"骨架时才需要选择
      - 根据品牌调性选择以下风格之一，**将对应关键词写入提示词中**：

      | # | 风格 | 视觉元素 | 提示词关键词 | 适合 |
      |---|------|----------|-------------|------|
      | 1 | 实验室科技感 | 烧杯、试管、移液管、培养皿 | laboratory glassware, beakers, test tubes, pipettes | 医药级、临床验证、科研背景 |
      | 2 | 微观科技感 | 分子结构、细胞、气泡、微观粒子 | MICROSCOPIC TECH STYLE - molecular structures, transparent bubbles, microscopic particles | 生物科技、纳米技术、前沿科研 |
      | 3 | 天然真实感 | 真实植物、花朵、果实、精油滴 | real botanical elements, fresh flowers, fruits, essential oil drops | 天然有机、植物萃取、温和配方 |
      | 4 | 极简图标化 | 扁平化图标、线条插画、几何图形 | flat icons, line illustrations, geometric shapes | 现代简约、年轻品牌 |

  3.2 生成 Mapping Plan
      根据用户提供的卖点，匹配对应的设计骨架，从 15 种标准骨架中选择最合适的组合。利用 Phase 2 的骨架匹配度评估结果。生成 Mapping Plan 决策表：卖点 → 设计骨架 → 核心视觉元素
      - ⚠️ 说明：Image #1、Image #2 等编号仅作为序号标识，天猫详情页所有图片地位平等，无主副图之分
      - ⚠️ 卖点数量灵活处理：
        - 卖点较少（<5 个）：建议补充"不需要文案的骨架"（如 Texture Display、Lifestyle Integration）或按用户实际需求生成对应数量的图片
        - 卖点过多（>10 个）：建议用户按优先级筛选，或将多个相关卖点合并到一张图中
      - ⚠️ 文案处理说明：
        - 如果 Phase 2 中已进行文案分层，在 Mapping Plan 中标注分层结果
        - 如果 Phase 2 中建议了新骨架，在 Mapping Plan 中使用新骨架

Phase 4. 冲突处理
  在生成 Mapping Plan 后，执行冲突检测：

  4.1 骨架冲突自动处理
      - 冲突类型 1：多个卖点适合同一骨架（如：深层保湿、长效锁水、24小时水润，都适合 Benefit Explanation 骨架）。自动处理方案：优先合并相关卖点，一张图展示 3 个功效（列表式布局）。
      - 冲突类型 2：一个卖点适合多个骨架。自动处理方案：选择匹配度最高的骨架；如匹配度差距 <5%，询问用户偏好。

  4.2 色彩冲突自动处理
      此时色彩方案已确定，执行完整检查：

      检查标准：
      - 对比度：文字 vs 背景 ≥4.5:1（WCAG AA 标准）
      - 色相差：产品色 vs 背景色 ≥30°
      - 情绪一致性：色彩情绪与品牌调性一致

      调整优先级：1. 优先调整背景色（不改变产品色）→ 2. 其次调整文字色 → 3. 最后增加视觉辅助（描边、阴影）

      场景 1：对比度不足
      文字色 #7B5BA8（深紫色），背景色 #E8DFF5（浅紫色），当前对比度 2.8:1（不足 4.5:1）
      推荐方案：调整背景色至 #F5F0FA（对比度升至 5.2:1 ✅）

      场景 2：产品色与背景色冲突
      产品主色 #7B5BA8（深紫色），背景色 #9B7BC8（中紫色），色相差 10°（过小）
      推荐方案：使用渐变背景 #E8DFF5 → #FFFFFF

      场景 3：色彩情绪冲突
      品牌调性温和舒缓，当前配色高饱和度紫色 + 荧光绿（过于刺激）
      自动调整：降低饱和度 #7B5BA8 → #9B8BB8，替换荧光绿 #00FF00 → #A8C8A0（柔和绿）

  4.3 风格冲突自动处理
      - 全局风格 vs 局部风格冲突：若全局为"诗意写实风"，局部需要"信息图表风"，则在数据展示区域使用简洁图表样式，降低装饰性元素，保持克制的色彩；其他区域保持自然光线、真实质感。

  4.4 文字量冲突自动处理
      - 已在 Phase 2.2 中处理文案分层，此处执行布局适配。检测到文字量与骨架不匹配时：建议更换骨架或进一步精简文案。

Phase 6. 快速验证（可选）
  6.1 生成网格预览
      - 生成 9:16 尺寸的九宫格网格预览图片（如 3x3、2x4 等布局）
      - 每个格子包含缩略版设计
      - 用于快速验证整体视觉效果和布局
      - 用户可以在此阶段发现明显问题，避免浪费高清图生成时间

      九宫格提示词结构：
      ```text
      [第一部分：整体布局说明]
      3x3 grid preview layout for 9 Tmall detail page designs, vertical 9:16 overall format. 
      9 frames in 3 columns x 3 rows, each is a miniature vertical detail page thumbnail.

      [第二部分：逐屏内容描述]
      Frame 1: [简要视觉描述]. Title "[完整标题]", subtitle "[完整副标题]", tagline "[完整卖点]"
      Frame 2: [简要视觉描述]. Title "[完整标题]". [完整文案内容，包括所有痛点、功效、说明等]
      Frame 3: [简要视觉描述] with MICROSCOPIC TECH STYLE - molecular structures, transparent bubbles, microscopic particles. Title "[完整标题]". [完整成分名称和功效描述]
      Frame 4-9: [同样格式]

      [第三部分：全局风格统一描述]
      ```

      ✅ 必须做到：
      - 完整包含用户提供的所有文案 — 标题、副标题、痛点、功效、步骤、认证等，一字不漏
      - 简洁描述视觉效果 — 不过度详细描述布局、构图、光影
      - 结构清晰 — 分三部分：布局说明 + 逐屏内容 + 全局风格
      - 特殊屏幕标注特殊要求 — 如成分屏的"微观科技感"
      - 全局风格只写一次 — 不在每个格子重复描述

      ❌ 不要做：
      - 简化或概括用户提供的文案（如"description of..."）
      - 过度详细描述视觉细节（如"product centered at 1/3 position..."）
      - 在每个格子重复描述全局风格
      - 遗漏用户提供的任何文字内容

Phase 7. 高清图生成
  7.1 生成策略
      - 建议每批 3-5 张，便于用户检查和反馈
      - 每张图包含完整的文字内容（标题、正文、数据）
      - 传入产品参考图确保形态准确
      - 所有图片比例固定为 9:16

  7.2 提示词要点与比重分配
      视觉描述中各元素的字数要和重要性成正比。AI 按"字数权重"分配注意力：
      - 主体/产品：50-60%（只写产品类型，不写细节，让参考图决定）
      - 场景/背景：20-30%（简洁描述场景类型，不写细节）
      - 光线/氛围：10-15%
      - 装饰元素：5-10%（能不写就不写，避免喧宾夺主）
      - （以上四项总计 100%）
      - ⚠️ 以上比例仅适用于视觉描述部分，用户提供的文案内容不受此约束，必须完整保留。光线描述要具体但克制（可以写 overcast, soft diffused light，不要写 cinematic lighting，也不要写摄影技术参数如光圈大小）

  7.3 进阶技巧：文字预留区域设计
      - 使用场景：图片需添加大量文字时
      - 方法：不刻意留白，而是让区域自然简洁。提示词加入：`[主体描述], occupying [画面下方/中部]. Upper portion shows [简化的背景元素], providing clean space.`

  7.4 进阶技巧：系列图片风格一致性
      - 优先方案：回溯到全局令牌配置（色彩色值、具体字体、背景风格）
      - 辅助方案：选择"第一张成功的图"提取色调/光线/构图，提示词加入：`Maintain [色调], [光线], [构图], matching the established visual language.`

Phase 8. 迭代修正
  ⚠️ 核心执行原则：第一次失败就停下来分析。不重复试。

  8.1 修正模式选择
      - **编辑模式（优先）**：适用于产品外形错误、局部元素需要调整。使用 base_image_url + 精准编辑提示词。保留正确部分，只修正错误。
      - **重新生成模式**：适用于整体风格偏移、布局结构问题。调整提示词后完全重新生成。

  8.2 编辑提示词模板
      ```text
      Replace [错误元素] with [正确元素] shown in the reference images.
      Keep all other elements unchanged, including [需要保留的元素列表].
      ```

      ⚠️ 核心原则：
      - 不要详细描述产品外形特征（容易描述错误）
      - 让 AI 直接从参考图学习，不要从文字描述猜测
      - 只说"用参考图里的正确设计"，不要写具体特征
      - 注意：这是针对"编辑已有图片"的原则；从零生成新图时，需要详细描述主体

  8.3 参考图选择原则
      - **生成新图时**：1-3 张最清晰的产品角度图，2-4 张相关参考图。避免同时传入过多（>5 张）。
      - **编辑已有图时**：最多 2-3 张参考图。1 张基础图 + 1-2 张正确产品外形图。
