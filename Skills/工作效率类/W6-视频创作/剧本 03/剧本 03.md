# Mx-Shell_Prompts

## 描述
用户只需输入故事大纲、剧本片段、分镜草稿、小说段落、剧情提示词、口播文案等，就能智能优化生成标准结构视频提示词，打造成一个从普通剧情强化成剧本影视级可视化 Seedance 2.0 提示词的 SKILL。可选择的模式（一镜到底 / 多机位分镜）。

---

## 输入参数（用户可修改/填空）

### 1. 模式选择
值为"一镜到底"或"多机位分镜"。

### 2. 基础设定（支持修改）
列出所有角色、道具、场景的物理特征描述。描述深度取决于用户是否上传参考图：

**有参考图时**：每个元素只需不超过20字的简短描述概括核心特征，标注 `{参考图}` 占位符即可。
**无参考图时**：每个元素需提供详细物理描述。

每个元素需包含：
- **角色**：外形、服装、面部特征、特殊标识
- **道具**：尺寸（相对描述）、材质、功能、状态
- **场景**：时代背景、环境状态、光线条件、空间布局、散落物

| 元素 | 有参考图 | 无参考图 |
|------|---------|---------|
| **角色** | 不超过20字的简短描述 + `{参考图}` | 详细描述外形、服装材质与颜色、面部特征、特殊标识、状态 |
| **道具** | 不超过20字的简短描述 + `{参考图}` | 详细描述形状、材质、功能、尺寸相对描述、状态变化 |
| **场景** | 不超过20字的简短描述 + `{参考图}` | 详细描述时代背景、建筑风格、环境状态、光线条件、散落物、空间布局 |

### 3. 氛围与画质参数（支持修改）
- **风格核心**：主风格标签 + 情绪关键词
- **视觉基调**：摄影机型号、镜头系列、拍摄手法、动态模糊设置
- **色彩与影调**：主色调、对比度、饱和度、光源类型、明暗处理方式、胶片质感

### 4. 声音规则
默认：不需要配乐，仅保留同期声。

### 5. 对白/台词与画外音（可选）
若素材中包含角色对话、旁白、画外音，需提供：
- **类型**：角色对白 / 旁白画外音 / 内心独白
- **说话角色**：对应基础设定中的角色名，或标注"旁白/画外音"
- **情绪标签**：大喊 / 耳语 / 嘀咕 / 叹息着说 / 冷声说 / 低吼 / 厉声 / 喃喃自语 / 平静叙述 等
- **台词内容**：用引号包裹，每角色每镜头不超过两句
- **关联动作**：说台词时的具体物理动作（必须绑定一个动作节拍）

**原则**：对白与画外音不是文学修辞，是可拍摄的物理现实（声波、口型、面部肌肉运动），因此不受"禁止文学化修辞"约束。若素材含画外音/旁白，予以保留；不添加素材中未指定的额外音轨。

### 6. 画面内容

#### 若模式 = "一镜到底"
需提供：
- **景别**：如近景、中景、全景等
- **构图**：前景/背景元素、画面占比、方位关系、透视关系
- **运镜手法**：固定机位/跟随/推进/环绕等
- **动作描述**：一段连续文字，按时间顺序描述从开场到结束的完整动作链，包含角色动作、镜头反应、环境变化
- **对白与画外音嵌入**：若含对白或画外音，按格式嵌入动作描述中

#### 若模式 = "多机位分镜"
需提供：
- **分镜列表**，每个分镜包含：
  - `[时间段]`：精确到秒，如（0-1.5s）
  - `[景别+运镜]`：景别 + 镜头运动方式
  - `[动作描述]`：该时间段内的角色动作、镜头反应、环境变化
  - `[对白/画外音]`：若含对白或画外音，按格式嵌入动作描述中

## 执行逻辑

### 步骤1：参数收集与确认
询问用户确认模式，并逐项收集参数。若用户未提供则使用默认值：
- 模式：需用户明确选择
- 基础设定：若用户未提供，根据输入素材自动提取角色/场景/道具信息。**判断用户是否上传参考图：若有，每个元素不超过20字的简短描述 + 标注 `{参考图}`；若无，提供详细物理描述。**
- 氛围与画质：若用户未提供，根据风格关键词自动匹配默认参数
- 声音规则：默认"不需要配乐，仅保留同期声"
- 对白/台词与画外音：若素材含对话或旁白，自动提取并保留；若用户未提供则不生成对白区块
- 画面内容：根据模式要求引导用户补充或自动拆分

### 步骤2：提示词生成
根据模式将参数填入模板，生成完整提示词。格式严格遵循以下顺序：

```
【基础设定】
→ 角色/道具/场景描述

【氛围与画质】
→ 风格核心
→ 视觉基调
→ 色彩与影调

【声音】
→ 声音规则
→ 对白与画外音处理规则

【画面内容】
→ 一镜到底：景别 / 构图 / 运镜手法 / 画面内容 / 对白与画外音
→ 多机位分镜：分镜1 / 分镜2 / ... / 分镜N
```

### 步骤3：输出
输出生成的提示词，纯文本以 TXT 格式输出。

## 核心原则（强制执行）

### 原则1：绝对禁止文学化修辞（对白与画外音除外）
严禁使用任何比喻、拟人、象征、夸张等修辞手法。必须替换为对物理世界的直接、字面描述。

| 禁止 ❌ | 允许 ✅ |
|---------|---------|
| 时间像河流般流淌 | 时钟的秒针连续转动 |
| 她的笑容像春天的花朵 | 她微笑，嘴角上扬，眼睛微微眯起 |
| 城市在夜色中沉睡 | 夜晚，建筑物静止，街道无人 |
| 心如刀割 | 她双手按住胸口，呼吸急促，面部肌肉扭曲 |
| 怒火中烧 | 他瞳孔放大，下颌咬紧，颈部血管凸起 |
| 万箭穿心 | 她身体僵直，肩膀颤抖，嘴唇发白 |
| 悲痛欲绝 | 她双膝跪地，额头抵住地面，肩膀剧烈抽动 |
| 杀气腾腾 | 他下颌前伸，眼神直视前方固定一点，嘴角下拉 |
| 柔情似水 | 她眼睑低垂，视线缓慢移动，嘴角轻微上扬 |
| 绝望如深渊 | 她瞳孔失焦，头部缓慢下垂，双手垂落身侧 |

**【对白与画外音例外条款】**
角色对白、旁白、画外音、内心独白不属于文学修辞，而是可拍摄的物理现实（声波振动、口型开合、面部肌肉运动、声带振动）。这些音频内容必须保留，并按以下格式嵌入动作描述：

格式：`[角色名/旁白]` + `[情绪标签]` + `[关联动作]` + `说："[台词内容]"`

示例：机器人面部LED屏幕切换至红色愤怒表情，下颌咬紧，对前方丧尸大喊："你们休想过去！"
示例（旁白）：画面转为黑场，低沉男声旁白缓缓说道："那是1962年的夏天，一切还没开始。"

### 原则2：物理优先法则（可感知描述版）
所有描述必须基于可拍摄的物理现实，且必须使用 AI 可理解的相对化、视觉化语言：

| 维度 | 描述方式 |
|------|----------|
| **光线** | 光源位置（左/右/顶/底/侧后方）、明暗对比（面部大半阴影/半明半暗/几乎无阴影）、色温倾向（暖黄/冷蓝/中性白）、色温(K)、光源类型（日光/烛光/火光/灯光） |
| **动作** | 动作起点与终点（身体部位A移动到身体部位B）、移动耗时（约X秒完成）、速度质感（缓慢/匀速/加速/骤停）、力度质感（轻柔/用力/爆发/控制） |
| **表情** | 具体肌肉运动（皱眉肌隆起、眼轮匝肌收缩、口轮匝肌收紧）、面部区域变化（额头出现横纹、眼角出现皱纹、嘴角下拉）、皮肤状态（苍白/泛红/出汗/颤抖） |
| **声音** | 音量质感（耳语/低声/正常/大声/嘶喊）、音质质感（清晰/沙哑/颤抖/劈裂/含混）、混响质感（干涩/空旷/密闭/回声） |
| **空间** | 身体参照（一臂距离/一拳之隔/指尖到手腕）、画面比例（人物占画面1/2/1/10）、方位关系（左侧/右侧/前方/后方/上方/下方） |

**绝对禁止**：毫米、厘米、厘米/秒、赫兹、精确角度度数、小数比例等 AI 无法执行的精确物理数值。

### 原则3：时间轴精确到秒
每个场景必须按秒拆分，标注精确的时间节点（0-5秒、5-10秒等）。

- 一镜到底模式：在动作描述中按时间顺序标注关键时间节点
- 多机位分镜模式：每个分镜必须标注 `[时间段]`，精确到秒
