# 已知出错点与改进备忘录

> 2026-05-29，复盘 `0_skill_entry.md` 流程时识别的高风险手写错误。
> 2026-05-29 更新：问题 1-3 已通过 `scripts/validate_project.py` 实现自动化校验。
> 标 ✅ 的为已修复/已有防护，标 ⏳ 的为待观察是否需要增加自动化校验。

## 1. 垫图路径写死版本号 🔴

**问题描述：** 手写 shot/video JSON 时，本能地写 `char_main_female_before.png`，但 `assets_manifest.json` 的 `latest` 实际可能是 `char_main_female_before_ar4x3.png`。Agent 需要手动翻 manifest 确认最新版本，容易忘记。

**解决方法：** ✅ `scripts/validate_project.py` 已实现
- 校验：`python scripts/validate_project.py <项目路径> --type data assets shots` 自动检查 refs 在 manifest 中的 latest 版本
- 集成：`scripts/board_to_video.py` 自动调用 `--type data`，执行装配前先校验
- 仍可在生图前手动执行脚本确认

---

## 2. 手写 JSON 的令牌遗漏 🔴

**问题描述：** `scripts/storyboard.py` / `scripts/board_to_video.py` 会自动从 `global_vars.json` 注入风格令牌，但手写 shot/video/asset config JSON 时，Agent 必须自己前置 `--token-*`。从"脚本自动注入"切换到"手工前置"这个上下文切换，容易遗漏令牌。

**解决方法：** ✅ `scripts/validate_project.py` 已实现
- 校验：`python scripts/validate_project.py <项目路径> --type assets shots` 自动检查 prompt 前 300 字符内的 `--token-*`
- 注意：`storyboard_data_N.json` 不由 Agent 注入令牌（脚本自动拼装），故 `--type data` 不检查令牌

---

## 3. 图片 prompt 混入 `$not` 🟡

**问题描述：** 同时手写 image + video JSON 时，大脑切来切去，可能顺手把视频 prompt 里的 `$not` 带进了图片 prompt。Lovart API 的图片模型会被 `$not` 混淆。

**校验规则：** 图片 prompt 中绝对不允许出现 `$not`。

**解决方法：** ✅ `scripts/validate_project.py` 已实现
- 校验：`python scripts/validate_project.py <项目路径> --type assets shots` 自动检查图片 prompt 中的 `$not`
- `--type video` 跳过此检查（视频 prompt 允许 `$not`）

---

## 4. Windows 路径分隔符混用 🟡

**问题描述：** 技能使用 `G:\` Windows 路径，但实际开发环境在 macOS。手写时可能把 `\` 和 `/` 混用，导致命令跑不通。

**解决方法：** ⏳ 待观察
- `run_image_generator.py` 和 `run_video_generator.py` 已经在脚本内部做了路径标准化
- Agent 写模板时统一用 `/`（Python 兼容 Windows）
- 当前不影响实际执行，暂不处理

---

## 5. shot video 的垫图用错 🟡

**问题描述：** 视频垫图应该用**故事板图**（`images/storyboard_N_*.png`）作为主垫图，而不是角色定妆照。但补 shot video 时，刚出的 shot 图就躺在手边，容易把角色图塞进去当视频垫图。

**解决方法：** ⏳ 待观察
- 规范：shot video 的第一张垫图必须是刚生成的 shot 图（`images/group{N}_shot_XX.png`），角色图可作为补充垫图排第二
- 手动：写完后确认 `reference_image_paths` 的第一张不是 `assets/` 下的角色图
- 检查清单：SKILL.md 的视频类 #1 已写明"垫图用故事板图，不是角色定妆照"，但针对的是 group video，shot video 场景未覆盖

---

## 6. 角色资产 prompt 的一致性废话 🟢

**问题描述：** `role_art_director.md` 的"稳定对齐约束"写 `same character, same outfit palette`。但角色定妆照是多视图网格排版，AI 一次出图不会画出两个人。一致性靠 `reference_image_paths` 垫图，不是靠 prompt 文字。

**解决方法：** ✅ 已修改
- 删掉"稳定对齐约束"整节
- JSON 示例末尾删掉对应的废话
- 新增垫图规则说明：变化版本的后一版必须引用前一版作为垫图
- `run_image_generator.py` 自动注入垫图说明，不需要在 prompt 里写
