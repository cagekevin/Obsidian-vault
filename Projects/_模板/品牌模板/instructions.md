# {{品牌名}}品牌项目结构

```
Projects/{{品牌名}}/
├── brand.md              ← 品牌资产总文档（核心，所有输出以此为准）
├── 资产/                 ← 共用：产品档案、垫图、文案库等定期沉淀
├── 别人的参考图/          ← 共用：别人的素材，仅参考
├── 作图经验/             ← 共用：作图方法论和经验沉淀
│
├── YYYY-MM-DD-项目名/    ← 具体项目（日期在最外层）
│   ├── 01-输入/          ← 原始需求/brief/素材
│   ├── 02-中间过程/      ← AI 生图过程的产物（备选图、中间图）
│   └── 03-输出/          ← 最终成品
│
└── ...更多项目/
```

## 共用层

`brand.md`、`资产/`、`别人的参考图/`、`作图经验/` — 所有项目共用，不重复建。

### 资产/

沉淀下来的精选内容，按产品建文件夹，不加日期前缀。

```
✅ 资产/虾青素胶囊精华/
✅ 资产/虾青素胶囊精华.md
```

每个产品有一个`.md`档案文件 + 一个同名文件夹放图片。

**源文件规则：**
- 真正的原始源文件（如.tif、.psd）放在产品文件夹下的`源文件/`子目录
- 常用的导出图（如.jpg、.png垫图）直接放在产品文件夹下
- `源文件/`里只放真正的源文件，不放重复的导出图

### 别人的参考图/

别人的素材，仅用于参考风格/构图/配色。按编号顺序命名，编号后标注渠道。

```
✅ 01-电商.jpg
✅ 02-电商.jpg
✅ 03-社媒.jpg
❌ 参考图1.jpg
```

渠道标识：电商、社媒、官网、视频、线下

### 作图经验/

作图方法论和经验沉淀。AI 每次出图前先来这里。

```
作图经验/
├── instructions.md          ← 自进化规则、文件清单、出图前流程
├── 00-{{品牌名}}出图工作流.md  ← 入口文件，先读这个
├── 风格/                    ← 风格定义（琥珀光影等）
├── 工作流/                  ← 流程和方法论（协作流程、专项技能）
├── 积木/                    ← 可复用的提示词技巧
├── 案例/                    ← 按场景分的踩坑记录
└── 复用/                    ← 已确认的正式版提示词
```

详细说明 → 读 `作图经验/instructions.md`

## 项目层

每个项目一个独立文件夹，日期在最外层，内部按序号排列。

### 命名规范

```
项目文件夹 → YYYY-MM-DD-项目名/              例: 2026-06-24-唯品会-{{品牌名}}主图/
01-输入/   → 保持原始文件名或简短描述           例: 唯品会设计需求.xlsx / 需求整理.md
02-中间过程 → 第N组-描述/                     例: 第1组-组合1垫图/
03-输出/   → NN-描述.扩展名                   例: 01-主图-最终版.jpg
```

项目内部的输入/中间过程/输出**不加日期前缀**，日期已体现在项目文件夹名上。

## Workflow

### 日常接单（Kevin + AI）

| 步骤 | 谁做 | 做什么 |
|------|------|--------|
| 1. 收需求 | **Kevin** | 建 `YYYY-MM-DD-项目名/`，把文案/素材扔进 `01-输入/` |
| 2. 理需求 | **AI** | 读 `01-输入/`，提取卖点/调性/必须元素，写需求摘要 |
| 3. 找参考 | **Kevin** | 看到好的参考图，扔 `别人的参考图/`，按 `NN-渠道` 编号命名 |
| 4. 生图 | **AI** | 读 `作图经验/` → 写提示词 → 逻辑矛盾检查 → Kevin 确认 → 出图，产物放 `02-中间过程/` |
| 5. 定稿 | **Kevin** | 选中的成品放 `03-输出/` |
| 6. 交付 | **Kevin** | 把成品发给甲方/业务方 |
| 7. 复盘 | **AI** | 出图过程中踩的坑 → 记到当天校准日志（标记 `→ 待沉淀`）。新 session 检查是否有待沉淀条目，满 2-3 次主动问 Kevin 是否建场景文件 |

### 定期沉淀（Kevin 触发，AI 执行）

| 频率 | 谁触发 | AI 做什么 |
|------|--------|-----------|
| 每周/月 | **Kevin 说"整理资产"** | 扫描所有项目的 `03-输出/`，按主题推荐分类到 `资产/` |
| 每周/月 | **Kevin 说"整理文案"** | 扫描所有项目的 `01-输入/` 里的文案，汇总精华到 `资产/文案库.md` |
| 项目结束 | **Kevin 说"复盘"** | 扫整个项目，写总结：做了什么、哪些效果好、踩了什么坑 |
