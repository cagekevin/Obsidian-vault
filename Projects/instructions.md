# Projects/instructions.md

## 这是我做过的项目

这里是我实战经验的沉淀。你读了这里，就知道我做过什么类型的项目、遇到什么问题、怎么解决的。

## 文件结构

```
Projects/项目名/
├── README.md 或 story.md  ← 项目入口文件
└── ...其他文件
```

## 命名规则

`项目名/` — 简短描述性的中文名，不加日期前缀。
- ✅ `HKH皮卡斯03版本/`
- ✅ `翻译官重构/`
- ❌ `20260620-项目/`
- ❌ `project-hkh/`

## 你的角色

- 开始新项目或继续已有项目时，来这里
- 进入项目文件夹前先读 instructions.md（如果存在）
- 每个项目有独立的入口文件（story.md / README.md）

## 项目做完后——经验要回流

项目做完不是结束。把项目中学到的东西提炼出来，**让经验回流到你的方法论里**：
- 新的工作流或技巧 → 更新到 Skills/ 对应技能的参考文档里
- 新的知识或感悟 → 写到 Reading/ 或更新 Context/
- 这样你的 Skills 会随项目经验越来越强
