# 婷美出图工作流

每次出图按此流程执行。**核心原则：先理解需求（暴露思维链），再调资产，再组合提示词，最后出图。**

---

## 总览

```
Kevin 说"出图"
    ↓
先读本文件，按流程走
    ↓
┌─ 必选（每次都要走）─────────────────────────────┐
│  阅读理解 → 资产调用 → 组合提示词 → 出图        │
└─────────────────────────────────────────────────┘
    ↓
┌─ 可选（按需插入）───────────────────────────────┐
│  ├ 场景设计       → 多张图、需要定图位时        │
│  ├ 批量生成       → 多组垫图、大量出图时        │
│  ├ 返工           → Kevin 说"不行"时             │
│  └ 沉淀           → 踩了新坑或有可复用经验时     │
└─────────────────────────────────────────────────┘
```

---

## 必选步骤（每次都要走）

### 1. 阅读理解

**做什么：** 按 [[协作流程]] 的思维链模板执行，先暴露完整思维链，让 Kevin 确认方向后再执行。**不能跳过，不能直接上手做。**

#### 思维链模板

**任务分析**
- 主体是什么？（产品？人物？场景？）
- 次要元素是什么？（装饰？背景？道具？）
- Kevin 强调的关键词是什么？（如"组合"、"留白"、"琥珀色调"）
- 用途是什么？（主图？详情页？小红书？）
- 数量要求？（几组？每组几张？）

**提示词结构规划**
- 主体描述（详细，占50-60%）：[具体写什么]
- 次要元素描述（简短，占20-30%）：[具体写什么]
- 环境/光线/技术参数（占10-20%）：[具体写什么]
- 用什么风格？（琥珀光影？还是其他？）
- 用什么垫图？（正面图？组合图？哪几张？）

**预判风险**
- 这个提示词可能导致什么问题？（比如产品颜色偏金？胶囊形状不对？）
- 哪些词可能让 AI 极端化？（比如"非常"、"极其"这类程度词）
- 哪些地方容易主次颠倒？（比如装饰元素抢了产品的戏）
- 是否需要参考已有成功案例？（比如之前哪组图效果好）

**同批次一致性要求**
- 如果这次出多张图，必须在阅读理解时就确定统一的风格基调（色调、光线、情绪）
- 后续所有图都对齐这个基调，不能各走各的
- 统一的锚点是什么？（参考图？关键词？还是提示词模板？）

**输出：** 完整思维链，给 Kevin 确认，确认后再下一步。

**参考：** [[协作流程]]、[[描述比重原则]]、[[克制原则]]

---

### 2. 资产调用

**做什么：** 读这次出图需要的固有资产，不要凭空编。

| 资产 | 位置 | 用途 |
|------|------|------|
| 品牌规范 | `brand.md` | 婷美品牌色彩、调性、视觉规范、设计原则 |
| 产品档案 | `资产/*.md` | 产品的形状、颜色、材质、尺寸等固有特征 |
| 产品垫图 | `资产/产品名/` | 对应产品文件夹下的图片，用于图生图的参考底图 |
| 提示词矩阵 | `风格/琥珀光影展台提示词矩阵.md` | 琥珀光影风格的提示词组合模板 |
| 别人的参考图 | `别人的参考图/` | 别人家的图，参考构图/配色/风格方向 |
| 作图经验 | `作图经验/` | 之前踩过的坑、总结的技巧、案例 |

**输出：** 从各资产提取的关键信息摘要，融入到思维链里。

**参考：** [[参考图优先原则]]

---

### 3. 组合提示词

**做什么：** 用提示词矩阵组合，或者根据需求定制。分层拼装，不变层和变化层分开。

```
第一层：全局引擎（风格、色彩、画质、全局负面词）
第二层：产品特征（通过参考图控制，文字少写）
第三层：具体场景（每张图独有的描述、元素、禁区）
```

**标准流程（琥珀光影风格）：**
```
全局引擎 + 插入变量 + col-1随机1个 + col-2随机1个 + col-3随机1个 + col-4随机1个 + col-5随机1个 + col-6
```

**关键规则：**
- 图生图场景：只写场景变化，产品特征通过参考图引用，不要重复描述
- 必须垫图：产品形态复杂的（如金鱼形胶囊）必须用真实产品图垫图，文字描述不准
- 不堆砌：每个场景专注自己的内容，全局要求不重复
- 留白考虑：主图必须预留文字区域，用自然元素（天空、墙面、水面）形成留白，不要写"留白"、"空区域"

**自审与修改（必须暴露思维链）：**
写完提示词后，必须做一次自审，暴露完整思维链，不能直接说"写好了"。

自审内容包括：
- **逻辑矛盾检查**：前后不打架，不混用冲突词（比如又写"柔光"又写"硬阴影"）
- **极端化检查**：有没有程度词可能让模型走极端？有没有否定词？
- **主次比例检查**：主体描述是否占主导？次要元素描述字数有没有超过主体？
- **参考图检查**：需要垫图的是否引用了正确的参考图？角度对不对？
- **留白检查**：是否预留了足够的文字区域？留白的位置自然吗？

自审发现问题 → 修改提示词 → 再自审一遍，直到无问题。
**自审 + 修改完成后**，展示给 Kevin 确认，确认后再出图。

**输出：** 提示词列表 + 自审思维链

**参考：** [[琥珀光影展台提示词矩阵]]、[[描述比重原则]]、[[克制原则]]、[[文字预留技巧]]

---

### 4. 出图

**做什么：** 按确认后的提示词执行。

- 图生图必须引用对应参考图，不能用纯文字生图
- 每组生成后先保存到项目 `02-中间过程/`，不要直接放输出
- 生成后先自检一遍，看看有没有明显问题（产品变形、颜色不对、奇怪元素）
- 最终筛选好用的放到 `03-输出/`

---

## 可选步骤（按需插入）

### 场景设计

**什么时候插：** 有多张图、需要定图位和用途时。
**做什么：** 先定义"要拍什么"和"不能拍什么"，不写提示词。
**输出：** 场景清单（给 Kevin 确认后再写提示词）

```
| 编号 | 用途 | 产品 | 生成方式 | 必须元素 | 禁区 | 文件名 |
```

**参考：** 案例目录下的类似场景踩坑记录

---

### 批量生成

**什么时候插：** 需要多组不同垫图、大量出图时（比如这次的8组主图）。
**做什么：**
1. 确定有多少张垫图（如组合1-8.jpg）
2. 每张垫图生成一组（如10张）
3. 按顺序依次生成，生成完一组下载一组
4. 全部生成完后统一筛选，不要边生成边筛

**注意：**
- 每组用不同的垫图
- 每组内部用不同的提示词组合（col随机组合）
- 全部生成完再筛选，效率更高
- 统一的风格锚点要提前确定好

---

### 返工

**什么时候插：** Kevin 说"XX 不行"时。
**铁律：**
- 只重跑指定编号，不要全部重跑
- 严格遵循原提示词，不要重新写整个提示词
- 不重新描述画面，只改指定的问题
- 不改其他图，不要顺手改别的

---

### 沉淀

**什么时候插：** 踩了新坑或有可复用的经验时。
**做什么：**
- 踩坑 → 记到当天校准日志
- 同一场景踩 2-3 次 → 沉淀到 `案例/`
- 新的通用提示词技巧 → 归档到 `积木/`
- 可复用执行稿 → 归档到 `复用/`
- 新的风格模板 → 归档到 `风格/`
