# 作图经验

所有和作图相关的经验沉淀。AI 每次出图前先来这里，按场景读对应文件。

这个文件夹的核心目标是**自进化** — AI 通过踩坑、记录、沉淀，越来越懂 Kevin 的作图方式。

---

## AI 的核心职责

AI 是这个文件夹的维护者。每次出图、每次踩坑，都要让这里的经验更丰富、更准确。

---

## 文件职责边界

每个文件有明确的职责范围，补充新内容时对号入座：

## 文件清单

| 位置 | 职责 | 什么时候加内容 |
|------|------|--------------|
| `00-婷美出图工作流.md` | **入口** — 先读这个，按流程执行 | 流程有问题时 |
| `工作流/` | **流程和方法论** — 协作流程、白底技能等 | 发现新的工作流经验时 |
| `风格/` | **风格定义** — 琥珀光影等画面风格 | 发现新的风格技巧时 |
| `积木/` | **可复用的提示词技巧** — 文字预留、燃烧场景技巧等 | 发现新的通用技巧时 |
| `案例/` | **场景踩坑记录** — 成功/失败案例 | 同一场景踩 2-3 次时建新文件 |
| `复用/` | **可跨项目复用的执行稿/模板** — 其他项目可直接引用，不用重复做 | 完成一次完整出图流程后归档通用模板 |

**补充新内容的规则：**

**放 `作图经验/` 的（方法论、踩坑、流程）：**
- 新的踩坑/避坑 → `工作流/协作流程.md`
- 新的风格技巧 → `风格/` 下建新文件
- 新的专项技能（如小红书出图）→ `工作流/` 下建新文件
- 新的通用提示词技巧 → `积木/` 下建新文件
- 新的场景经验 → `案例/` 下建新文件
- 新的可复用执行稿 → `复用/` 下建新文件
- 流程有问题 → `00-婷美出图工作流.md`

**不放 `作图经验/` 的（固有资产）：**
- 新产品建档 → `资产/` 下建档案
- 品牌视觉规范更新 → `brand.md`
- 新的垫图 → 对应 `资产/产品名/` 下加图片
- 别人的参考图 → `别人的参考图/` 下加图片

## 出图前 AI 要做的事

1. 先读 `00-婷美出图工作流.md` — 按流程执行
2. 按步骤调用对应文件（资产/风格定义/场景案例）
3. **场景设计必须先给 Kevin 确认，再写提示词**
4. 提示词分层拼装：全局风格 → 产品特征 → 具体场景
5. **逻辑矛盾检查**（前后不打架，不混用冲突词）
6. 按编号出图，按编号返工

## 自进化机制

### 1. 踩坑 → 校准日志（立即）

每次出图踩坑，AI 必须立即记到当天校准日志（`.codebuddy/memory/YYYY-MM-DD.md`），并用标记标注：

```
## 作图踩坑
- **问题**：[具体问题]
- **修复**：[怎么修的]
- **→ 待沉淀**：如果这是同类坑的第 2-3 次
```

### 2. 校准日志 → 经验文件（AI 主动触发）

新 session 启动时，AI 检查最近 7 天的校准日志：
- 发现有标记 `→ 待沉淀` 的条目
- 同一个场景已出现 2-3 次
- AI 主动问 Kevin："这个坑出现 X 次了，要建 `案例/XXX场景.md` 吗？"

Kevin 说"建" → AI 创建文件，写入经验。
Kevin 说"不用" → 去掉 `→ 待沉淀` 标记，继续观察。

### 3. 经验文件 → 执行（出图时自动引用）

每次出图前，AI 按场景路由到对应文件。文件越多，AI 越少踩坑。

### 4. 旧文件淘汰（AI 定期提醒）

- 文件超过 3 个月未被引用 → AI 在 session 末尾提醒："`XXX场景.md` 最近 3 个月没被引用，是否归档？"
- Kevin 说"归档" → 移到 `案例/_归档/`
- Kevin 说"保留" → 重置计时

## 命名规则

`案例/` 下的文件用场景名命名，不加日期前缀：
```
✅ 海岸场景.md
✅ 燃烧场景.md
✅ 手持产品.md
❌ 2026-06-22-海岸场景.md
```
