# 作图经验

所有和作图相关的经验沉淀。AI 每次出图前先来这里，按场景读对应文件。

## 文件清单

| 文件 | 说明 |
|------|------|
| `风格定义.md` | 诗意写实风格定义、禁用词/推荐词、提示词结构公式 |
| `协作流程.md` | 作图协作流程、思维链模板、避坑指南 |
| `场景案例/` | 按场景分的踩坑记录和成功案例 |

### 场景案例/

| 文件 | 适用场景 |
|------|---------|
| `场景案例/2026-06-22-岩玫瑰海岸场景.md` | 海岸/自然场景，含文字预留技巧、燃烧场景铁律 |

## 出图前 AI 要做的事

1. 先读 `风格定义.md` — 确定风格和词汇规则
2. 判断场景类型 → 去 `场景案例/` 找对应文件读
3. 输出提示词 → 逻辑矛盾检查（前后不打架，不混用冲突词）
4. 展示给 Kevin 确认

## 如何沉淀新经验

踩坑后立即记到当天校准日志（`.codebuddy/memory/YYYY-MM-DD.md`）。

同一类坑反复 2-3 次后，在 `场景案例/` 下建新文件沉淀。新文件按场景命名，不加日期前缀。

## 命名规则

`场景案例/` 下的文件用场景名命名，不加日期前缀：
```
✅ 海岸场景.md
✅ 燃烧场景.md
✅ 手持产品.md
❌ 2026-06-22-海岸场景.md
```

## 自进化规则

- 文件 < 10 个：平铺在 `作图经验/`
- 文件 10-30 个：按主题建子文件夹（场景/技巧/词汇）
- 文件 30+ 个：AI 定期检查未引用文件，提醒归档
