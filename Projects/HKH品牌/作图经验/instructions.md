# 作图经验

所有和作图相关的经验沉淀。AI 每次出图前先来这里，按场景读对应文件。

这个文件夹的核心目标是**自进化** — AI 通过踩坑、记录、沉淀，越来越懂 Kevin 的作图方式。

---

## AI 的核心职责

AI 是这个文件夹的维护者。每次出图、每次踩坑，都要让这里的经验更丰富、更准确。

## 文件清单

| 文件 | 说明 |
|------|------|
| `风格定义.md` | 诗意写实风格定义、禁用词/推荐词、提示词结构公式 |
| `白底产品照技能.md` | 电商白底产品图建档与多角度扩展完整技能 |
| `协作流程.md` | 作图协作流程、思维链模板、避坑指南 |
| `场景案例/` | 按场景分的踩坑记录和成功案例 |

### 场景案例/

| 文件 | 适用场景 |
|------|---------|
| `场景案例/岩玫瑰海岸场景.md` | 海岸/自然场景，含文字预留技巧、燃烧场景铁律 |

## 出图前 AI 要做的事

1. 先读 `风格定义.md` — 确定风格和词汇规则
2. 判断场景类型 → 去 `场景案例/` 找对应文件读
3. 输出提示词 → **逻辑矛盾检查**（前后不打架，不混用冲突词）
4. 展示给 Kevin 确认

## 自进化机制

### 1. 踩坑 → 校准日志（立即）

每次出图踩坑，AI 必须立即记到当天校准日志（`.codebuddy/memory/YYYY-MM-DD.md`），并用标记标注：

```
## 作图踩坑
- **问题**：[具体问题]
- **修复**：[怎么修的]
- **→ 待沉淀**：如果这是同类坑的第 2-3 次
```

### 2. 校准日志 → 经验文件（AI 主动触发）

新 session 启动时，AI 检查最近 7 天的校准日志：
- 发现有标记 `→ 待沉淀` 的条目
- 同一个场景已出现 2-3 次
- AI 主动问 Kevin："这个坑出现 X 次了，要建 `场景案例/XXX场景.md` 吗？"

Kevin 说"建" → AI 创建文件，写入经验。
Kevin 说"不用" → 去掉 `→ 待沉淀` 标记，继续观察。

### 3. 经验文件 → 执行（出图时自动引用）

每次出图前，AI 按场景路由到对应文件。文件越多，AI 越少踩坑。

### 4. 旧文件淘汰（AI 定期提醒）

- 文件超过 3 个月未被引用 → AI 在 session 末尾提醒："`XXX场景.md` 最近 3 个月没被引用，是否归档？"
- Kevin 说"归档" → 移到 `场景案例/_归档/`
- Kevin 说"保留" → 重置计时

## 命名规则

`场景案例/` 下的文件用场景名命名，不加日期前缀：
```
✅ 海岸场景.md
✅ 燃烧场景.md
✅ 手持产品.md
❌ 2026-06-22-海岸场景.md
```

## 自进化阶段

| 阶段 | 文件数 | 结构 | 触发条件 |
|------|--------|------|---------|
| 当前 | <10 | 平铺 | — |
| 中期 | 10-30 | 按主题分子文件夹 | AI 自动建 |
| 成熟 | 30+ | 同上 + 淘汰提醒 | AI 定期检查 |
