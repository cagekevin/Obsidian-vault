# 出图工作流

每次出图按此流程执行。**核心原则：先理解需求，再调资产，再写提示词，最后出图。**

---

## 总览

```
必选步骤（每次都要走）：
  阅读理解 → 资产调用 → 拼提示词 → 出图

可选步骤（按需插入）：
  ├─ 场景设计（多张图时）
  ├─ 物料包→正式版（提示词需要审阅时）
  ├─ 返工（Kevin 说不行时）
  └─ 沉淀（踩了新坑或有可复用执行稿时）
```

---

## 必选步骤

### 1. 阅读理解

**做什么：** 按 `03-协作流程.md` 的思维链模板执行，先想清楚再动手。
**参考：** `03-协作流程.md`

思维链模板要求想清楚：
- 主体是什么？次要元素是什么？
- Kevin 强调的关键词是什么？
- 有什么风险？哪些词可能导致 AI 极端化？
- 是否需要参考已有成功案例？

**同批次一致性要求：** 如果这次出多张图，必须在阅读理解时就确定统一的风格基调（色调、光线、情绪），后续所有图都对齐这个基调，不能各走各的。

**输出：** 一段需求理解，给 Kevin 确认

---

### 2. 资产调用

**做什么：** 读这次出图需要的所有"不变信息"。
**参考：** 按需选择以下模块

| 模块 | 文件 | 什么场景用 |
|------|------|-----------|
| 产品档案 | `资产/*.md` | 每次都用，读对应产品 |
| 风格定义 | `02-风格定义.md` | 确定风格时 |
| 品牌规范 | `brand.md` | 需要品牌视觉规范时 |
| 参考图 | `参考/` | 需要参考构图/配色时 |
| 白底技能 | `04-白底产品照技能.md` | 白底图时 |

**输出：** 从各模块提取的视觉信息摘要

---

### 3. 拼提示词

**做什么：** 分层拼装，不变层和变化层分开。

```
第一层：全局要求（风格、色彩、比例、光线、全局负面词）
第二层：产品特征（名称、形状、颜色、材质、禁区）
第三层：具体场景（每张图独有的描述、元素、禁区）
```

**关键规则：**
- 图生图场景：只写场景变化，产品特征通过参考图引用
- 不堆砌：每个场景专注自己的内容，全局要求不重复
- 逻辑矛盾检查：前后不打架，不混用冲突词

**输出：** 提示词

---

### 4. 出图

**做什么：** 按提示词执行。
- 图生图必须引用对应参考图
- 输出到项目 `03-输出/`

---

## 可选步骤

### 场景设计

**什么时候插：** 有多张图、需要定图位和用途时。
**做什么：** 先定义"要拍什么"和"不能拍什么"，不写提示词。
**参考：** `场景案例/` 类似场景的踩坑记录
**输出：** 场景清单（给 Kevin 确认后再写提示词）

```
| 编号 | 用途 | 产品/人物 | 生成方式 | 必须元素 | 禁区 | 文件名 |
```

---

### 物料包 → 正式版

**什么时候插：** 提示词需要给 Kevin 审阅时。
**做什么：**
1. 先写**物料包**（准备稿，含讨论过程和选择理由）
2. Kevin 确认后，整理成**正式版**（去掉"我觉得""建议"等废话，只保留模型直接执行的指令）

---

### 返工

**什么时候插：** Kevin 说"XX 不行"时。
**铁律：**
- 只重跑指定编号
- 严格遵循原提示词
- 不重新描述画面
- 不改其他图

---

### 沉淀

**什么时候插：** 踩了新坑或有可复用的执行稿时。
**做什么：**
- 踩坑 → 记到当天校准日志
- 同一场景踩 2-3 次 → 沉淀到 `场景案例/`
- 可复用执行稿 → 归档到 `可复用执行稿/`
