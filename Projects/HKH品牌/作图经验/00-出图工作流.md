# 出图工作流

每次出图按此流程执行。**核心原则：先理解需求，再调资产，再写提示词，最后出图。**

---

## 总览

```
Kevin 说"出图"
    ↓
先读本文件，按流程走
    ↓
┌─ 必选（每次都要走）─────────────────────────────┐
│  阅读理解 → 资产调用 → 拼提示词 → 出图          │
└─────────────────────────────────────────────────┘
    ↓
┌─ 可选（按需插入）───────────────────────────────┐
│  ├ 场景设计       → 多张图、需要定图位时        │
│  ├ 物料包→正式版  → 提示词需要审阅时            │
│  ├ 返工           → Kevin 说"不行"时             │
│  └ 沉淀           → 踩了新坑或有可复用执行稿时   │
└─────────────────────────────────────────────────┘
```

---

## 必选步骤（每次都要走）

### 1. 阅读理解

**做什么：** 按 `工作流/协作流程.md` 的思维链模板执行，先想清楚再动手。

先想清楚：
- 主体是什么？次要元素是什么？
- Kevin 强调的关键词是什么？
- 有什么风险？哪些词可能导致 AI 极端化？
- 是否需要参考已有成功案例？

**同批次一致性要求：** 如果这次出多张图，必须在阅读理解时就确定统一的风格基调（色调、光线、情绪），后续所有图都对齐这个基调，不能各走各的。

**输出：** 一段需求理解，给 Kevin 确认

**参考：** `工作流/协作流程.md`

---

### 2. 资产调用

**做什么：** 读这次出图需要的固有资产。

| 资产 | 位置 | 用途 |
|------|------|------|
| 产品档案 | `资产/*.md` | 产品的形状、颜色、材质等固有特征 |
| 产品垫图 | `资产/产品名/` | 对应产品文件夹下的图片，用于图生图的参考底图 |
| 品牌规范 | `brand.md` | 品牌色彩、调性等固有视觉规范 |
| 风格定义 | `风格/诗意写实风格.md` | 画面风格描述和词汇规则 |
| 别人的参考图 | `别人的参考图/` | 别人家的图，参考构图/配色/风格方向 |
| 通用技巧积木 | `积木/` | 可复用的提示词技巧（文字预留、燃烧场景等） |

**输出：** 从各资产提取的关键信息摘要

---

### 3. 拼提示词

**做什么：** 分层拼装，不变层和变化层分开。

```
第一层：全局要求（风格、色彩、比例、光线、全局负面词）
第二层：产品特征（名称、形状、颜色、材质、禁区）
第三层：具体场景（每张图独有的描述、元素、禁区）
```

**关键规则：**
- 图生图场景：只写场景变化，产品特征通过参考图引用
- 不堆砌：每个场景专注自己的内容，全局要求不重复

**自审与修改（必须暴露思维链）：**
写完提示词后，AI 必须做一次自审，暴露完整思维链，不能直接说"写好了"。

自审内容包括：
- **逻辑矛盾检查**：前后不打架，不混用冲突词
- **极端化检查**：有没有词可能让模型走极端
- **主次比例检查**：主体描述是否占主导，次要元素是否喧宾夺主
- **参考图检查**：需要垫图的是否引用了正确的参考图

自审发现问题 → 修改提示词 → 再自审一遍，直到无问题。
**自审 + 修改完成后**，展示给 Kevin 确认，确认后再出图。

**输出：** 提示词

**参考：** `积木/`（可复用的提示词技巧）

---

### 4. 出图

**做什么：** 按提示词执行。
- 图生图必须引用对应参考图
- 输出到项目 `03-输出/`

---

## 可选步骤（按需插入）

### 场景设计

**什么时候插：** 有多张图、需要定图位和用途时。
**做什么：** 先定义"要拍什么"和"不能拍什么"，不写提示词。
**输出：** 场景清单（给 Kevin 确认后再写提示词）

```
| 编号 | 用途 | 产品/人物 | 生成方式 | 必须元素 | 禁区 | 文件名 |
```

**参考：** `案例/` 类似场景的踩坑记录

---

### 物料包 → 正式版

**什么时候插：** 提示词需要给 Kevin 审阅时。
**做什么：**
1. 先写**物料包**（准备稿，含讨论过程和选择理由）
2. Kevin 确认后，整理成**正式版**（去掉"我觉得""建议"等废话，只保留模型直接执行的指令）

---

### 返工

**什么时候插：** Kevin 说"XX 不行"时。
**铁律：**
- 只重跑指定编号
- 严格遵循原提示词
- 不重新描述画面
- 不改其他图

---

### 沉淀

**什么时候插：** 踩了新坑或有可复用的执行稿时。
**做什么：**
- 踩坑 → 记到当天校准日志
- 同一场景踩 2-3 次 → 沉淀到 `案例/`
- 新的通用提示词技巧 → 归档到 `积木/`
- 可复用执行稿 → 归档到 `复用/`
