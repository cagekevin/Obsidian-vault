# HKH 品牌项目结构

```
Projects/HKH品牌/
├── brand.md              ← 品牌资产总文档（核心，所有输出以此为准）
├── 参考/                 ← 别人的素材，仅参考
├── 输入/                 ← 需求/brief/原始素材
├── 输出/                 ← Kevin 的成品，按渠道分
├── 资产/                 ← 定期沉淀的品牌资产
└── 草稿/                 ← 未完成/废弃的（不进 Git）
```

## 命名规范

### 输入/

按需求批次建文件夹：`YYYY-MM-DD-需求描述/`

```
✅ 2026-06-22-详情页需求/
✅ 2026-06-23-社媒素材/
❌ 详情页需求/
```

### 输出/

单文件：`YYYY-MM-DD-项目名-内容描述.扩展名`

```
✅ 2026-06-22-HKH详情页-主图.jpg
✅ 2026-06-23-HKH社媒-小红书-产品图.jpg
❌ 主图.jpg
❌ 最终版.jpg
```

### 参考/

`YYYY-MM-DD-来源-内容描述.扩展名`

```
✅ 2026-06-22-竞品-兰蔻详情页.jpg
✅ 2026-06-23-灵感-岩玫瑰质感.jpg
❌ 参考图1.jpg
```

### 资产/

沉淀下来的精选内容，按主题建文件夹或文件，不加日期前缀。

```
✅ 资产/视觉库/岩玫瑰质感/
✅ 资产/视觉库/胶囊产品/
✅ 资产/文案库.md
```

## Workflow

### 日常接单（Kevin + AI）

| 步骤 | 谁做 | 做什么 |
|------|------|--------|
| 1. 收需求 | **Kevin** | 把甲方给的文案/素材扔进 `输入/YYYY-MM-DD-需求描述/` |
| 2. 理需求 | **AI** | 读输入文件夹，提取卖点/调性/必须元素，写到 `输入/YYYY-MM-DD-需求描述/需求摘要.md` |
| 3. 找参考 | **Kevin** | 看到好的参考图，扔 `参考/` 对应渠道，按 `YYYY-MM-DD-来源-描述` 命名 |
| 4. 出图 | **Kevin** | 设计产出，成品扔 `输出/` 对应渠道 |
| 5. 归图 | **AI** | 出图后检查命名是否规范，不规范则按 `YYYY-MM-DD-项目名-内容` 修正 |
| 6. 交付 | **Kevin** | 把成品发给甲方/业务方 |

### 定期沉淀（Kevin 触发，AI 执行）

| 频率 | 谁触发 | AI 做什么 |
|------|--------|-----------|
| 每周/月 | **Kevin 说"整理资产"** | 扫描 `输出/` 所有渠道，按主题推荐分类到 `资产/视觉库/` |
| 每周/月 | **Kevin 说"整理文案"** | 扫描 `输入/` 各批次里的文案，汇总精华到 `资产/文案库.md` |
| 项目结束 | **Kevin 说"复盘"** | 扫整个项目，写总结：做了什么、哪些效果好、踩了什么坑 |

### 命名规范速查

```
输入/      → YYYY-MM-DD-需求描述/         例: 2026-06-22-详情页需求/
输出/      → YYYY-MM-DD-项目名-内容.扩展名  例: 2026-06-22-HKH详情页-主图.jpg
参考/      → YYYY-MM-DD-来源-描述.扩展名    例: 2026-06-22-竞品-兰蔻详情页.jpg
资产/      → 按主题，不加日期              例: 资产/视觉库/岩玫瑰质感/
```
