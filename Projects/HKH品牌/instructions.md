# HKH 品牌项目结构

```
Projects/HKH品牌/
├── brand.md              ← 品牌资产总文档（核心，所有输出以此为准）
├── 资产/                 ← 共用：视觉库、文案库等定期沉淀
├── 参考/                 ← 共用：别人的素材，仅参考
│
├── YYYY-MM-DD-项目名/    ← 具体项目（日期在最外层）
│   ├── 01-输入/          ← 原始需求/brief/素材
│   ├── 02-中间过程/      ← AI 生图过程的产物（备选图、中间图）
│   └── 03-输出/          ← 最终成品
│
└── ...更多项目/
```

## 共用层

`brand.md`、`资产/`、`参考/` — 所有项目共用，不重复建。

### 资产/

沉淀下来的精选内容，按主题建文件夹或文件，不加日期前缀。

```
✅ 资产/视觉库/岩玫瑰质感/
✅ 资产/视觉库/胶囊产品/
✅ 资产/文案库.md
```

### 参考/

别人的素材，仅用于参考风格/构图/配色。按编号顺序命名，编号后标注渠道。

```
✅ 01-电商.jpg
✅ 02-电商.jpg
✅ 03-社媒.jpg
❌ 参考图1.jpg
```

渠道标识：电商、社媒、官网、视频、线下

## 项目层

每个项目一个独立文件夹，日期在最外层，内部按序号排列。

### 命名规范

```
项目文件夹 → YYYY-MM-DD-项目名/              例: 2026-06-22-小红书-岩玫瑰溯源/
01-输入/   → 保持原始文件名或简短描述           例: 小红书-岩玫瑰溯源.xlsx / 配图/
02-中间过程 → NN-描述.扩展名                  例: 01-岩玫瑰海岸-v1.jpg
03-输出/   → NN-描述.扩展名                   例: 01-岩玫瑰-主图.jpg
```

项目内部的输入/中间过程/输出**不加日期前缀**，日期已体现在项目文件夹名上。

## Workflow

### 日常接单（Kevin + AI）

| 步骤 | 谁做 | 做什么 |
|------|------|--------|
| 1. 收需求 | **Kevin** | 建 `YYYY-MM-DD-项目名/`，把文案/素材扔进 `01-输入/` |
| 2. 理需求 | **AI** | 读 `01-输入/`，提取卖点/调性/必须元素，写需求摘要 |
| 3. 找参考 | **Kevin** | 看到好的参考图，扔 `参考/`，按 `NN-渠道` 编号命名 |
| 4. 生图 | **Kevin** | AI 生图，产物放 `02-中间过程/` |
| 5. 定稿 | **Kevin** | 选中的成品放 `03-输出/` |
| 6. 交付 | **Kevin** | 把成品发给甲方/业务方 |

### 定期沉淀（Kevin 触发，AI 执行）

| 频率 | 谁触发 | AI 做什么 |
|------|--------|-----------|
| 每周/月 | **Kevin 说"整理资产"** | 扫描所有项目的 `03-输出/`，按主题推荐分类到 `资产/视觉库/` |
| 每周/月 | **Kevin 说"整理文案"** | 扫描所有项目的 `01-输入/` 里的文案，汇总精华到 `资产/文案库.md` |
| 项目结束 | **Kevin 说"复盘"** | 扫整个项目，写总结：做了什么、哪些效果好、踩了什么坑 |
