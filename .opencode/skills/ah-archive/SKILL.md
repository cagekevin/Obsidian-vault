# 项目归档引导技能

归档目标：在“项目结束”与“知识沉淀”之间建立闭环，避免项目结束后资料散落或经验丢失。

## 启动即执行

1. 读取 `{{metaPaths.memory}}/STATUS.md`，检查是否有“待归档项目”提示。
2. 读取 `{{notePaths.activeProjects}}/`，生成可选项目清单（只使用真实数据）。
3. 若用户未指定项目，先让用户确认要归档的项目编号或名称。

## 流程锁定规则（高优先级）

- 用户在复盘中提到“阅读/卡片/整理”等词时，视为归档输入，不自动切技能。
- 只有两种情况允许中断：
  - 用户明确输入其他命令（如 `/ah-card`）
  - 用户明确说停止归档流程
- 如果用户选择“现在处理可提炼内容”，先完成归档动作，再建议进入 `/ah-card`。

## 归档主流程

1. 确认项目：项目名、领域、起止时间、原目标。
2. 检查完成度：读取成功标准并确认是否允许“部分完成归档”。
3. 引导复盘：收集成果、收获、教训、可复用经验。
4. 标记提炼项：选择“现在处理 / 先归档后处理 / 跳过”。
5. 更新项目主页：写入归档区块并保留可追溯链接。
6. 更新 YAML 状态：`状态: 已完成` 与 `完成日期`。
7. 确认归档层级：优先复用用户已有层级；若无现有层级则先与用户确认新层级。
8. 移动项目目录：从 `{{notePaths.activeProjects}}/` 移动到确认后的归档层级路径。
9. 更新领域页：将项目从“活跃项目”迁移到“历史归档”。
10. 输出归档报告：列出完成动作、归档位置和后续待办。

## 执行约束

- 归档是移动，不是删除。
- 任何路径操作前，先确认目标目录存在；不存在则先创建。
- 归档层级推荐必须先参考用户当前 `{{notePaths.archive}}/` 的实际结构，不直接套用固定模板。
- 若用户是首次归档（无现有层级），必须先给出候选层级方案并与用户确认后再落盘。
- 归档完成后必须同步更新：
  - 项目主页
  - 领域页
  - 最终汇报

## 快速归档模式

当用户时间有限时，允许跳过深度复盘，但仍必须完成：

1. 项目确认
2. 完成度结论（完成/部分完成）
3. 项目主页 + YAML 更新
4. 归档层级确认（沿用现有或新建）
5. 目录移动
6. 领域页同步
7. 归档结果汇报

## 输出标准

- 复盘内容优先保留用户原话，再做轻量整理。
- 待提炼内容必须明确状态：
  - 已处理
  - 待处理（推荐 `/ah-card`）
  - 跳过
- 归档结果中必须说明层级来源：沿用已有层级或本次新建层级。
- 最终汇报必须可执行、可追溯，包含准确路径。

## 检查清单

- [ ] 已确认归档项目
- [ ] 已完成完成度检查（含未完成分支决策）
- [ ] 已完成复盘或快速归档最小集
- [ ] 已标记可提炼内容去向
- [ ] 已更新项目主页与 YAML
- [ ] 已确认归档层级来源（沿用/新建）
- [ ] 已完成目录移动
- [ ] 已更新领域页项目位置
- [ ] 已输出归档完成报告
