# 周回顾引导技能

目标：完成“本周复盘 + 残留清理 + 下周计划”闭环；先产出下周框架并与用户确认，再写入确认版下周计划。

## 启动即执行

1. 读取本周每日笔记（周一到周日）：`{{notePaths.dailyNotes}}/`。
2. 读取上周周记：`{{notePaths.weeklyReviews}}/{{上周年份}}-W{{上周周数}}.md`（若存在）。
3. 读取上月月记：`{{notePaths.monthlyReviews}}/{{上月年份}}-{{上月MM}}.md`（若存在，用于提取“下个月的方向/计划”作为本月计划基线）。
4. 读取状态文件：`{{metaPaths.memory}}/STATUS.md` 的“本周待处理”。
5. 确认用户时间：完整回顾或快速回顾。

## 流程锁定规则（高优先级）

- 用户在周回顾中提到“阅读/项目/卡片”等词，默认视为周回顾输入，不自动切技能。
- 仅在两种情况允许切换：
  - 用户明确输入其他命令（如 `/ah-card`）
  - 用户明确要求停止本次周回顾
- 调用 `ah-card` 处理记录后，必须返回 `ah-week` 继续本轮流程。
- 流程完成标志：
  - 已完成“上周计划 vs 本周执行”对照分析
  - 残留记录与“本周待处理”已处理
  - 下周计划已写入本周周记 `## 下周计划`
  - 输出了完成总结

## 重要提醒

- 禁止使用示例数据；所有内容必须来自真实文件与用户输入。
- 周回顾不是报表展示，最终周记使用自然语言，避免空洞术语。
- 下周计划不能凭空生成，必须先完成“上周计划执行对照”。
- 若用户时间不足，走快速回顾模式，但不能跳过下周计划确认。

## 触发机制

- 主动触发：用户输入“周回顾 / 本周总结 / 周复盘”等。
- 钩子触发：`ah-note` 在周末提醒后，用户进入 `/ah-week`。

## 主流程

1. 阶段一：汇总本周执行数据（任务、记录、项目推进）。
2. 阶段二：对照上周 `## 下周计划` 与本周执行结果。
3. 阶段三：处理本周残留记录 + STATUS“本周待处理”。
4. 阶段四：引导反思（成就/挑战/洞察/改进）。
5. 阶段五：基于“上周差异 + 上月月记中的下个月方向（本月计划基线）”生成下周框架建议（待确认）。
6. 阶段六：用户确认并补充，形成确认版下周计划。
7. 阶段七：写入周记并完成汇报。

## 关键执行规则（确定性）

- 周记路径固定：`{{notePaths.weeklyReviews}}/{{YYYY}}-W{{周数}}.md`。
- 上周计划基线固定：读取上周周记的 `## 下周计划`（若不存在则明确说明“无上周计划基线”）。
- 若当前周与上周跨年，必须优先定位“上一年最后一周”的周记作为上周基线来源，避免年份错位读取。
- 本月计划基线固定：读取上月月记的“下个月的方向/计划”区块（若不存在则明确说明“无本月计划基线”）。
- 对照分析固定输出三类状态：`完成 / 部分完成 / 未完成`，并记录主要原因。
- 残留记录处理固定四选一：
  1. 转永久笔记（调用 `ah-card`）
  2. 转任务
  3. 标记已处理
  4. 延期到 `/ah-inbox`
- STATUS“本周待处理”任务必须逐条定去向：
  - 纳入下周计划
  - 移到“以后再说”
  - 删除/取消
- 下周计划确认必须使用固定提示：
  - `基于上周未完成项和本月目标，我建议下周先做：{{下周计划建议}}。您看这样是否可行？还有什么需要补充的吗？`
- 分支规则：
  - 用户确认可行且无补充 -> 使用建议框架作为确认版下周计划。
  - 用户提出补充/修改 -> 与建议框架合并去重后形成确认版下周计划。
  - 用户明确不可行 -> 基于反馈重生成一版框架并再次确认。
- 下周计划写入位置固定：本周周记同一文件的 `## 下周计划`，仅写入用户确认后的版本。
- 若发生写入失败：报告失败位置 -> 停止后续写入 -> 修复后再继续（禁止静默失败）。

## 快速回顾模式（时间不足）

触发：用户明确“时间不够/快速收尾”。

最小闭环：

1. 完成“上周计划 vs 本周执行”最小对照（覆盖关键项）。
2. 提取“上月月记中下个月方向（本月计划基线）”的剩余目标，生成下周框架建议并征求确认。
3. 处理 STATUS“本周待处理”全部条目去向。
4. 写入确认版下周计划到本周周记。

## 输出标准

- 必须输出：
  - 本周回顾结论
  - 上周计划执行对照摘要（完成/部分完成/未完成）
  - 残留处理统计（转笔记/转任务/已处理/延期）
  - 本周待处理去向统计（纳入下周/以后再说/删除）
  - 下周框架建议与确认结果（通过/补充/重生成）
  - 下周计划结果（确认版）
  - 周记保存路径

## 检查清单

- [ ] 已读取本周每日笔记
- [ ] 已读取上周周记 `## 下周计划`（或明确无基线）
- [ ] 已读取上月月记的“下个月方向/计划”区块（或明确无本月计划基线）
- [ ] 已完成“上周计划 vs 本周执行”对照
- [ ] 已处理残留记录与 STATUS“本周待处理”
- [ ] 已完成下周框架建议并与用户确认
- [ ] 已写入确认版本周周记 `## 下周计划`
- [ ] 已输出可追溯总结
