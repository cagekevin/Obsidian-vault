# 年度回顾引导技能

目标：完成“全年复盘 + 年度计划对照 + 新年计划确认”闭环；先给新年框架建议，再与用户确认后落盘。

## 启动即执行

1. 读取全年月记：`{{notePaths.monthlyReviews}}/{{YYYY}}-*.md`。
2. 读取上一年年记：`{{notePaths.yearlyReviews}}/{{YYYY-1}}.md`（若存在，用于提取“新一年的方向/重点计划”作为本年计划基线）。
3. 读取本年年记：`{{notePaths.yearlyReviews}}/{{YYYY}}.md`（若存在，用于增量更新而非覆盖）。
4. 确认用户时间：完整回顾或快速回顾。

## 流程锁定规则（高优先级）

- 用户在年回顾中提到“阅读/项目/卡片”等词，默认视为回顾输入，不自动切技能。
- 仅在两种情况允许切换：
  - 用户明确输入其他命令（如 `/ah-card`）
  - 用户明确要求停止本次年回顾
- 调用 `ah-card` 处理年度洞察后，必须返回 `ah-year` 继续本轮流程。
- 流程完成标志：
  - 已完成“年初计划 vs 年度执行”对照
  - 已完成年度反思
  - 已完成新年框架确认并写入当年年记（回顾区块 + 新一年的方向区块）
  - 已输出完成总结

## 重要提醒

- 禁止使用示例数据；所有内容必须来自真实文件与用户输入。
- 年回顾不是数据报表，最终年记使用自然语言，像写给未来自己的年度信。
- 新年计划不能直接生成，必须先给框架建议并与用户确认。
- 若上一年年记缺少“新一年的方向/重点计划”区块，要明确说明“无本年计划基线”，再继续回顾。

## 触发机制

- 主动触发：用户输入“年回顾 / 年度总结 / 年终复盘”等。
- 钩子触发：`ah-note` 在年末提醒后，用户进入 `/ah-year`。

## 主流程

1. 阶段一：汇总全年执行数据（优先月记，缺失时补读周记/关键日记）。
2. 阶段二：对照年初计划与全年执行结果。
3. 阶段三：引导年度反思（成就/挑战/成长/关系/价值观）。
4. 阶段四：生成人生维度的新年框架建议（待确认）。
5. 阶段五：用户确认并补充，形成确认版新年计划。
6. 阶段六：写入当年年记（同一文件内包含“本年回顾 + 下一年计划”）。
7. 阶段七：输出完成汇报与下一步入口。

## 关键执行规则（确定性）

- 年记文件路径固定：`{{notePaths.yearlyReviews}}/{{YYYY}}.md`。
- 年初计划基线固定：读取上一年年记 `{{notePaths.yearlyReviews}}/{{YYYY-1}}.md` 中“新一年的方向/重点计划”区块（若不存在，明确“无本年计划基线”）。
- 对照分析固定输出三类状态：`完成 / 部分完成 / 未完成`，并记录主要原因。
- 新年框架建议必须同时包含：
  - 本年未完成且应延续的事项
  - 本年反思得到的新方向
  - 可执行的启动动作
- 新年计划确认必须使用固定提示：
  - `基于今年未完成项和你的年度反思，我建议新一年先按这个框架推进：{{新年框架建议}}。您看这样是否可行？还有什么需要补充的吗？`
- 分支规则：
  - 用户确认可行且无补充 -> 使用建议框架作为确认版新年计划。
  - 用户提出补充/修改 -> 与建议框架合并去重后形成确认版计划。
  - 用户明确不可行 -> 基于反馈重生成一版框架并再次确认。
- 写入规则：
  - 年度回顾写入 `{{YYYY}}.md` 的回顾区块。
  - 确认版新年计划写入 `{{YYYY}}.md` 的“新一年的方向”区块。
- 若发生写入失败：报告失败位置 -> 停止后续写入 -> 修复后再继续（禁止静默失败）。

## 快速回顾模式（时间不足）

触发：用户明确“时间不够 / 快速收尾”。

最小闭环：

1. 完成“年初计划 vs 全年执行”最小对照（至少关键目标对照）。
2. 产出新年框架建议并征求确认。
3. 写入 `{{YYYY}}.md`（同一文件内更新精简版年度回顾与确认版新年计划）。

## 输出标准

- 必须输出：
  - 年度回顾结论
  - 年初计划执行对照摘要（完成/部分完成/未完成）
  - 新年框架建议与确认结果（通过/补充/重生成）
  - 新年计划结果（确认版）
  - 已更新文件路径（当年年记同一文件）
- 若有高价值洞察，提示可用 `/ah-card` 转永久笔记。

## 检查清单

- [ ] 已读取全年月记（缺失部分已补读）
- [ ] 已读取上一年年记中的本年计划基线（或明确无基线）
- [ ] 已完成“年初计划 vs 年度执行”对照
- [ ] 已完成年度反思引导
- [ ] 已完成新年框架建议并与用户确认
- [ ] 已写入当年年记（年度回顾 + 下一年计划，同一文件）
- [ ] 已输出可追溯总结
