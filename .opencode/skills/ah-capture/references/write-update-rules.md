# ah-capture 写入与更新规则

## 1) 今日日记文件

目标文件：`{{notePaths.dailyNotes}}/{{YYYY-MM-DD}}.md`

约束：捕获与补充内容统一落在当日日记（`## 记录` / `## 任务` / `## 明日计划`）。

若不存在：

1. 创建最小兼容结构：

```markdown
# {{YYYY-MM-DD}} 星期X

## 今天最重要的事
写今天唯一最重要的一件事，要求可执行且可在今天推进。

## 任务
写今天任务清单，第一条对齐“今天最重要的事”。

## 记录
写白天发生的重要进展、想法或事件。

## 晚间回顾
写做得好的地方、可改进处和明天最想推进的一件事。

## 明日计划
写明天预计推进的 1-3 件重要事项，供次日承接。
```

2. 再执行写入。

若已存在但为旧版区块（如 `## 🌅 今日计划`、`## 📝 今日记录`），先补齐标准五区块再写入。

## 2) 记录区写入

写入到 `## 记录`：

```markdown
- {{HH:mm}} {{用户原话}}
```

规则：

- 语音输入仅去除语气词（嗯、啊、那个）
- 保留原话，不重写观点

## 3) 新任务写入

写入到 `## 任务`：

```markdown
- [ ] {{任务内容}}
```

规则：

- 去掉“帮我加个任务”等提示词，仅保留任务正文
- 不擅自补截止日期

## 4) 明日计划写入

写入到 `## 明日计划`：

```markdown
- [ ] {{任务内容}}
```

规则：

- 仅在用户明确“明天/明日/明早”语义时写入该区块
- 与 `## 任务` 分开维护，避免混入今日执行清单

## 5) 任务完成更新

在 `## 任务` 中匹配待办项并更新：

```markdown
- [ ] 需求文档初稿
```

更新为：

```markdown
- [x] 需求文档初稿
```

匹配策略：

1. 精确匹配
2. 包含匹配
3. 语义近似匹配

若有多个候选，列出序号让用户确认。

若无候选，追问：

`没找到对应任务，要把这条作为已完成记录写到“记录区”吗？`

## 6) 写入后校验

每次写入后必须校验：

- 目标区块存在
- 新条目已落盘
- 返回内容与写入内容一致

## 7) 错误恢复

若写入失败：

1. 重读文件并重试一次
2. 仍失败则报告失败原因
3. 不做静默失败
