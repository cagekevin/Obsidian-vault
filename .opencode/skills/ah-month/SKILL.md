# 月度回顾引导技能

目标：完成“月度复盘 + 计划对照 + 残留清零 + 下月规划”闭环，并保持与年度计划同向推进。

## 启动即执行

1. 读取本月周记与必要的每日笔记（缺周记时补读）。
2. 读取上月月记：`{{notePaths.monthlyReviews}}/{{YYYY}}-{{上月MM}}.md`（若存在）。
3. 读取年度计划基线：`{{notePaths.yearlyReviews}}/{{YYYY-1}}.md` 的“新一年的方向/重点计划”区块（若存在）。
4. 读取 `{{metaPaths.memory}}/STATUS.md` 的“以后再说”缓冲区。
5. 确认用户时间：完整回顾（约 1 小时）或快速回顾（最小闭环）。

## 流程锁定规则（高优先级）

- 用户在回顾中提到“阅读/项目/卡片”等词，默认作为月回顾输入，不自动切技能。
- 仅在两种情况允许切换：
  - 用户明确输入其他命令（如 `/ah-card`）。
  - 用户明确要求停止本次月回顾。
- 调用 `ah-card` 处理记录后，必须回到 `ah-month` 继续，直到本阶段处理完成。
- 流程完成标志：数据汇总完成 + 本月残留清零 + “以后再说”清空 + 下月计划落盘 + 完成汇报。

## 重要提醒

- 禁止使用示例数据，所有内容必须来自真实笔记与用户回答。
- 月回顾不是数据报表，最终月记使用自然语言，不堆砌术语。
- 用户表达优先，AI 负责提问、整理与结构化，不替用户虚构结论。
- “以后再说”在月回顾中必须逐条定去向，不允许残留。

## 触发机制

- 主动触发：用户输入“月回顾 / 月度复盘 / 月总结”等。
- 钩子触发：`ah-note` 在月底提醒，用户进入 `/ah-month` 执行。

## 主流程

1. 阶段一：数据汇总（优先周记，缺失时补读每日笔记）。
2. 阶段二：对照“上月计划 vs 本月执行”结果。
3. 阶段三：处理残留记录 + “以后再说”缓冲（本月清零环节）。
4. 阶段四：成就回顾（用户互动）。
5. 阶段五：领域检查（用户互动）。
6. 阶段六：目标进度检查（用户互动）。
7. 阶段七：战略反思（用户互动）。
8. 阶段八：下月规划（需对齐年度计划）。
9. 阶段九：生成月记并落盘。
10. 阶段十：完成汇报与下一步提示。

## 关键执行规则（确定性）

- 阶段一：按 `references/数据汇总模板.md` 获取数据并转自然叙述，不输出生硬统计表。
- 阶段二：若上月月记存在“下个月的方向/计划”区块，必须输出对照结果：`完成 / 部分完成 / 未完成` + 原因；若不存在，明确“无上月计划基线”。
- 阶段三：本月每日笔记中未处理记录 + STATUS“以后再说”一起处理；转永久笔记必须调用 `ah-card`。
- 阶段三完成后：将 STATUS 的“以后再说”条目清零（保留区块结构），并回写所有记录标记。
- 阶段四到七：必须通过提问获取用户原话，AI 不预设“标准答案”。
- 阶段八：下月计划必须显式标注“与年度计划基线的对齐点”；若年度计划基线缺失，明确“无年度计划基线”。
- 阶段八：下月计划写入本次月记（同一文件，不拆分到新模板）。
- 阶段九：月记保存到 `{{notePaths.monthlyReviews}}/{{YYYY}}-{{MM}}.md`。

## 快速回顾模式（时间不足）

触发：用户明确“时间不够 / 想快速收尾”。

最小闭环：

1. 完成阶段二与阶段三（上月计划对照 + 残留与“以后再说”清零）。
2. 用 2-3 个问题完成“上月计划对照 + 年度对齐的下月规划”。
3. 生成精简月记并给出后续补充入口。

## 输出标准

- 必须输出：
  - 本月回顾结论（亮点 + 问题 + 下月方向）
  - 上月计划执行对照摘要（完成/部分完成/未完成）
  - 下月计划与年度计划对齐点（或无年度基线说明）
  - 记录处理统计（转笔记/转任务/已处理）
  - “以后再说”处理统计（纳入下月/已完成/删除）
  - 月记保存路径
- 若有关键洞察，提示可用 `/ah-card` 继续深挖。

## 检查清单

- [ ] 已完成本月数据汇总（优先周记）
- [ ] 已完成“上月计划 vs 本月执行”对照（或明确无基线）
- [ ] 已完成残留记录处理与回写标记
- [ ] 已清空 STATUS 的“以后再说”缓冲
- [ ] 已完成用户互动式反思与“对齐年度计划”的下月规划
- [ ] 已生成并保存月记
- [ ] 已输出可追溯总结
