# 月度数据汇总指引

> 本文档指导 AI 在月度回顾时如何收集数据，并以**自然叙述**的方式融入月记。
> 不要生成统计表格或列表——用讲故事的方式描述这个月发生了什么。

---

## 数据获取

AI 需要先执行以下命令获取原始数据，然后将数据转化为自然语言融入月记正文。

```bash
# 获取当前日期信息
current_year=$(date +%Y)
current_month=$(date +%m)

# 统计本月永久笔记
ls "{{notePaths.permanentNotes}}/"

# 统计本月日记
ls "{{notePaths.dailyNotes}}/" | grep "${current_year}-${current_month}"

# 统计本月周记
ls "{{notePaths.weeklyReviews}}/"

# 获取文献笔记
ls "{{notePaths.literatureNotes}}/"

# 获取主题笔记
ls "{{notePaths.topicNotes}}/"

# 获取领域页
ls "{{notePaths.domainPages}}/"

# 读取缓冲区状态
cat "{{metaPaths.memory}}/STATUS.md"

# 获取活跃项目
ls "{{notePaths.activeProjects}}/" 2>/dev/null
```

---

## 叙述化指引

拿到数据后，不要直接贴统计表，用自然语言写进月记正文。

- 知识积累段：写本月新增笔记、阅读整理和主题推进情况。
- 行动复盘段：写执行节奏、完成度和主要阻塞原因。
- 缓冲区段：写“以后再说”当前规模与下月处理策略。

写法要求：

- 只使用本次实际扫描到的数据。
- 不写虚构书名、数字或示例项目。
- 数据只做支撑，重点是结论与下月动作。

---

## 核心原则

1. **数据是素材，不是结果** — 统计数字是为了帮用户看清全貌，不是为了展示给谁看
2. **像朋友聊天一样写** — "这个月写了几篇笔记"比"本月新建永久笔记：N篇"更自然
3. **只提有意义的数据** — 如果某项数据没什么变化或不值一提，就不需要专门说
4. **穿插在正文中** — 不要单独开一个"数据统计"章节，把数据融入各个回顾环节
