# 5个为什么（执行版）

## 何时使用

- 问题反复出现，怀疑只处理了表面现象。
- 需要从“症状”追到“根因”。
- 复盘中要形成可行动改进点。

## 不适用

- 多因并发的复杂系统（需配合因果回路）。
- 紧急止血场景（先止血再追因）。

## 最小流程（5步）

1. 明确问题（可观察现象）。
2. 连续追问“为什么”3-5次。
3. 每次只沿一条主因链，不分叉。
4. 到达“可行动根因”后停止。
5. 写出防复发动作并验证可执行性。

## 引导问题（可直接问）

- 这个现象为什么会发生？
- 这个原因背后还有什么原因？
- 如果这条根因被解决，原问题会明显缓解吗？

## 与笔记工作流结合

- `ah-review`：把“今日问题”追到根因，再写明日改进动作。
- `ah-card`：卡片写“问题链条 + 根因 + 对策”。

## 输出模板（3-8行）

```text
模型：5个为什么
问题：{{problem}}
追问链：{{why_chain}}
根因：{{root_cause}}
防复发动作：{{prevention_action}}
```

## 退出条件

- 已得到“可行动根因”。
- 用户确认动作可执行并准备落盘。
