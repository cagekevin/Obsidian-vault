# 因果回路（执行版）

## 何时使用

- 问题反复回弹，单点修复无效。
- 需要识别强化回路或平衡回路。

## 不适用

- 单点故障且原因清晰。

## 最小流程（4步）

1. 定义关键变量（3-6个）。
2. 画出变量因果方向。
3. 标记强化环/平衡环。
4. 选择可干预点并给动作。

## 引导问题（可直接问）

- 哪些变量会互相影响并形成循环？
- 你最有把握先改变哪个变量？

## 输出模板（3-8行）

```text
模型：因果回路
核心变量：{{variables}}
主要回路：{{loops}}
杠杆点：{{leverage_point}}
干预动作：{{intervention}}
```
