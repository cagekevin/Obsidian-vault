# 第一性原理（执行版）

## 何时使用

- 现有做法卡住，想打破惯例。
- 需要把方法论写成“可迁移”的卡片。
- 需要区分“事实”与“习惯性假设”。

## 不适用

- 小问题已有成熟解法，且无需创新。
- 用户只要快速行动，不需要重构问题。

## 最小流程（3步）

1. 列出当前假设（哪些是“大家都这么做”）。
2. 拆到基础事实（不可省略的约束）。
3. 从基础事实重新组合方案。

## 引导问题（可直接问）

- 你现在的做法依赖了哪些默认假设？
- 这里哪些是硬约束，哪些只是惯例？
- 如果只保留硬约束，你会如何重做一次？

## 与笔记工作流结合

- `ah-read`：把作者方案拆成“事实层/假设层”。
- `ah-card`：卡片标题写结论，正文写“事实 -> 推导 -> 适用边界”。

## 输出模板（3-8行）

```text
模型：第一性原理
问题：{{problem}}
关键假设：{{assumptions}}
基础事实：{{facts}}
重构方案：{{rebuild_solution}}
下一步：{{test_or_record}}
```

## 退出条件

- 已明确“假设清单 + 基础事实 + 新方案”。
- 用户确认先记录，后续再验证。
