# 苏格拉底式提问（执行版）

## 何时使用

- 观点模糊、论证链条不清。
- 需要在不直接下结论的前提下引导思考。
- 讨论中出现“感觉对，但说不出为什么”。

## 不适用

- 用户只要快速结论，不愿被追问。
- 纯事务执行，不涉及观点检验。

## 最小流程（4类问题）

1. 澄清定义：你说的 `X` 具体指什么？
2. 检查假设：这个说法依赖哪些前提？
3. 检查证据：支持/反对证据分别是什么？
4. 检查后果：如果按此行动，会带来什么后果？

## 引导问题（可直接问）

- 你这里的关键词是如何定义的？
- 如果这个前提不成立，结论还成立吗？
- 有什么证据会让你修正这个观点？
- 这个观点最可能在哪种情境下失效？

## 与笔记工作流结合

- `ah-read`：把作者论断拆成“定义-假设-证据-后果”。
- `ah-card`：卡片可加“反驳/限制条件”字段。

## 输出模板（3-8行）

```text
模型：苏格拉底式提问
被检验观点：{{claim}}
关键假设：{{assumption}}
证据评估：{{evidence_assessment}}
潜在反例：{{counterexample_or_none}}
当前结论：{{revised_conclusion}}
```

## 退出条件

- 观点已从“模糊”变为“可检验”。
- 用户说“先记下来/先到这里”。
