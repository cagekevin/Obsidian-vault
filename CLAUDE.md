# 第二大脑入口

我是 Kevin。这个 vault 不只是我的笔记库——它是你了解我这个人、跟上我的思维方式的入口。AI 每次运行必须先读这个文档。

---

## 首次启动流程

你第一次进入这个 vault 时，按这个顺序做，不要跳步：

1. **读本文件（CLAUDE.md）** — 了解我是谁、文件夹地图、核心规则
2. **去 `Context/` 读所有文件** — 了解我的身份、目标、品牌、风格、做事方式
3. **去 `Daily Notes/` 读最近几天的日志** — 了解我最近在做什么
4. **等我给你第一个任务** — 在这之前不要自作主张做任何事

之后每次新 session 只做第 1 步和第 3 步（第 2 步 Context/ 只需读一次）。

## 文件夹地图——我的世界是怎么组织的

每个文件夹都反映了我生活的一个侧面。**你按这个地图导航，不要扫整个 vault。**

| 文件夹 | 这是关于我的什么 | 你要怎么做 |
|--------|-----------------|-----------|
| `Context/` | **我是谁** — 我的身份、目标、品牌、风格 | **每次 session 启动先读这里所有文件** |
| `Skills/` | **我的工作方法论** — 自动化流程和最佳实践 | 我需要你做事时，**来这里找对应技能的 SKILL.md** |
| `Reading/` | **我的知识来源** — 我看过的书、受过的启发 | 我导入读书高亮后，**你来这里整理成规范笔记** |
| `Clippings/` | **我关注的世界** — 我在网上保存的有价值内容 | 我需要引用之前保存的内容时，**来这里找** |
| `Daily Notes/` | **我今天的脑子** — 我每天在想什么、卡在哪里 | **新 session 先读最近 3 天的日志**，跟上我的节奏 |
| `Projects/` | **我做过的项目** — 我的实战经验 | 开始或继续项目时，**来这里找项目入口文件** |
| `Wiki/` | **AI 构建的知识图谱** — 从原始资料提取的原子化概念页面 | 我放入新资料后，**你来这里创建/更新页面**；我提问时，**优先基于这里回答** |
| `Tools/` | **我的工具箱** — 我常用的实用脚本 | 我说"兔子"、"用 xx 工具"时，**来这里找对应脚本** |

**什么情况下不用读：**
- 日常聊天、简单问答 → 不用读任何文件夹
- 只需要查一个信息 → 先去 CLAUDE.md 判断去哪个文件夹，**只读那一层**

**具体场景举例：**
- 我要写视频脚本 → **先去读 `Context/brand.md`** 了解我的内容调性，**再去 `Skills/` 找视频创作技能**
- 我导入了 Apple Books 高亮 → **去 `Reading/` 把高亮内容整理成规范笔记**
- 我要继续昨天的项目 → **先读最近几天的 `Daily Notes/`** 了解进度，**再去 `Projects/` 找对应项目**
- 我突然有了灵感或碎碎念 → 记录到当天的 `Daily Notes/` 里，**你后续主动帮我整理和二次创作**
- 我想整理读书笔记 → **去 `Reading/` 文件夹看**
- 我不知道怎么处理 Token 消耗 → **去读 Daily Notes 的 instructions.md**，里面有渐进式披露的规则

## 你的工作流程

每次我让你做任何事，你按这个流程走：

1. **先读 CLAUDE.md**（这个文件）——了解我是谁、有哪些文件夹、规则是什么
2. **判断要去哪个文件夹** —— 根据任务类型决定
3. **读目标文件夹的 instructions.md** —— 了解这个文件夹的结构、命名规则、操作方法
4. **执行任务** —— 去对应的子文件夹读写文件

**AI 永远只读当下需要的那一层。** 不需要把整个笔记库全扫一遍。

---

## 核心规则

1. **进入任何文件夹前，必须先读该文件夹里的 instructions.md** — 这是强制规则。
2. **渐进式披露，按需索取** — 每次只给你当前任务所需的上下文。
3. **Daily Notes 接力** — 新 session 先读最近几天的 Daily Notes 了解进度。一天结束时你把今天做了什么、需要跟进的事情写进当天日志。
4. **不改 Context/** — Context/ 下的文件由我手动维护，你不要修改。
5. **本地文件 + 代码化 = AI 最擅长** — 能本地化、能用代码编译的东西，你就可以很好地操作。
6. **Folder as a App** — 在 AI 时代，很多问题用一个文件夹就能解决。文件按结构组织好，你就能当应用用。

## 依赖安装

首次使用这个 vault 时：

**Python 依赖（uv）**
`uv pip install -r requirements.txt`

**Node 依赖（pnpm）**
`pnpm install`

如果某个项目有特殊依赖，会在该项目目录下放自己的 `requirements.txt` 或 `package.json`。
