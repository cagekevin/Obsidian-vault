# AI 图像生成提示词 28 案例

> 来源：B站视频《我拆了OpenAI官方28个案例，发现提示词的核心根本不是词汇量》
> 作者：擎苍与爱
> 原始文件：`Clippings/raw/我拆了OpenAI官方28个案例，发现提示词的核心根本不是词汇量.md`

B站视频的文字稿，拆解了 OpenAI 官方 28 个图像生成案例，分为三个递进层次：生成（建世界）、编辑（画边界）、生产（融入真实商业流程）。虽然案例基于 GPT-4o，但提示词方法论和写法技巧是通用的，适用于各类图像生成模型。

## 核心方法论

- **产物先行** — 先定义交付物类型，再写提示词。不同的产物触发不同的精修和布局逻辑
- **变量与不变量分离** — 编辑任务的核心：先锁死不变量（人脸/身形/姿态/背景），再开放一个变量
- **小步迭代** — 不要让模型一次生成全部，先生成再编辑，每步只改一个变量
- **编辑三要素** — 变量（允许改什么）、不变量（必须保持什么）、关键句（只改什么，不要改变其他元素）
- **反向约束** — 明确说不要什么（"不要电影光"、"不要额外文字"、"不要重新设计角色"）
- **角色锚定** — 用详细的角色描述锁定角色一致性，后续场景复用角色描述

## 提示词模板八要素

1. 交付物 — 融资详情页？手机界面？教学图？
2. 受众 — 学生？客户？投资人？
3. 画布 — 横版？竖版？手机框？海报？
4. 文字 — 所有必须出现的文字用引号逐字引用
5. 层级 — 标题？图表？说明？角标？留白？
6. 视觉 — 白底？饱和度？无装饰噪声？
7. 数据 — 直接写入真实数据
8. 质量档次 — 小字/密集文字用高档

## 三个关键能力

- **文字可控性** — 广告牌/图表/UI文案可以用引号引用形式，不完美时收紧布局和文字约束
- **身份可守性** — 近景人像/试穿/人物入场可以通过输入输出锁定身份
- **结构可读性** — 信息图/科学图/生产力图片都可以写成有交付规格的结构化指令

## 质量档位选择策略

- 低档：快，适合批量生成、草图探索、方向初筛
- 中档：质量和速度平衡，适合多数工作流的默认起点
- 高档：细节最好但慢，适合高保真最终图、密集文字、近景人像
- **不要一上来就用高档**，先从低到高比较后再决定
