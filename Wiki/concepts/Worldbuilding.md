# Worldbuilding

世界观构建。为故事创建一个有内部逻辑、有规则、有历史的世界。

适用于科幻、奇幻、架空历史等有自己规则的故事。现实背景的故事可以简化。

---

## 6 大模块

### 1. 世界规则
- **和我们的世界有什么不同？**（3-5个核心差异）
- **和我们的世界有什么相同？**（重力？生物？钱？国家？）
- **科技/魔法水平**

### 2. 系统（魔法/超能力/科技）
如果有"魔法系统"、"超能力"、"特殊科技"，必须严格定义。

- **能做什么**
- **不能做什么**（关键！没有限制就没有戏剧）
- **使用代价**（使用者要付出什么？）
- **谁会用，谁不会**

**核心原则：没有限制的系统 = 作弊器。** 主角想干嘛干嘛，观众就不会紧张。

### 3. 世界历史
- **过去的大事件**（和剧情相关的，3个左右）：什么时候、发生了什么、后果是什么
- **故事开始时的当前状态**：政治、社会、自然/科技

### 4. 势力/派系
- **势力 A**：目标、方法、领导人
- **势力 B**：目标、方法、领导人
- **冲突点**：它们之间为什么打架

### 5. 关键地点
不是每个地点都写，只写重要的：
- 是什么地方
- 谁住在那/在那工作
- 剧情为什么需要这个地方
- 哪些场景在这里发生

### 6. 神器/物体地雷
每个重要的物体都要有 setup 和 payoff：

| 物体 | 在哪里出现（setup） | 在哪里发挥作用（payoff） | 代表什么 |
|---|---|---|---|
|  |  |  |  |

和 [[Chekhov's Gun|契诃夫之枪]] 是同一个原则。

---

## 隐藏的第 7 层：知识层级

谁知道什么，谁不知道什么，谁以为自己知道但错了。

| 角色 | 知道 | 不知道 | 以为知道（但错了） |
|---|---|---|---|
|  |  |  |  |

**很多冲突的本质就是信息差。** 观众知道的比角色多 → 悬念；角色知道的比观众多 → 神秘感。

---

## 世界观展示的原则

**不要通过对话解释世界观。**

❌ "你还记得吗，我们的魔法是这样运作的..."
✅ 展示系统的效果 → 主角像对待日常一样反应 → 观众自己明白"哦，原来这里这是正常的"

和 [[Show Don't Tell|展示而非讲述]] 是同一个底层逻辑。

---

## 核心思想

世界观不是背景板，是故事的一部分。
- 世界规则限制角色 → 产生戏剧
- 世界历史塑造角色 → 解释动机
- 势力冲突推动剧情 → 提供 stakes
- 信息差制造悬念 → 吸引观众

好的世界观不是"设定得越多越好"，而是"每个设定都在剧情里发挥作用"。

---

## 不仅适用于虚构故事

产品生态、品牌宇宙、游戏世界，都可以用这套方法：
- 产品的"规则"是什么？用户能做什么、不能做什么？
- 品牌的"历史"是什么？怎么走到今天的？
- 生态里有哪些"势力"？它们的关系是什么？
