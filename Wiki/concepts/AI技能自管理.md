# AI技能自管理

AI 自动创建、优化、归档技能的方法论。核心是三个机制：**技能自动生成**、**技能使用追踪**、**技能生命周期管理**。

---

## 机制一：技能自动生成

AI 做完复杂任务后，自动回顾操作步骤，按固定模板生成 SKILL.md。

**核心逻辑：**
1. AI 回顾刚才的操作步骤（或用户指定的来源）
2. 按硬性模板填空（不是自由发挥）
3. 模板结构固定：标题 → 触发条件 → 前置依赖 → 操作步骤 → 常见坑 → 验证方法
4. 禁止编造命令，所有指令必须来自实际操作验证

**可复用的规则：**
- skill 模板必须标准化，8 个固定章节，AI 不能自己发明结构
- frontmatter 必须有 name + description，校验不通过就报错
- 附属文件只能放在 references/templates/scripts/assets 四个目录下

---

## 机制二：技能使用追踪

记录每个 skill 的最后使用时间，作为生命周期管理的依据。

**追踪内容：**
- 最后使用时间
- 使用次数
- 修改次数
- 查看次数
- 状态（active / stale / archived）

---

## 机制三：技能生命周期管理

分两层：自动判断（不用 AI）+ AI 合并（可选）。

### 第一层：自动归档（纯逻辑，不需要 AI）

定时检查每个 skill 的使用时间：

| 条件 | 操作 |
|------|------|
| 30 天没用 | 标记为 stale（过时） |
| 90 天没用 | 自动归档到 .archive/ |
| stale 后又被用了 | 自动恢复为 active |
| 用户标记为 pin | 跳过所有自动操作 |

归档不是删除，是移到 `.archive/` 目录，随时可恢复。

### 第二层：AI 合并（可选，默认不开启）

把相关的多个小 skill 合并成一个"大类"skill：
- 比如 `pdf提取`、`docx提取`、`csv提取` → 合并成 `文档提取`
- 合并后保留每个子 skill 的独特内容作为章节
- 旧的 skill 归档

---

## 在你的系统里怎么用

这三个机制不需要装 Hermes 也能用：

1. **技能模板规范** — 写死 SKILL.md 的结构，AI 按规范填空
2. **使用时间追踪** — 记录每个 skill 最后使用时间，超过阈值自动归档
3. **归档可恢复** — 不是删文件，是移走，随时可以恢复
