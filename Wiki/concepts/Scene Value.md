# Scene Value

场景价值转换。每个场景都必须有价值的正负变化——价值进入场景时是一个状态，离开时是另一个状态。

**没有价值转换的场景 = 不是场景，只是描述。**

---

## W6 怎么用

### 写场景之前必须回答 4 个问题

1. **什么价值进入这个场景？**（比如：信任）
2. **进入时是什么状态？**（比如：团队信任领导者）
3. **离开时是什么状态？**（比如：团队怀疑领导者）
4. **怎么发生的？**（什么行动翻转了价值）

如果回答不上来 → 场景要么不需要，要么需要重新想。

---

### 常见价值对

| 正面 | 负面 |
|---|---|
| 控制 | 无助 |
| 信任 | 背叛 |
| 爱 | 执念 |
| 生 | 死 |
| 自由 | 束缚 |
| 纪律 | 混乱 |
| 团队 | 工具化 |
| 希望 | 绝望 |
| 知识 | 无知 |

---

### 审计时怎么检查

在因果性审计中，每个场景都要打标签：
- ⚠ [ЦЕННОСТЬ] — 场景没有推动价值变化，没有 +/–

如果一个场景没有价值变化，那它就是多余的。
