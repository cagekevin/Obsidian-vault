## 一、阿里云盘的"多盘"结构（2026-06-14 实测定论）

阿里云盘账号下不是单文件系统，而是**多个独立 drive（盘）**，每个 drive 有自己的 `root` 目录，互不交叉。CLI 默认只操作 **`default_drive_id`**。

用 `ali.v2_user_get()` 可以一次拿到关键盘 ID：

```python
u = ali.v2_user_get()
print(u.resource_drive_id)   # 资源盘
print(u.backup_drive_id)     # 备份盘
print(u.default_drive_id)    # 默认操作盘（通常=备份盘）
print(u.sbox_drive_id)       # SBox 加密备份盘
```

用 `ali.list_my_drives()` 看到全部盘（含 `drive_name`）：

```python
drives = ali.list_my_drives()
# id=545500515  name=resource    ← 资源盘（手动存文件的地方）
# id=693900515  name=Default     ← 备份盘（自动备份 + 你手动放的 AutoAI）
# id=793900515  name=Default     ← SBox 备份盘（locked，空的）
# id=4074105150 name=alibum      ← 相册盘
# id=3104105150 name=sharedAlbum ← 共享相册
# id=2782005150 name=note_drive  ← 笔记盘
```

### ⚠️ 关键陷阱（和直觉相反！）

| drive_id | `drive_name` | 阿里云盘界面叫法 | 实际用途 | `get_file_list('root')` 不加参数能看见吗 |
|---|---|---|---|---|
| `545500515` | `resource` | **资源盘**（App 里"我的文件"） | 你手动上传/移动的文件 | ❌ 不指定 `drive_id` 时看不到 |
| `693900515` | `Default` | **备份盘**（App 里"备份"） | 自动备份照片、你放的 AutoAI | ✅ aligo 默认看这个 |
| `793900515` | `Default` | **SBox 加密备份** | 需 PIN 的高级备份（空的） | ❌ |

> **一句话**：你 App 里看到的"资源库/我的文件" = `545500515`（resource）；
> aligo 默认 `get_file_list('root')` 不加 `drive_id` 看到的是 `693900515`（备份盘）。
> **默认操作的是备份盘，不是资源盘！**

### 怎么操作资源盘

```python
# 方式一：每次都传 drive_id
items = ali.get_file_list(parent_file_id='root', drive_id='545500515')

# 方式二：临时切换默认盘
ali.default_drive_id = '545500515'
items = ali.get_file_list('root')  # 现在看到的是资源盘

# 方式三：用 v2_user_get 动态获取
u = ali.v2_user_get()
ali.default_drive_id = u.resource_drive_id
```

## 二、分享文件下载完整流程（已验证稳定）

### 核心结论

- ✅ **`get_share_token(share_id)` 返回 `GetShareTokenResponse` 对象**（含 `share_id`、`share_token`、`expire_time` 等字段）
- ✅ **`share_file_save_all_to_drive(share_token_obj)`** 传的是对象，不是字符串——正常工作返回 `status=201`
- ❌ `get_share_link_download_url` **已失效**（`NotImplementedError`）
- ❌ `share_file_saveto_drive` **已失效**（参数兼容问题）

**唯一稳定路径：转存 → 等异步完成 → 在默认盘找到文件 → download_file**

### 完整脚本

```python
#!/usr/bin/env python3
"""分享链接 → 转存到自己的云盘 → 下载到本地"""
import sys, time
from aligo import Aligo

def download_from_share(share_url, share_pwd='', local_dir='.'):
    """
    share_url:  https://www.alipan.com/s/XXXX
    share_pwd:  提取码（无密码留空或 None）
    local_dir:  本地下载目录
    """
    ali = Aligo(level=30)

    # 1) 提取 share_id
    info = ali.share_link_extract_code(share_url)
    share_id = info.share_id
    share_pwd = share_pwd or info.share_pwd or ''

    # 2) 获取 share_token（返回的是对象，不是字符串！）
    share_token = ali.get_share_token(share_id, share_pwd=share_pwd)
    print(f'转存到默认盘 ({ali.default_drive_id})...')

    # 3) 触发异步转存（传 share_token 对象）
    result = ali.share_file_save_all_to_drive(share_token)
    print(f'转存状态: {result}')

    # 4) 等服务器异步处理（200KB 文件约 1-2 秒）
    time.sleep(2)

    # 5) 在默认盘根目录找转存过来的文件
    target_name = info.share_file_name or ''
    items = ali.get_file_list('root')  # 默认盘
    for f in items:
        if not f: continue
        # 转存后可能自动重命名（加 (1)）
        if target_name in f.name or target_name.split('.')[0] in f.name:
            print(f'找到: {f.name} ({f.size} bytes)')
            ali.download_file(file=f, local_folder=local_dir)
            print(f'✅ 下载完成 → {local_dir}/{f.name}')
            return

    # 6) 没找到？再等几秒试一次
    time.sleep(5)
    items = ali.get_file_list('root')
    for f in items:
        if not f: continue
        if target_name in f.name or target_name.split('.')[0] in f.name:
            print(f'找到（延时后）: {f.name}')
            ali.download_file(file=f, local_folder=local_dir)
            print(f'✅ 下载完成 → {local_dir}/{f.name}')
            return

    print(f'❌ 转存后未找到 {target_name}')
```

## 三、常见 API 签名速查（2026-06-14）

| API | 第一个参数 | 第二个参数 | 返回值 |
|---|---|---|---|
| `get_share_token` | `share_id: str` | `share_pwd: str=''` | `GetShareTokenResponse`（对象） |
| `share_file_save_all_to_drive` | `share_token: GetShareTokenResponse`（传对象！） | `to_parent_file_id='root'` | `List[BatchSubResponse]` |
| `get_share_file_list` | `share_token: GetShareTokenResponse`（传对象！） | `parent_file_id='root'` | `List[BaseShareFile]` |
| `get_file_list` | `parent_file_id='root'` | `drive_id: str=None` | `List[BaseFile]` |
| `list_my_drives` | 无参数 | — | `List[BaseDrive]`（含 `drive_name`） |
| `v2_user_get` | 无参数 | — | `UserResponse`（含 `resource_drive_id` / `backup_drive_id`） |
| `v2_sbox_get` | 无参数 | — | `SBoxResponse` |

### 已失效的 API

| API | 错误信息 |
|---|---|
| `get_share_link_download_url` | ❌ `NotImplementedError: get_share_link_download_url 已失效` |
| `share_file_saveto_drive` | ❌ `'NoneType' object has no attribute 'share_id'` |
| `search_file` | ⚠️ DeprecationWarning → 改用 `search_files`（但 `search_files` 也有 bug） |

## 四、文件找不到的排查顺序

```
发现文件不在根目录
├─ 1. ali.list_my_drives() → 看每个盘的 used_size
│   └─ used > 0 的盘里才有文件
├─ 2. ali.v2_user_get() → 拿到 resource_drive_id / backup_drive_id
├─ 3. 分别列两个盘：
│   ├─ ali.get_file_list('root')                          ← 备份盘（default_drive_id）
│   └─ ali.get_file_list('root', drive_id=u.resource_drive_id)  ← 资源盘
├─ 4. ali.get_file_by_path('/文件名') 精确查找
└─ 5. 递归 walk 全盘（但限速很快，慎用）
```

## 五、一句话总结

> **aligo 默认操作的是备份盘（`693900515`），不是资源盘（`545500515`）。** 要操作资源盘必须显式传 `drive_id='545500515'`。分享转存最稳的路径是：`get_share_token(对象)` → `share_file_save_all_to_drive(对象)` → 等 2 秒 → 默认盘找文件 → `download_file`。
