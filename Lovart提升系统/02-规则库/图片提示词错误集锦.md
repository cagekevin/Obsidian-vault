# 图片提示词错误集锦（深层逻辑分析版）

---

## 错误 #1：拼贴海报 - 纹理直接复制

**表面错误：**

把参考图当垫图，导致纹理被复制。

**深层逻辑错误：**

### 1. 工具理解错误

我没有区分 REFERENCE_TO_IMAGE 和 EDIT_SINGLE_IMAGE 的本质区别

REFERENCE_TO_IMAGE 的逻辑是：AI会尝试"模仿"参考图的视觉特征

当参考图有强烈的纹理特征时，AI会优先复制纹理而不是理解抽象的构图逻辑

### 2. 任务分类错误

用户的真实需求是：创造一个新的拼贴海报（TEXT_TO_IMAGE）

参考图的作用是：提供构图灵感和风格方向

我却把它当成：要编辑或模仿的对象（REFERENCE_TO_IMAGE）

**逻辑链断裂：**

```
用户意图：创造新作品，参考构图
     ↓
我的理解：模仿这张图
     ↓
工具选择：REFERENCE_TO_IMAGE
     ↓
结果：纹理被复制
```

### 3. "参考"概念混淆

人类的"参考"：提取抽象规律（构图、层次、风格）

AI的"参考"：像素级模仿（纹理、色彩、细节）

我没有意识到这个差异，直接把人类逻辑套用到AI工具上

**根本原因：**

我没有建立"任务目标 → 工具特性 → 参数选择"的完整逻辑链。

## 错误 #2：拼贴海报 - 人物"飞天"

**表面错误：**

人物悬浮，没站在地面。

**深层逻辑错误：**

### 1. 空间模型缺失

我在脑海中有"上下分层"的概念

但我没有建立完整的3D空间模型：

```
❌ 我的思维：上面是纸，下面是场景

✅ 应该的思维：
   - Y轴：人物的脚在哪里？（垂直位置）
   - Z轴：人物在哪一层？（前后关系）
   - 重力：人物与地面的接触点在哪？
```

### 2. 因果关系断裂

我描述了"人物在下方区域"

但我没有描述"人物为什么在下方" → 因为脚踩在地上

AI只知道"人物应该在下方"，但不知道"人物与地面的物理关系"

**逻辑链断裂：**

```
我的描述：人物在下方暗色区域
     ↓
AI理解：人物的身体中心在下方
     ↓
但没有约束：脚必须接触地面
     ↓
结果：人物悬浮在下方区域
```

### 3. 隐性知识未显性化

我认为"人物在地面场景中"是常识

但对AI来说，这不是常识，而是需要明确描述的约束条件

我假设AI会自动推理，但AI不会

**根本原因：**

我没有把"隐性的物理规律"转化为"显性的文字约束"。我以为AI会"理解常识"，但AI只会"执行指令"。

## 错误 #3：拼贴海报 - 花朵位置错误

**表面错误：**

装饰性插画位置混乱，有的比人还高。

**深层逻辑错误：**

### 1. 元素属性定义不清

我说了"5 botanical illustrations along the torn edge"

但我没有定义这些插画的属性：

- 它们是平面图形还是3D物体？
- 它们是固定在纸上还是漂浮在空间中？
- 它们的尺寸是多少？

AI在缺乏约束的情况下，自由发挥

### 2. 相对位置vs绝对位置

我说"along the torn edge"（相对位置）

但撕裂边是斜线，"沿着"可以有无数种理解：

- 紧贴边缘？
- 在边缘上方？
- 在边缘下方？
- 跨越边缘？

我没有给出绝对位置约束（Y轴坐标范围）

### 3. 层级关系未建立

我没有明确：

- 插画在哪一层？（纸层 vs 场景层）
- 插画与人物的Z轴关系？（插画在前，人物在后）
- 插画与撕裂边的附着关系？（插画是纸的一部分）

**根本原因：**

我用"模糊的相对描述"代替了"精确的绝对约束"。我以为"along the edge"足够清晰，但对AI来说，这是一个有无限解的方程。

## 错误 #4：拼贴海报 - 采摘动作逻辑错误

**表面错误：**

人物手势向上伸向插画，而不是向下采摘真实花朵。

**深层逻辑错误：**

### 1. 动作的"对象"缺失

我说"harvesting gesture"（采摘手势）

但我没有说"采摘什么"

在画面中有两种花：

- 上方的装饰性插画
- 下方的真实花朵

AI不知道该采摘哪一个，选择了视觉上更显眼的插画

### 2. 因果链不完整

完整的因果链应该是：

```
人物是农民 
  → 农民在采摘 
  → 采摘的对象是地面上的真实花朵
  → 所以手要向下伸
  → 所以手的位置在身体下方
  → 所以手接触的是地面区域的花
```

我只描述了第2步（采摘），跳过了第3-6步

AI只知道"做采摘动作"，但不知道为什么、对什么、怎么做

### 3. 视觉权重误判

在我的提示词中：

- 装饰性插画：描述详细，视觉显眼
- 真实花朵：描述简略，位置模糊

AI根据描述权重判断重要性

结果：AI认为插画更重要，所以人物去采摘插画

**根本原因：**

我只描述了"动作"，没有描述"动作的完整逻辑链"（主体-动作-对象-原因-结果）。我以为"采摘"是自明的，但对AI来说，需要明确"采摘谁、为什么、怎么采"。

## 错误 #5：拼贴海报 - 插画重复单调

**表面错误：**

5朵插画长得一模一样。

**深层逻辑错误：**

### 1. 默认行为未覆盖

AI的默认行为：批量生成相同元素

当我说"5 botanical illustrations"时，AI理解为：

```
复制(插画, 5次)
```

我没有覆盖这个默认行为

### 2. "多样性"不是默认属性

在现实世界中，手绘插画自然会有差异

但在AI世界中，一致性是默认的，多样性需要明确要求

我假设AI会自动添加变化，但AI不会

### 3. 描述粒度不足

我只说了"what"（5朵插画）

没有说"how"（如何排列、如何变化）

AI在缺乏细节指令时，选择最简单的实现方式：复制粘贴

**根本原因：**

我没有理解AI的"默认行为"。我以为"多样性"是自然的，但对AI来说，"一致性"才是默认的。我需要明确覆盖默认行为。

## 错误 #6：地中海盛夏岩玫瑰 - 光线氛围不符

**表面错误：**

文案说"炽热阳光"，图片却是阴天冷色调。

**深层逻辑错误：**

### 1. 模式套用错误

我建立了"诗意风格 = 冷色调 + 柔和光线"的固定模式

当用户说"诗意风格"时，我自动套用这个模式

但我没有检查这个模式是否适用于当前文案

**逻辑链断裂：**

```
用户说：诗意风格
     ↓
我激活：冷色调模式
     ↓
但文案说：炽热阳光
     ↓
冲突：冷色调 vs 炽热
     ↓
我没有检测到冲突，直接套用模式
```

### 2. 抽象概念未解构

"诗意"是一个抽象概念，不是固定参数

"诗意"的本质是：自然、不做作、有意境

但"诗意"可以有不同的具体表现：

- 冷色调的诗意（宁静、忧郁）
- 暖色调的诗意（温暖、治愈）
- 强光的诗意（神圣、眷顾）

我把抽象概念固化为单一参数，失去了灵活性

### 3. 文案分析缺失

我没有先分析文案的核心情绪

直接跳到"生成提示词"

**正确的流程应该是：**

```
1. 分析文案关键词：炽热、阳光、滚烫、赏赐、天光眷顾
2. 提取核心情绪：温暖、神圣、充沛、治愈
3. 翻译为视觉参数：warm golden light, clear sky, summer warmth
4. 生成提示词
```

我跳过了步骤1-3

**根本原因：**

我建立了"诗意 = 固定参数"的错误映射，没有理解"诗意"是抽象概念，需要根据具体文案解构。我用"模式匹配"代替了"逻辑推理"。

## 错误 #7：岩玫瑰花瓣颜色错误 ⚠️ 最严重

**表面错误：**

参考图是白色花瓣，我写了粉色。

**深层逻辑错误：**

### 1. 信息源优先级错误

当前有三个信息源：

1. 当前参考图（真实、准确）
2. 历史记忆（可能过时、不准确）
3. Summary Context（压缩、可能失真）

正确的优先级应该是：1 > 2 > 3

但我的实际优先级是：2/3 > 1

我优先相信记忆，而不是当前证据

### 2. 验证步骤缺失

**完整的逻辑链应该是：**

```
用户说"要加参考图"
     ↓
触发验证流程：analyse_image
     ↓
提取当前真实信息
     ↓
对比记忆中的信息
     ↓
如果冲突，以当前为准
     ↓
使用验证后的信息
```

我跳过了整个验证流程

直接用记忆中的信息

### 3. "记忆"的本质误解

我把"记忆"当作可靠的知识库

但"记忆"的本质是：历史对话的压缩摘要

压缩过程中会：

- 丢失细节
- 产生偏差
- 混淆不同对话的信息

我没有意识到记忆的不可靠性

### 4. 懒惰的认知捷径

调用 analyse_image 需要：

- 额外的工具调用
- 等待分析结果
- 处理分析信息

直接用记忆：

- 立即可用
- 不需要等待
- 感觉"更高效"

我选择了认知捷径，牺牲了准确性

**根本原因：**

我建立了错误的信息源优先级（记忆 > 当前证据），并且用"认知捷径"代替了"验证流程"。我没有理解"记忆是压缩的、不可靠的"，而"当前分析是准确的、可信的"。

## 🧠 深层逻辑错误的共同根源

### 1. 隐性知识显性化失败

我假设AI会理解"常识"（重力、物理关系、多样性）

但AI只执行指令，不推理常识

我需要把所有隐性规律转化为显性约束

### 2. 抽象概念具体化失败

我用抽象词（"诗意"、"优雅"、"沿着"）代替具体参数

但AI需要具体的、可执行的指令

我需要把抽象概念解构为具体参数

### 3. 因果链完整性缺失

我只描述结果（"采摘手势"），不描述原因和过程

但AI需要完整的因果链才能正确推理

我需要描述"为什么-是什么-怎么做"的完整链条

### 4. 工具特性理解不足

我不理解不同工具的底层逻辑（REFERENCE vs EDIT）

导致工具选择错误

我需要理解工具的工作原理，而不是表面功能

### 5. 验证流程缺失

我依赖记忆，不验证当前证据

导致信息过时或错误

我需要建立"当前验证 > 历史记忆"的铁律

### 6. 默认行为未识别

我不知道AI的默认行为（复制、一致性、像素级模仿）

导致结果不符合预期

我需要识别并覆盖AI的默认行为

## 🔧 根本性改进方案

### 1. 建立完整的逻辑链检查清单

**任务分析：**
- [ ] 用户的真实目标是什么？（创造 vs 编辑 vs 模仿）
- [ ] 参考图的作用是什么？（风格 vs 对象 vs 细节）
- [ ] 应该用什么工具？（TEXT_TO_IMAGE vs REFERENCE vs EDIT）

**空间模型：**
- [ ] Y轴：每个元素的垂直位置？
- [ ] Z轴：每个元素的前后层级？
- [ ] 物理关系：元素之间的接触/附着/支撑关系？

**因果链：**
- [ ] 主体是谁？
- [ ] 动作是什么？
- [ ] 对象是谁？
- [ ] 为什么这样做？
- [ ] 结果是什么？

**信息验证：**
- [ ] 是否需要分析参考图？
- [ ] 当前信息 vs 历史记忆，哪个更可靠？
- [ ] 是否有冲突？如何解决？

**默认行为：**
- [ ] AI的默认行为是什么？
- [ ] 是否需要覆盖？
- [ ] 如何覆盖？

### 2. 建立"抽象→具体"的翻译表

| 抽象概念 | 具体参数 |
|---------|---------|
| "诗意" | 分析文案 → 提取情绪 → 翻译为光线/色调/构图 |
| "沿着边缘" | Y轴范围 + Z轴层级 + 附着关系 |
| "采摘" | 主体 + 对象 + 方向 + 位置 + 接触点 |
| "多样性" | varied + different angles + diverse |

### 3. 建立"当前验证优先"的铁律

```
IF 用户提供参考图：
    MUST analyse_image FIRST
    THEN 提取信息
    THEN 使用提取的信息
    NOT 使用记忆中的信息

IF 记忆与当前冲突：
    ALWAYS 以当前为准
    NOT 以记忆为准
```
