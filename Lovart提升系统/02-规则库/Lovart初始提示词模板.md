# Lovart 初始提示词模板（规则库）

> 从历史错误中提炼的行为准则。定期复盘更新。
> 每次与 Lovart 对话开始时，把相关规则复制给 Lovart。

---

## 一、任务分类与工具选择

### 规则1.1：编辑任务识别
**触发特征：** "替换XX"、"保持XX改变XX"、"在XX基础上"
**工具选择：** 图生图-编辑模式
**关键词：** Based on [image]
**结构：** Changes: [列表] + Keep: [列表]
**来源：** 错误#[X]

### 规则1.2：创意任务识别
**触发特征：** "创作XX"、"生成XX"、无需保留原图元素
**工具选择：** 文生图-创意 或 文生图-参考
**关键词：** 描述目标，参考图用"in the style of"
**来源：** 错误#[X]

---

## 二、精确描述原则

### 规则2.1：数值化描述
**适用：** 所有涉及尺寸/数量/程度的描述
**禁止：** significantly, much, a few, very
**必须：** 1.5x, 50% larger, 3-5个, increase by 30%
**来源：** 错误#[X]

### 规则2.2：空间位置描述
**适用：** 多元素布局任务
**禁止：** "沿着"、"附近"等模糊词
**必须：** Y轴位置（上/中/下 + 具体范围）+ 物理关系（接触/附着/支撑）
**来源：** 错误#[X]

---

## 三、参考图处理

### 规则3.1：必须先分析
**触发：** 用户提供参考图 + 涉及颜色/细节/结构
**必须：** 先调用analyse_image
**禁止：** 依赖记忆/Summary Context
**来源：** 错误#[X]

### 规则3.2：明确参考图作用
**必须区分：**
- 风格参考 → "in the style of"
- 编辑基础 → "based on"
- 元素提取 → "using elements from"
**来源：** 错误#[X]

---

## 四、隐性逻辑显性化

### 规则4.1：元素去除
**禁止：** 通过"不提某元素"暗示去除
**必须：** 明确写"Remove [元素] completely"
**来源：** 错误#[X]

### 规则4.2：多样性要求
**禁止：** 假设AI会自动添加变化
**必须：** 明确写"varied, different angles, diverse"
**来源：** 错误#[X]

---

## 五、任务类型决策树

```
用户需求
    ↓
是否需要保留原图部分元素？
    ├─ 是 → 图生图-编辑
    │       ↓
    │       用Based on + 分层描述Changes/Keep
    │
    └─ 否 → 是否有参考图？
            ├─ 有 → 用户是要反推提示词（把图变成文字）？
            │       ├─ 是 → 反推提示词
            │       │       ↓
            │       │       用analyse_image分析画面元素
            │       │       + 输出可复用提示词
            │       │
            │       └─ 否 → 文生图-参考
            │               ↓
            │               先analyse_image + 用in the style of
            │
            └─ 无 → 文生图-创意
                    ↓
                    描述目标场景
```

### 规则5.1：反推提示词
**触发：** 用户给了一张图，说"反推"、"提取提示词"、"分析提示词"、"这个图怎么写"
**工具：** analyse_image
**必须输出：**
- 风格判断（摄影/插画/3D渲染等）
- 构图描述（主体位置、视角、景别）
- 光线分析（光源方向、光质、氛围）
- 色彩方案（主色、辅色、点缀色）
- 材质/纹理
- 关键元素列表
- 可复用的提示词（中英文）
**来源：** 本次添加
