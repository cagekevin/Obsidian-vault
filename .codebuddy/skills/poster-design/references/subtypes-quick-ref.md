# 平面设计 12 子类型要点速查（高密度）

> **何时 read**：开始 brief 时快速判断本子类型的重点 + 高危雷区。详细 playbook 走 `subtypes/{NN}-{name}.md`。

| 子类型 | 重点 | 高危雷区 |
|---|---|---|
| **xiaohongshu-cover** | 首屏 60%+ 焦点占面 + 大字标题（占宽 60%+）+ 暖色调 | 字太小手机看不清；冷色调（违和小红书生活感）；信息过密 |
| **wechat-cover** | 高级感留白（30%+）+ 标题字位规范（中央 / 偏上）| 留白不足显廉价；标题压主体；信息过密 |
| **weibo-banner** | 16:9 横幅 + 信息密度可高（叙事感强）| 比例错（用 1:1 会被自动裁掉两侧）；配图脱离主题 |
| **story-vertical** | 9:16 全屏沉浸 + 大字标题（占宽 60%+）+ 关键信息上半屏（避免被 UI 遮）| 关键信息在底部被 UI 遮；横版改竖版直接拉伸 |
| **movie-poster** | 戏剧性主视觉（占满）+ 标题字位（底部 1/3）+ 主演署名规范 | 主视觉与标题打架；署名字号过小；裁切人脸 |
| **event-poster** | 主视觉 + 时间地点信息层级清晰 + 主办方 logo 位 | 信息过密无层级；时间地点字号过小；主视觉抢眼无视信息 |
| **music-festival-poster** | lineup 排版（艺人字号梯度）+ 视觉冲击 + 时间地点 | lineup 字号无梯度；过度炫技丢失信息 |
| **magazine-cover** | 刊头位（顶部）+ 主体人物视线方向（影响版式）+ cover lines（左下 / 右下）| 刊头被主体遮挡；视线引导出框；cover lines 字号不一致 |
| **book-cover** | 概念图像（不是说明图）+ 标题字处理 + 作者署名规范 | 标题字太具象失去概念；作者署名抢眼；类型错配（言情用工业风）|
| **album-cover** | 1:1 + 主体居中 + 留出唱片标签位（中央圆环）+ 标志性记忆点 | 主体偏离中心；标签位被主体压；过度复杂失去 iconic |
| **ad-banner** | 产品占 40-50% + 卖点文案显眼 + CTA 区（右下 / 底部）| 产品占比不足；卖点不显眼；CTA 区被主图压 |
| **promo-poster** | 折扣字号最大（占宽 50%+）+ 倒计时 / 价格 + 紧迫感配色（红 / 橙）| 折扣字号不显眼；价格放小；冷色调（无紧迫感）|
