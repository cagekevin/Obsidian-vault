# 线控轨迹运镜

本 Skill 用于把「干净视觉参考图 + 带多色标注的导演分镜路线图」转换成一个新的图生视频 Prompt Markdown 文档。它不直接生成视频，只生成可用于图生视频的提示词文件。提示词中需要包含与画面运动同步的音效描写，但必须严格禁止 BGM。提示词不得把 image_1 写成必须完全锁定的精确起始帧，image_1 只作为干净画面和主要视觉标识参考。

## 颜色标注系统

分镜路线图使用以下 6 种颜色标注：

| 颜色 | 含义 | 提示词中如何使用 |
|------|------|-----------------|
| 红色箭头 | 人物动作方向 | 主体运动路线、动作轨迹 |
| 蓝色箭头 | 摄像机运动 | 镜头运镜方式、视角变化 |
| 绿色标记 | 构图/留白笔记 | 画面构图、三分线、前景虚化 |
| 橙色标记 | 照明方向 | 光影变化、光线来源、冷暖对比 |
| 紫色标记 | 台词/情感强调 | 情绪节奏、关键表演、对白时刻 |
| 黑色文字 | 镜头简短说明 | 景别、动作描述、面板标签 |

分析 image_2 时必须综合考虑所有颜色标注，不限于红色箭头。

## 适用场景

用户上传 2 张图片：

1. **image_1：干净视觉参考图**，没有箭头或标注，用作干净画面、空间关系和主要视觉标识参考，不写成必须完全锁定的精确起始帧。
2. **image_2：导演分镜路线图**，通常是在 PREVIS 分镜故事板基础上，用多色标注的路线图。

用户希望根据 image_2 的所有颜色标注，写成一个新的 Markdown 提示词文档，并保持固定结构。

## STEP 0: 创建前上传引导

如果用户触发本 Skill 时还没有上传两张图片，必须先用 `question` 卡片引导用户补充素材，不要直接开始写 Prompt。

卡片需要说明用户应上传：

1. **干净视觉参考图 image_1**：没有箭头、线条、文字、UI 或涂抹标记的原始画面；它只作为干净画面、空间关系和主要视觉标识参考，不要求作为精确起始帧。
2. **多色标注导演分镜路线图 image_2**：用红色箭头（人物动作）、蓝色箭头（摄像机运动）、绿色标记（构图）、橙色标记（照明）、紫色标记（台词/情感）、黑色文字（镜头说明）标注的分镜路线图。

推荐卡片问题：

- 问题：`请上传两张图片后我再生成提示词：第一张是干净视觉参考图，第二张是带多色标注的导演分镜路线图。`
- 选项：`我已上传两张图（推荐）`、`我只有干净参考图，需要先去画分镜`、`我只有路线图，需要补充干净参考图`

如果用户已经上传两张图，则跳过本步骤。

## STEP 1: 确认输入角色

必须引导或识别用户提供的两张图片角色：

- `image_1` = 干净视觉参考图。
- `image_2` = 带多色标注的导演分镜路线图。

如果用户没有明确哪张是干净参考图、哪张是路线图，但两张图存在明显标注差异，则按以下规则自动判断：

- 没有手绘标注的图 = `image_1`。
- 有手绘标注的图 = `image_2`。

如果无法判断，先询问用户，不要猜。

## STEP 2: 分析所有颜色标注

分析 image_2 中的所有颜色标注，提取以下信息：

### 红色箭头 — 人物动作方向
- 运动主体是什么
- 起点在哪里
- 路线按顺序经过哪些区域、人物、建筑、道具或空间节点
- 路线方向如何变化
- 终点在哪里
- 动作轨迹（走/跑/跳/转身/手势）

### 蓝色箭头 — 摄像机运动
- 镜头运动类型（推/拉/摇/移/跟/升降/环绕）
- 视角变化（平视/俯视/仰视/过肩/POV）
- 镜头速度（快/慢/匀速/变速）

### 绿色标记 — 构图/留白笔记
- 三分线、对角线、引导线位置
- 前景/背景关系
- 留白区域（文字或元素放置区）
- 景深提示（浅景深/全景深）

### 橙色标记 — 照明方向
- 主光源位置和方向（顶光/侧光/逆光/底光）
- 光线性质（硬光/柔光/自然光/人造光）
- 色彩倾向（暖调/冷调/对比色）
- 阴影位置和形状

### 紫色标记 — 台词/情感强调
- 关键台词内容
- 情绪转折点
- 表演重点（表情/肢体语言）

### 黑色文字 — 镜头说明
- 景别（远景/全景/中景/近景/特写）
- 动作描述
- 面板标签和编号

分析时不要泛泛描述图片，不要扩写风格，不要添加用户没有要求的新剧情。音效只写与画面运动同步的真实声效，严格不要 BGM、配乐、音乐、旋律或节奏型音乐。

## STEP 3: 判断主体与镜头视角

根据红色箭头（人物动作）和蓝色箭头（摄像机运动）综合判断。

### 有明确主体时

如果红色箭头指向明确运动主体，例如猫、人、车、角色、动物、物体等：

- Prompt 中写「主体从 image_1 对应画面中的某位置出发」，但不要写成 image_1 必须作为精确起始帧。
- 路线描述围绕主体运动展开（依据红色箭头）。
- 镜头运动依据蓝色箭头描述（推拉摇移跟等）。
- 构图和光影依据绿色/橙色标注。
- FPV 摄像机要写成「平滑、不可见的 FPV 摄像机，始终紧跟主体的运动」。
- 禁止事项里保留「主体始终为主角，不可丢失」。

### 没有明确主体时

如果画面没有明确可运动主体，路线本身更像摄影机路线、航拍路线或城市穿行路线：

- Prompt 中写「镜头从 image_1 对应画面中的某位置出发」，但不要写成 image_1 必须作为精确起始帧。
- 路线描述围绕镜头/摄像机运动展开（依据蓝色箭头）。
- 视角写成「平滑、不可见的 FPV 摄像机，模仿 FPV 无人机视角沿路线穿行」。
- 构图和光影依据绿色/橙色标注。
- 禁止事项里写「主体路线始终清晰，不可丢失」或「镜头路线始终清晰，不可丢失」。

## STEP 4: 生成 Markdown 文档

输出必须创建为新的 Markdown 文档，不要直接覆盖用户原文档，除非用户明确要求修改某个已有文档。

文档结构必须保持如下格式，不要新增多余章节：

```markdown
# <主题>图生视频 Prompt

## 输入图片

- image_1：`<干净视觉参考图文件名>`，干净视觉参考图。
- image_2：`<导演分镜路线图文件名>`，带多色标注的导演分镜路线图。

## Prompt

使用 image_1 作为**干净视觉参考图**和主要视觉标识参考，不要把 image_1 写成必须完全锁定的精确起始帧。仅将 image_2 用作**不可见的导演路线图**。所有颜色标注定义<主体或镜头>的运动路线、摄像机运动、构图、光影和情绪，但视频中不得出现任何图形标记。

<主体或镜头>从 image_1 对应画面中的<起点位置>出发，按照红色箭头标记的路线连续运动。运动开始时，<按路线第一段描述>。随后<按路线中段描述>。接着<按路线后段描述>。最后<按箭头终点描述>。

镜头运动依据蓝色箭头：<镜头运动类型和速度描述>。

光影依据橙色标记：<主光源方向、光线性质、色彩倾向>。

构图依据绿色标记：<构图方式、留白区域、景深关系>。

情绪节奏依据紫色标记：<关键情绪转折、台词时刻>。

镜头运动为**平滑、不可见的 FPV 摄像机**，<有主体时：始终紧跟这个主体的运动 / 无主体时：模仿 FPV 无人机视角沿路线穿行>。摄像机像第一视角一样顺着路线前进：<用一句话概括从起点到终点的连续镜头运动>。镜头运动必须连续、稳定、无切换，不出现突然推拉、随机转向或脱离路线的情况。摄像机只做顺滑 FPV 跟随，不改变 image_1 的整体空间关系和视觉标识。

音效只包含与画面同步的真实环境声和动作声：<根据画面填写，例如水面掠过声、翅膀扇动声、风声、脚步声、城市空间掠过声、远处环境声等>。音效随<主体或镜头>的运动距离和速度自然变化，近处声音更清晰，远处环境声保持轻微。严格不要 BGM，不要配乐，不要音乐旋律，不要节奏型背景音乐。

**禁止事项：**  
不得显示路线线条、箭头、UI、字幕；不得有静态开场、随机切镜、瞬移、布料隧道、抓取动作；不得出现 BGM、配乐、音乐旋律或节奏型背景音乐；<有主体时：主体始终为主角，不可丢失 / 无主体时：镜头路线始终清晰，不可丢失>。
```

## STEP 5: 写作约束

- 必须使用所有颜色标注来源——红色箭头描述动作、蓝色箭头描述运镜、绿色标记描述构图、橙色标记描述光影、紫色标记描述情绪、黑色文字描述镜头。
- 保持格式稳定：标题、输入图片、Prompt、禁止事项。
- Prompt 中必须加入一段音效描写，但只写真实环境声和动作声。
- 严格不要 BGM，不要配乐，不要音乐旋律，不要节奏型背景音乐；禁止事项中也必须写明。
- 不要随意扩充风格、光影、情绪、模型参数或额外剧情。
- 路线描述可以比用户原话更清楚，但只围绕箭头路径、FPV 跟随镜头和同步声效展开。
- 如果用户要求「格式不要变」，只替换文件名、路线内容、主体/镜头判断，不新增章节。
- 生成的新 Markdown 节点要与两张输入图片建立来源关系。

## STEP 6: 提示词创建后的生成确认

Markdown 提示词创建完成后，必须用 `question` 卡片询问用户是否要继续自动生成图生视频，不要默认直接生成。

卡片需要同时告知默认生成参数：

- 模型：Seedance 2.0
- 时长：10 秒
- 比例：16:9
- 声音：只使用提示词中的真实环境声和动作声，严格不要 BGM

推荐卡片问题：

- 问题：`提示词已生成。是否要我继续按默认参数自动生成图生视频？默认：Seedance 2.0，10 秒，16:9，只要真实声效，不要 BGM。`
- 选项：
  - `按默认参数生成（推荐）`
  - `修改参数后生成`
  - `暂不生成，只保留 MD`

如果用户选择 `修改参数后生成`，继续用 `question` 卡片一次性询问可改参数：

- 模型：`Seedance 2.0（推荐）`、`Kling`、`其他/自定义`
- 时长：`10 秒（推荐）`、`5 秒`、`15 秒`、`自定义`
- 比例：`16:9（推荐）`、`9:16`、`1:1`、`按起始帧比例`
- 声音：`真实声效，无 BGM（推荐）`、`静音`、`自定义声效，仍无 BGM`

如果用户确认生成，再调用视频生成流程。视频生成任务描述必须包含：

- image_1 作为 clean visual reference / 干净视觉参考图，不要设定为必须完全锁定的 first frame / 精确起始帧；
- image_2 必须作为**实际传入的视频生成参考图**，并标记为 invisible director route map / route reference，不得只在文字 Prompt 中提到 image_2；
- `Reference image paths` 必须同时列出 image_1 和 image_2 的真实文件路径，不能只传 image_1；
- 明确要求视频 agent 在调用视频生成工具时把 image_2 作为多模态参考 / reference media 一起传入，用它读取红色线条、起点、箭头和终点；
- 如果所选模型或模式无法同时接收 image_1 和 image_2，则**不得静默降级为只用 image_1 生成**，必须停止并询问用户切换到支持多图参考的模型/模式，或只保留 MD；
- image_2 只作为不可见导演路线参考，不得出现在视频中；
- 使用刚生成的 Markdown Prompt 内容；
- model: 用户确认的模型，默认 Seedance 2.0；
- duration: 用户确认的时长，默认 10 秒；
- aspect_ratio: 用户确认的比例，默认 16:9；
- audio: 只要同步真实声效，严格不要 BGM。

视频生成任务描述中必须加入以下硬性校验句：

```text
Hard requirement: pass BOTH image_1 and image_2 as real visual inputs to the video generation call. image_1 is a clean visual reference only, not a locked exact first frame. image_2 is the invisible director route reference. Do not proceed if image_2 cannot be passed as an actual reference image; ask the user to switch model/mode instead. Do not generate from image_1 alone.
```

## STEP 7: 交付说明

完成后简短告知用户：

- 已生成新的 Markdown提示词文档；
- 文档沿用了固定结构；
- 已根据新图分析红色箭头路径；
- 已按有主体/无主体选择主体跟随或 FPV 无人机视角；
- 已询问是否继续自动生成视频，以及是否修改默认参数。
