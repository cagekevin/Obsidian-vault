# 海报五模块质量评分模型 + 19 项指标 + 12 子类型权重覆盖

> **何时 read**：每张海报生成后 evaluation 阶段。**评分是 mandatory，不可跳过**——出图后必走 5 模块自检。
> 海报场景专属的 5 模块 19 项加权评分体系。
> **与 `ecommerce/quality-scoring.md` 区别**：电商六维评分聚焦商品转化，本文件聚焦海报的视觉传达 + 信息层级 + 排版质量。

## At-a-Glance（5 模块默认权重）

| # | Module | Core question | Default weight |
|---|--------|---------------|----------------|
| 1 | **基础视觉质量** | 构图 / 层次 / 配色 / 留白基本功是否合格？ | 0.30 |
| 2 | **排版与文字** | 字体规范 + 文字可读性是否达标？（海报最大失败点）| 0.25 |
| 3 | **信息传达** | 3 秒内能否抓住核心 + 文案是否准确？ | 0.20 |
| 4 | **风格与创意** | 风格是否统一 + 是否匹配场景 + 有无情绪/概念？ | 0.15 |
| 5 | **技术规范** | 分辨率 / 尺寸 / 边距 / 清晰度等硬规范？ | 0.10 |

> 每项指标含 P0/P1/P2/P3 优先级标记。**任一 P0 子项 < 6 分 → 整图判 fail，必须 regen**。

---

## 1. 基础视觉质量（4 项，权重 0.30）

**原则**：海报的下限——基本功不到位，再好的创意也救不了。

| Sub-criterion | 优先级 | Pass | Fail |
|---|---|---|---|
| 构图合理性 | P0 | 符合三分法 / 居中 / 对称 / 引导线等基本构图原则 | 主体偏到角落 / 视线被截断 / 重心失衡 |
| 视觉层次 | P0 | 主体突出 + 前/中/背景层次分明 | 主体被背景吃掉 / 多焦点抢戏 / 平铺无层次 |
| 色彩和谐度 | P1 | 配色协调舒适 + 符合主题氛围| 撞色 / 浑浊 / 7+ 颜色无主从 / 与主题氛围相反 |
| 留白合理性 | P0 | 有适当呼吸空间，避免拥挤或空旷 | 满图无喘息 / 大面积无意义空白 / 留白被渐变填充 |

**1–10 评分**：
- 9–10：基本功扎实，构图层次完美，色彩克制，留白讲究
- 7–8：基本功合格，1 项小弱点
- 5–6：勉强及格，明显某项弱（如多焦点 / 配色凌乱）
- 3–4：基本功不行，多项 fail
- 1–2：构图崩 / 层次失 / 色彩灾难

---

## 2. 排版与文字（5 项，权重 0.25）

**原则**：海报 80% 失败在文字。字体错 / 文字压主体 / 错字 / 看不清都直接判 fail。

| Sub-criterion | 优先级 | Pass | Fail |
|---|---|---|---|
| 字体变形检查 | P0 | 文字无拉伸 / 压缩 / 不当变形 | 字被强行拉宽拉高 / 不当倾斜 / 笔画扭曲 |
| 字体数量控制（≤ 3 种）| P2 | 全图字体不超过 3 种（含中英文计数）| 4+ 种字体堆砌 / 风格各异无主从 |
| 文字-背景对比度 | P0 | 文字与背景对比度 ≥ WCAG AA（4.5:1 正文 / 3:1 大字标题）| 浅色文字落浅背景 / 文字落复杂纹理上 |
| 最小字号检查 | P1 | 正文不低于该子类型可读下限（参考 `subtypes/{NN}.md` 字号建议）| 小字 < 16pt（屏显）或 < 9pt（印刷）→ 缩略图看不清 |
| 文字遮挡检查 | P0 | 文字不被其他元素遮挡 + 不压在复杂纹理上 | 文字压在人脸 / logo / 复杂图案上 → 命中 visual-weight-zones 失败模式 #1 |

**1–10 评分**：
- 9–10：排版即设计——字体调性匹配 + 间距讲究 + 层级清晰 + 全图可读
- 7–8：排版合格，1 项小问题（字号略小 / 字体多 1 种 / ...）
- 5–6：勉强可读，对比度弱或文字被遮挡部分
- 3–4：多项 fail，可读性受影响
- 1–2：排版灾难——错字 / 字形扭曲 / 文字被吞 / 完全看不清

---

## 3. 信息传达（2 项，权重 0.20）

**原则**：海报的本质是**信息载体**。3 秒内传达不到核心 = 失败。

| Sub-criterion | 优先级 | Pass | Fail |
|---|---|---|---|
| 核心信息突出度（3 秒抓眼）| P0 | 3 秒内能捕获最核心信息（标题 / 主视觉），视觉权重三区分明| 主信息被次信息抢戏 / 等大焦点抢夺 / 文字飘在 Light 区中央无锚点 |
| 文案准确性 | P0 | 文案无错别字 + 无事实错误 + 时间/价格/联系方式/url 等关键数据**逐字正确** | 错字 / 半字 / 时间错（"05.21" 写成 "05.12"）/ 价格错 / 重复（同一文案出现 2 次）|

**1–10 评分**：
- 9–10：核心信息瞬间抓眼 + 文案逐字准确 + 无重复
- 7–8：信息层级清晰，文案 1 处小瑕疵（标点 / 空格）
- 5–6：信息传达模糊但能看懂，文案有 1-2 处错字
- 3–4：核心信息被埋没 / 多处错字
- 1–2：完全看不出主信息 / 关键数据全错

---

## 4. 风格与创意（4 项，权重 0.15）

**原则**：决定海报的**上限**——是平庸还是出彩。

| Sub-criterion | 优先级 | Pass | Fail |
|---|---|---|---|
| 全图风格统一 | P0 | 不会出现"一半极简一半花哨"的割裂感| 风格混搭（极简 + 花哨 / 写实 + 动漫 / 复古 + 未来）|
| 风格匹配场景 | P0 | 海报风格与使用场景匹配（小红书暖色 / 金融严肃 / 母婴温馨 / ...）| 小红书出冷色调 / 金融海报用糖果色 / 母婴用赛博朋克 |
| 情绪传达 | P1 | 整体传达预期情绪 / 氛围（治愈 / 热血 / 高级 / 紧迫 / ...）| 情绪不明 / 与主题氛围相反（活动海报死气沉沉）|
| 视觉概念 | P3 | 有超越"信息罗列"的视觉叙事或隐喻（不是通用标准）| 纯素材堆砌，无故事 / 无记忆点 / 无品牌感 |

**1–10 评分**：
- 9–10：风格统一 + 调性匹配 + 情绪到位 + 有视觉概念，记忆点强
- 7–8：基础风格合格，缺少情绪锚点或视觉概念
- 5–6：风格统一但调性偏弱 / 没特点
- 3–4：风格割裂或与场景错位
- 1–2：完全没有风格意图

> **视觉概念是 P3 但决定上限**：当 1-3 模块都 ≥ 8 分时，视觉概念决定这张图是 8 分还是 9.5 分；当基础不到位时，视觉概念无意义。

---

## 5. 技术规范（5 项，权重 0.10）

**原则**：硬规范。任一 fail = 直接降权或下架，总分救不了。

| Sub-criterion | 优先级 | Pass | Fail |
|---|---|---|---|
| 分辨率达标 | P0 | 印刷 ≥ 300dpi / 屏显 ≥ 72dpi | 印刷品 < 300dpi → 印出来糊 |
| 尺寸规格正确 | P0 | 严格匹配用户要求比例 + 像素（参考 `poster/iron-laws.md` #1）| 出错比例（要 3:4 出 1:1）/ 像素低于子类型最低要求 |
| 安全边距 | P1 | 重要元素不在裁切危险区域（印刷出血 3mm / 屏显安全区 5%）| 标题被裁 / 关键信息压到边缘 / 小红书底 12% 被 UI 遮 |
| 图片清晰度 | P0 | 素材无模糊 / 无明显噪点 / 无压缩痕迹 | 模糊 / 噪点严重 / JPEG 压缩条 |
| 元素无变形 | P0 | 图片 / 图标 / logo 无拉伸变形 | logo 被强行拉宽 / 圆形被压成椭圆 |

**1–10 评分**：
- 9–10：所有规范完美达标
- 7–8：基本合规，1 项小瑕疵（边距略紧）
- 5–6：勉强合规，明显某项弱
- 3–4：多项 fail
- 1–2：核心规范错（分辨率 / 比例错）

> 本模块**精神上是二元的**——P0 任一子项 fail，分数应直接降到 ≤ 4，不论艺术上多好看。

## 加权总分

```
total = Σ (module_score × module_weight)
```

默认权重见 At-a-Glance 表格。子类型覆盖见末尾。

---

## Quality Bars（出图能否上线）

| Bar | Score | 含义 |
|---|---|---|
| Hero / 主推海报 | ≥ 8.5 | 批准用作头条 / 主推 / 印刷 |
| 常规发布海报 | ≥ 8.0 | 批准发布（社媒 / banner）|
| Below 8.0 | regenerate | 修 prompt + brief 迭代，不是只换 seed |
| **任一 P0 子项 < 6** | fail regardless | 直接 regen，不论总分多高 |
| 任一 severe AI defect | fail regardless | 错字 / 重复文案 / 风格割裂 / 文字压脸 等 |

---

## 自检流程（出图后必走）

```
出图 → 5 模块逐项打分 → 计算加权总分 + 检查 P0 子项
  │
  ├─ 总分 ≥ Bar + 所有 P0 ≥ 6 → 通过，进入下一步
  ├─ 总分 < Bar 但所有 P0 ≥ 6 → regen，调 prompt 提分
  └─ 任一 P0 < 6 → 直接 regen，定位是哪个 P0 失败 → 看是 #1-#5 哪类问题
```
