# 04. Story 竖屏 (story-vertical)

> **优先质量维度**: Visual Appeal + Immersion(全屏沉浸 + 移动竖屏适配)
> **默认比例**: 9:16(1080×1920)
> **核心转化驱动**: 全屏沉浸视觉 + 中央焦点 + 上下 15% 安全区预留贴纸 / 平台 UI

---

## 一、必出元素清单

> 信息递进流:**沉浸视觉占满屏 → 中央焦点抓眼 → 安全区互动元素引导**
> 核心逻辑:竖屏全屏体验,用户单手浏览,焦点必须在视线中央(屏幕 30-70% 高度区间),上下安全区被平台 UI 占用。

| # | 必出元素 | 视觉权重区 | 占比 | 位置 | 信息职责 |
|---|----------|-----------|------|------|---------|
| 1 | 全屏主背景 | Heavy | 100% | 全画布 | 沉浸氛围 — 全屏铺满,无明显边界 |
| 2 | 中央焦点 | Heavy | 50%+ | 屏幕 30-70% 高度区 | 视觉抓眼 — 主体在视线中央 |
| 3 | 主文案 / 标题 | Medium | 标题字宽 50-70% | 中央或上安全区下沿 | 价值传达 — 一句话讲清吸引点 |
| 4 | 互动引导(贴纸位 / CTA) | Medium | 10-15% | 下安全区上沿(约 70-75% 高度) | 引导互动 — 「滑动」「点击」「投票」位预留 |
| 5 | 装饰元素(可选) | Light | < 8% | 边缘 / 留白区 | 氛围辅助 — 几何图标 / 表情贴 |

---

## 二、整体风格与质感

| 维度 | 要求 |
|------|------|
| **风格方向** | 全屏沉浸 / 移动优先 / 故事感 / 个人化 |
| **色温** | 不限,但要全图统一情绪(温暖 / 冷峻 / 能量),情绪混乱破坏沉浸 |
| **背景** | 大场景 / 渐变 / 主体延展型背景;**不**留大段死白(移动屏感觉空) |
| **滤镜** | 全屏氛围一致;推荐电影感 / 高级灰 / 暖橘茶 |
| **质感** | 真实摄影 + 浅景深 / 插画 + 全屏背景;避免拼贴感(破坏沉浸) |

---

## 三、版式构图要求

| 构图方式 | 适用场景 | 焦点位置 |
|---------|---------|---------|
| 中央焦点 + 上下安全区文案 | 人物 vlog / 产品展示 | 焦点 30-70% 高度 |
| 全屏背景 + 中央叠加文字 | 引言 / 名言 / 引流 | 文字居中,背景虚化 |
| 上图下文(上 70% 图 + 下 25% 文) | 故事讲述 / 产品介绍 | 图上、文下,文区在下安全区上沿 |
| 全屏主体延展(人物全身 / 物品大特写) | 时尚 / 服装 / 大物件 | 主体跨屏 60-80% 高度 |

**字体位置规则**:
- **上下安全区各 15%(约 288px)禁放关键信息**:上区被平台 header / 用户名占用,下区被滑动条 / 评论 / 点赞按钮占用
- 主文案落在 30-70% 高度区间(视线舒适带)
- 互动贴纸预留位(投票 / 提问 / 链接)通常在 60-80% 高度

---

## 四、信息层级映射

| 层级 | 视觉权重 | 字号建议(基于 1080×1920 画布) |
|------|---------|----------------------|
| 主文案 | Heavy | 90-140pt(占宽 50-70%) |
| 副文案 / CTA | Medium | 50-70pt |
| 装饰文字 / Hashtag | Light | 24-36pt |

---

## 五、文字渲染策略(按 Iron Law #3)

| 文字情况 | 推荐策略 + 模型 |
|---|---|
| 中文短句 ≤ 4 字 | A 嵌入:qwen / nano_banana 直接渲染 |
| 中文 5-15 字 | **B 纯底图**:seedream / midjourney 出底图 + 后期加文字(默认) |
| 中文 > 15 字 | **B 纯底图**:必须后期加文字 |
| 全英 / 标签 | **A 嵌入**:openai 直接渲染 |

**B 策略 prompt 写法**:`vertical 9:16 mobile story format, no text, no words, leave clean negative space at center 30-70% height for headline overlay, immersive full-screen composition, top and bottom 15% safe zones reserved for UI overlays`

---

## 六、必填项(Story 竖屏 Brief 缺一不可)

不写这些 → sub-agent 凭训练先验补全 → 移动体验失败:

- **目标平台**:Instagram Story / 抖音 / 朋友圈 / 小红书 Story(决定安全区与互动元素位置)
- **主焦点类型**:人物 / 产品 / 风景 / 文字主视(决定构图)
- **是否预留互动位**:投票 / 提问 / 链接贴纸 → 需要在 60-80% 高度留位
- **主文案** + 字数(B 策略默认)
- **是否有人物**:Y → 主体一致性 Lock

## 七、易错点(Watch For)

- **关键信息进入安全区** — 标题放在最顶 / 最底 → 被平台 header / 操作条遮 → 强制 30-70% 区间
- **横屏构图竖屏化失败** — 直接给 16:9 横构图素材 → 关键焦点跑偏屏幕外 → 必须竖屏重新构图
- **背景留大段死白** — 上下大段空白 → 移动全屏感觉空荡荡 → 用渐变 / 延展背景填满
- **多焦点抢夺** — 多个等大主体 → 单手浏览看不清重点 → 强制单焦点 50%+ 占比
- **文字超出可读尺寸** — 字号 < 80pt(基于 1080×1920)→ 移动竖屏读起来累 → 主文案 ≥ 90pt
- **拼贴感破坏沉浸** — 多张图硬拼 → 失去全屏沉浸 → 改单主体 + 延展背景
- **顶部贴边的 Logo** — Logo 放最顶 → 与平台用户名重叠 → Logo 移到 15-25% 高度
