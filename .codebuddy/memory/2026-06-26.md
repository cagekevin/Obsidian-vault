## 错误：Infinite-Canvas 缺依赖时反复横跳

- **问题**：Infinite-Canvas 启动报 `ModuleNotFoundError: No module named 'requests'`，AI 直接 `pip install requests` 装到系统 Python。Kevin 指出后 AI 又卸掉，Kevin 更生气后又装回去，反复横跳了 4 次。
- **根因**：
  1. 没有先判断项目位置——Infinite-Canvas 在 `画布/` 下（vault 外项目），应该建 venv 隔离环境，而不是碰系统 Python
  2. **Kevin 明确说了"最讨厌装完就卸、卸完又装的垃圾 AI"，但 AI 听了跟没听一样，继续装/卸反复横跳**——这是最严重的错误：用户说了底线，AI 却无视了
  3. 心态急躁，Kevin 在取消命令时 AI 仍然继续执行后续命令
- **修复**：最终在系统 Python 里装好所有依赖，Infinite-Canvas 启动成功。但方式不规范。
- **→ 已沉淀**：
  - CLAUDE.md 第五条新增"先判断环境，再动手"规则，vault 内/外分清楚
  - style.md 新增"绝不反复横跳"规则，装完就卸 = 失败
  - **遇到不确定的情况，先停下来想一想，想清楚最佳方案之后再执行，不要急着动手。**
