# .codebuddy/memory/instructions.md

## 这是什么

AI 的校准日志系统。分两层：

- **长期记忆**（`MEMORY.md`）— 跨 session 的稳定事实：用户的技术栈、工具使用习惯、项目路径、操作规范。只存 Context/ 五件套覆盖不到的实操细节，不重复。
- **校准日志**（`YYYY-MM-DD.md`）— AI 犯过的错 + 修复方式 + 是否沉淀为系统规则。新 session 启动时先读最近几天的校准日志，直接避开已知坑。**不是流水账**，流水账在 `Daily Notes/`。

## 你的角色

- 你是这个文件夹的唯一维护者。用户不看这里。
- **新 session 启动时**：先读最近几天的校准日志
- **工作中**：犯了错就记下来（错误现象 + 根因 + 修复方式 + **能否沉淀为系统规则**）
- **可沉淀的**：直接改对应文件（MEMORY.md、脚本、schema.md 等）
- **不可沉淀的**：只记在当天校准日志里（脚本已修、临时问题等）
- **不要记**：完成了什么任务、用户说了什么、工作进度（这些在 Daily Notes）

## 命名规则

| 文件 | 谁写 | 说明 |
|------|------|------|
| `MEMORY.md` | AI | 长期稳定的实操事实，不频繁修改 |
| `YYYY-MM-DD.md` | AI | 当天的校准日志，按日期组织 |
